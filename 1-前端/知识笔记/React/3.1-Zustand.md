## 一、Zustand 概述

### 1.1 什么是 Zustand

**Zustand**（德语"状态"）是 React 生态中轻量级的状态管理库，由 **pmndrs**（Poimandres）团队维护（同团队还有 Jotai、React Three Fiber 等）。2025 年起 npm 周下载量已超过 Redux，成为 React 社区最流行的状态管理方案。

| 项目         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **定位**     | 轻量、极简的 React 状态管理库                                |
| **包体积**   | ~1KB gzipped，远小于 Redux Toolkit（~11KB）                  |
| **核心理念** | store 就是一个 hook，无 Provider、无样板代码、无严格约束      |
| **适用场景** | 中小型到大型项目的客户端状态管理；配合 React Query 管理服务端状态 |

### 1.2 核心特点

| 特点               | 说明                                                         |
| ------------------ | ------------------------------------------------------------ |
| **极简 API**       | 只有 `create` 一个核心函数，返回一个可直接在组件中使用的 hook |
| **无 Provider**    | 不需要用 `<Provider>` 包裹根组件，任何组件直接调用 store hook 即可 |
| **无样板代码**     | 不需要 action type、reducer、dispatch 等概念                 |
| **原生异步支持**   | action 中直接 async/await，无需中间件                        |
| **按需订阅**       | 通过 selector 精确选取状态，未选取的状态变化不触发重渲染     |
| **框架无关**       | 提供 `createStore` 创建 vanilla store，可在 React 外使用     |

### 1.3 与 Redux（RTK）对比

| 对比项         | **Zustand**                                  | **Redux（RTK）**                                     |
| -------------- | -------------------------------------------- | ---------------------------------------------------- |
| **学习成本**   | 低，几分钟上手                               | 中等，需理解 action / reducer / dispatch / slice 等   |
| **样板代码**   | 极少，定义 + 使用各几行                      | 较多（createSlice + configureStore + Provider + useSelector + useDispatch） |
| **状态修改**   | 在 action 中直接 `set({ ... })`              | dispatch(action) → reducer 纯函数计算新 state        |
| **异步处理**   | action 中直接 async/await                    | 需 thunk / createAsyncThunk / RTK Query              |
| **Provider**   | 不需要                                       | 需要 `<Provider>` 包裹                               |
| **DevTools**   | 中间件接入 Redux DevTools                    | 原生支持                                             |
| **持久化**     | persist 中间件，配置简单                     | 需要第三方库（redux-persist）                        |
| **适用规模**   | 中小型首选，大型也可用                       | 大型 / 企业级 / 多人协作项目有优势                   |
| **生态**       | 轻量，通常配合 React Query 做数据请求        | 内置 RTK Query，一站式方案                           |

> 💡 新项目优先考虑 Zustand + React Query；遇到存量 Redux 项目或面试需要时，再用 RTK。

### 1.4 Vue 3 对照：Pinia

Zustand 的设计理念与 Pinia 非常接近，都是"去掉前辈繁重样板代码"的产物。从 Pinia 切换到 Zustand 几乎无缝。

| 对比项         | **Zustand**                                 | **Pinia**                                            |
| -------------- | ------------------------------------------- | ---------------------------------------------------- |
| **定义 Store** | `create((set, get) => ({ ... }))`           | `defineStore('id', { state, actions, getters })`     |
| **使用 Store** | 组件中调用 `useStore(selector)`             | 组件中调用 `useStore()`，天然响应式                  |
| **getter**     | 无内置概念，用 `get()` 手动派生或写普通函数 | 内置 `getters`，等同 `computed`                      |
| **异步**       | action 中直接 async/await                   | action 中直接 async/await                            |
| **Provider**   | 不需要                                      | `app.use(createPinia())` 注册一次                    |
| **响应式**     | 需 selector 精确选取，避免多余渲染          | Vue 响应式系统自动追踪，天然精确更新                 |
| **DevTools**   | 中间件接入                                  | Vue DevTools 开箱即用                                |
| **持久化**     | persist 中间件                              | pinia-plugin-persistedstate 插件                     |

## 二、基础用法

### 2.1 安装

```bash
npm i zustand
```

Zustand v5 要求 **React 18+**、**TypeScript 4.5+**。无需安装额外的连接库（不像 Redux 还需要 react-redux）。

### 2.2 创建 Store

用 **create** 函数定义 store，参数是一个回调，接收 `set`（更新状态）和 `get`（读取当前状态），返回包含**状态 + action**的对象。`create` 的返回值就是一个 React hook。

```js
// store/counterStore.js
import { create } from 'zustand'

const useCounterStore = create((set, get) => ({
  // ========== 状态 ==========
  count: 0,

  // ========== 同步 action ==========
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
  reset: () => set({ count: 0 }),

  // ========== 带参数的 action ==========
  addTo: (val) => set({ count: val }),

  // ========== 使用 get() 读取当前状态 ==========
  double: () => set({ count: get().count * 2 }),
}))

export default useCounterStore
```

| 参数    | 说明                                                     |
| ------- | -------------------------------------------------------- |
| **set** | 更新状态的函数，支持传对象（浅合并）或函数（基于旧状态计算） |
| **get** | 获取当前最新状态，适合在 action 中读取其他字段           |

### 2.3 组件中使用

直接调用 store hook，通过 **selector** 选取需要的状态或 action：

```jsx
import useCounterStore from './store/counterStore'

function Counter() {
  // selector 精确选取 → 只有 count 变化时才重渲染
  const count = useCounterStore((state) => state.count)
  const increment = useCounterStore((state) => state.increment)
  const addTo = useCounterStore((state) => state.addTo)

  return (
    <div>
      <span>{count}</span>
      <button onClick={increment}>+1</button>
      <button onClick={() => addTo(10)}>设为 10</button>
    </div>
  )
}
```

> **注意**：不写 selector 直接 `useCounterStore()` 会返回整个 store 对象，store 中**任何**状态变化都会触发该组件重渲染。始终用 selector 精确选取。

### 2.4 set 函数详解

`set` 的两种用法：

```js
// 用法一：传对象（浅合并到 state 中）
set({ count: 10 })

// 用法二：传函数（基于旧 state 计算新值，适合依赖当前值的场景）
set((state) => ({ count: state.count + 1 }))
```

`set` 默认是**浅合并**（类似 `Object.assign`），只覆盖传入的字段，不影响其他字段：

```js
// state: { count: 0, name: 'test' }
set({ count: 10 })
// 结果: { count: 10, name: 'test' }  ← name 不受影响
```

如果需要**完全替换**整个 state（而非合并），传第二个参数 `true`：

```js
set({ count: 10 }, true)  // 整个 state 被替换为 { count: 10 }
```

### 2.5 Vue 3 对照：定义与使用

**定义 Store：**

```js
// Zustand：create 一个函数，返回 hook
const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 }))
}))
```

```js
// Pinia：defineStore 定义 state + actions
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  actions: {
    increment() { this.count++ }
  }
})
```

**组件中使用：**

```jsx
// Zustand：selector 选取
const count = useCounterStore((state) => state.count)
const increment = useCounterStore((state) => state.increment)
```

```vue
<!-- Pinia：直接使用 -->
<script setup>
const store = useCounterStore()
const { count } = storeToRefs(store)
store.increment()
</script>
```

| 对比项         | **Zustand**                                | **Pinia**                                          |
| -------------- | ------------------------------------------ | -------------------------------------------------- |
| **状态修改**   | 通过 `set()` 返回新状态片段               | 直接 `this.count++`                                |
| **读取状态**   | selector 手动选取                          | 响应式自动追踪                                     |
| **定义方式**   | 一个函数搞定                               | Options API 或 Setup API 两种风格                  |

## 三、异步操作

### 3.1 异步 action

action 中直接用 async/await，无需中间件或额外 API：

```js
// store/listStore.js
import { create } from 'zustand'
import axios from 'axios'

const useListStore = create((set) => ({
  list: [],
  loading: false,
  error: null,

  fetchList: async () => {
    set({ loading: true, error: null })
    try {
      const res = await axios.get('/api/xxx')
      set({ list: res.data, loading: false })
    } catch (err) {
      set({ error: err.message, loading: false })
    }
  }
}))

export default useListStore
```

```jsx
function ListPage() {
  const list = useListStore((state) => state.list)
  const loading = useListStore((state) => state.loading)
  const fetchList = useListStore((state) => state.fetchList)

  useEffect(() => {
    fetchList()
  }, [fetchList])

  if (loading) return <div>加载中...</div>
  return <ul>{list.map((item) => <li key={item.id}>{item.name}</li>)}</ul>
}
```

### 3.2 与 Redux 对比

| 对比项         | **Zustand**                          | **Redux（createAsyncThunk）**                                |
| -------------- | ------------------------------------ | ------------------------------------------------------------ |
| **写法**       | action 中直接 async/await + `set()`  | `createAsyncThunk` + `extraReducers` 分 pending/fulfilled/rejected |
| **loading 管理** | 在同一个函数中 try/catch 管理      | 在 `extraReducers` 的三个 `addCase` 中分别处理               |
| **中间件**     | 不需要                               | 需要 redux-thunk（RTK 默认集成）                             |
| **组件调用**   | `store.fetchList()`                  | `dispatch(fetchList())`                                      |

> 💡 Zustand 的异步写法与 Pinia 的 async action 几乎一致，从 Vue 过来会非常熟悉。Redux 的 `createAsyncThunk` 将异步流程强制拆分为三个阶段，可追溯性更强但写法更繁琐。

## 四、中间件

### 4.1 中间件机制

Zustand 通过**函数嵌套**的方式使用中间件，中间件包裹在 `create` 的回调外层：

```js
import { create } from 'zustand'
import { devtools, persist } from 'zustand/middleware'

const useStore = create(
  devtools(
    persist(
      (set) => ({
        count: 0,
        increment: () => set((state) => ({ count: state.count + 1 }))
      }),
      { name: 'counter-storage' }   // persist 配置
    ),
    { name: 'CounterStore' }        // devtools 配置
  )
)
```

### 4.2 devtools（Redux DevTools）

接入浏览器 Redux DevTools 扩展，可查看 state 快照和 action 历史：

```js
import { create } from 'zustand'
import { devtools } from 'zustand/middleware'

const useCounterStore = create(
  devtools(
    (set) => ({
      count: 0,
      // 第三个参数为 action name，会显示在 DevTools 中
      increment: () => set(
        (state) => ({ count: state.count + 1 }),
        false,
        'increment'
      )
    }),
    { name: 'CounterStore' }  // DevTools 中显示的 store 名称
  )
)
```

| 配置项       | 说明                                           |
| ------------ | ---------------------------------------------- |
| **name**     | 在 DevTools 中显示的 store 名称                |
| **enabled**  | 是否启用，可按环境控制：`process.env.NODE_ENV === 'development'` |

### 4.3 persist（状态持久化）

将 store 状态自动持久化到 localStorage / sessionStorage 等存储中，刷新页面后自动恢复：

```js
import { create } from 'zustand'
import { persist, createJSONStorage } from 'zustand/middleware'

const useSettingsStore = create(
  persist(
    (set) => ({
      theme: 'light',
      lang: 'zh',
      setTheme: (theme) => set({ theme }),
      setLang: (lang) => set({ lang })
    }),
    {
      name: 'app-settings',                              // 存储 key（必填）
      storage: createJSONStorage(() => localStorage),     // 默认 localStorage
      partialize: (state) => ({ theme: state.theme }),    // 只持久化 theme 字段
    }
  )
)
```

| 配置项           | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| **name**         | 存储的 key 名称（必填）                                     |
| **storage**      | 存储后端，默认 `localStorage`；可换 `sessionStorage` 或自定义 |
| **partialize**   | 选择性持久化，只存需要的字段                                 |
| **version**      | 版本号，配合 `migrate` 做数据迁移                            |
| **migrate**      | 版本变更时的迁移函数                                         |
| **skipHydration** | 跳过自动恢复，适用于 SSR 场景                               |

### 4.4 immer 中间件

Zustand 的 `set` 默认需要返回新状态片段（不可变更新）。引入 **immer 中间件**后，可像 Redux Toolkit 一样直接"修改"状态：

```bash
npm i immer
```

```js
import { create } from 'zustand'
import { immer } from 'zustand/middleware/immer'

const useListStore = create(
  immer((set) => ({
    items: [{ id: 1, name: 'A', done: false }],

    // 可直接修改 state，Immer 底层生成不可变新对象
    toggleItem: (id) => set((state) => {
      const item = state.items.find((i) => i.id === id)
      if (item) item.done = !item.done      // ✅ 直接修改
    }),

    addItem: (item) => set((state) => {
      state.items.push(item)                 // ✅ 直接 push
    })
  }))
)
```

不用 immer 时，同样的操作需要手动写不可变更新：

```js
// ❌ 不用 immer：需要手动展开、映射
toggleItem: (id) => set((state) => ({
  items: state.items.map((item) =>
    item.id === id ? { ...item, done: !item.done } : item
  )
}))
```

> 💡 简单状态（扁平对象、基本类型）不需要 immer；**嵌套对象、数组操作**频繁时用 immer 可大幅简化代码。

### 4.5 中间件组合

多个中间件可以嵌套使用，顺序为**从外到内**：

```js
import { create } from 'zustand'
import { devtools, persist } from 'zustand/middleware'
import { immer } from 'zustand/middleware/immer'

const useStore = create(
  devtools(
    persist(
      immer(
        (set) => ({
          // 状态和 action
        })
      ),
      { name: 'my-storage' }
    ),
    { name: 'MyStore' }
  )
)
```

推荐顺序：`devtools` → `persist` → `immer` → store 定义。

## 五、Store 拆分与组织

### 5.1 多 Store 拆分

Zustand **天然支持多 Store**，每个 `create` 调用生成独立的 store，按业务域拆分即可：

```
store/
├── useCounterStore.js     // 计数相关
├── useUserStore.js        // 用户信息
├── useCartStore.js        // 购物车
└── useSettingsStore.js    // 应用设置
```

```js
// store/useUserStore.js
const useUserStore = create((set) => ({
  user: null,
  token: null,
  login: async (credentials) => {
    const res = await axios.post('/api/login', credentials)
    set({ user: res.data.user, token: res.data.token })
  },
  logout: () => set({ user: null, token: null })
}))
```

各 store 完全独立，组件中按需导入。这和 Pinia 的多 store 模式一致。

### 5.2 Slice 模式（单 Store 拆分）

如果需要将多个模块合并到**同一个 store**（类似 Redux 的 combineReducers），可用 Slice 模式：

```js
// 定义各 slice（普通函数）
const createCounterSlice = (set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 }))
})

const createUserSlice = (set) => ({
  user: null,
  setUser: (user) => set({ user })
})

// 合并到一个 store
const useBoundStore = create((...args) => ({
  ...createCounterSlice(...args),
  ...createUserSlice(...args)
}))
```

| 方式           | 适用场景                                                 |
| -------------- | -------------------------------------------------------- |
| **多 Store**   | 推荐。各 store 独立，按需导入，简单清晰                 |
| **Slice 模式** | 需要跨 slice 读取状态时（通过 `get()` 访问合并后的完整 state） |

### 5.3 与 Redux 对比

| 对比项         | **Zustand**                                      | **Redux（RTK）**                                     |
| -------------- | ------------------------------------------------ | ---------------------------------------------------- |
| **模块定义**   | 每个 `create` 调用就是一个独立 store             | 各写 `createSlice`，需在 `configureStore` 中组合     |
| **根 store**   | 不需要                                           | 必须有，通过 `configureStore` 统一管理               |
| **模块间访问** | 多 store 模式：在一个 store 中导入另一个使用；Slice 模式：通过 `get()` 访问 | 通过 `getState()` 拿到根 state 访问其他模块          |

## 六、Selector 与性能优化

### 6.1 Selector 精确选取

Zustand 的核心优化手段是 **selector**。默认通过 `Object.is` 比较前后值，值不变则不重渲染：

```jsx
// ✅ 精确选取单个字段 → 只有 count 变化时重渲染
const count = useStore((state) => state.count)

// ❌ 不写 selector → store 中任何状态变化都会重渲染
const store = useStore()
```

### 6.2 useShallow（选取多个字段）

需要同时选取多个字段时，selector 每次返回新对象引用（`{}` !== `{}`），导致每次都重渲染。使用 **useShallow** 进行浅比较：

```jsx
import { useShallow } from 'zustand/react/shallow'

// ✅ useShallow 浅比较：只有 count 或 name 的值变化时才重渲染
const { count, name } = useStore(
  useShallow((state) => ({ count: state.count, name: state.name }))
)

// 也可以选取为数组
const [count, name] = useStore(
  useShallow((state) => [state.count, state.name])
)
```

```jsx
// ❌ 不用 useShallow：每次都返回新对象引用，每次 state 变化都重渲染
const { count, name } = useStore((state) => ({ count: state.count, name: state.name }))
```

### 6.3 Vue 3 对照

| 对比项         | **Zustand**                                      | **Pinia / Vue**                                  |
| -------------- | ------------------------------------------------ | ------------------------------------------------ |
| **精确更新**   | 需手动用 selector / useShallow 控制              | Vue 响应式系统自动追踪依赖，天然精确更新         |
| **多字段选取** | 需 `useShallow` 防止多余渲染                     | `storeToRefs` 解构即可，自动响应式               |

> 💡 这是 React 和 Vue 框架层面的差异。React 没有响应式系统，状态变化靠引用比较判断是否重渲染，因此需要开发者通过 selector 手动优化。Vue 的 Proxy 响应式自动追踪，不存在这个问题。

## 七、TypeScript 支持

### 7.1 基本类型定义

Zustand 支持泛型推断，通常不需要手动标注类型。若需要显式定义：

```ts
import { create } from 'zustand'

interface CounterState {
  count: number
  increment: () => void
  addTo: (val: number) => void
}

const useCounterStore = create<CounterState>()((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  addTo: (val) => set({ count: val })
}))
```

> **注意**：使用泛型时需要柯里化写法 `create<State>()(...)` —— 第一个 `()` 传泛型，第二个 `()` 传 store 定义。这是 TypeScript 的限制（无法同时推断和手动指定部分类型参数）。

### 7.2 中间件与 TypeScript

使用中间件时，类型推断可能需要辅助。常见写法：

```ts
import { create } from 'zustand'
import { devtools, persist } from 'zustand/middleware'
import { immer } from 'zustand/middleware/immer'

interface StoreState {
  count: number
  increment: () => void
}

const useStore = create<StoreState>()(
  devtools(
    persist(
      immer((set) => ({
        count: 0,
        increment: () => set((state) => { state.count += 1 })
      })),
      { name: 'my-store' }
    )
  )
)
```

## 八、在 React 外使用（Vanilla Store）

### 8.1 createStore

`createStore` 创建不绑定 React 的 vanilla store，可在非组件代码（工具函数、Node.js、测试）中使用：

```js
import { createStore } from 'zustand/vanilla'

const counterStore = createStore((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 }))
}))

// 在任意 JS 代码中使用
counterStore.getState()            // 获取当前状态
counterStore.setState({ count: 5 }) // 更新状态
counterStore.subscribe(console.log) // 订阅变化
```

### 8.2 在 React 中使用 Vanilla Store

通过 `useStore` hook 将 vanilla store 接入 React 组件：

```jsx
import { useStore } from 'zustand'
import { counterStore } from './store/counterStore'

function Counter() {
  const count = useStore(counterStore, (state) => state.count)
  const increment = useStore(counterStore, (state) => state.increment)
  return <button onClick={increment}>{count}</button>
}
```

| API               | 说明                                                 |
| ----------------- | ---------------------------------------------------- |
| **create**        | 创建绑定 React 的 store（返回 hook），日常开发用这个 |
| **createStore**   | 创建 vanilla store，可在 React 外使用                |
| **useStore**      | 将 vanilla store 接入 React 组件的 hook              |

## 九、常见模式与最佳实践

### 9.1 计算属性（Derived State）

Zustand 没有内置 getter/computed。两种实现方式：

**方式一**：在 store 中用 `get()` 定义普通函数（每次调用重新计算）：

```js
const useCartStore = create((set, get) => ({
  items: [],
  totalPrice: () => get().items.reduce((sum, i) => sum + i.price * i.qty, 0)
}))

// 组件中
const totalPrice = useCartStore((state) => state.totalPrice())
```

**方式二**：selector 中直接计算（推荐，更符合 React 思维）：

```jsx
const totalPrice = useCartStore((state) =>
  state.items.reduce((sum, i) => sum + i.price * i.qty, 0)
)
```

### 9.2 Store 间通信

一个 store 的 action 中可以直接读取另一个 store 的状态：

```js
import useUserStore from './useUserStore'

const useCartStore = create((set, get) => ({
  items: [],
  checkout: async () => {
    const token = useUserStore.getState().token  // 直接读取另一个 store
    await axios.post('/api/checkout', { items: get().items }, {
      headers: { Authorization: token }
    })
    set({ items: [] })
  }
}))
```

> 💡 `useXxxStore.getState()` 可在 React 组件外的任意位置调用，获取该 store 的当前快照。

### 9.3 重置 Store

Zustand 没有内置 `$reset()`（Pinia 有）。常用做法是保存初始状态，手动重置：

```js
const initialState = { count: 0, name: '' }

const useStore = create((set) => ({
  ...initialState,
  increment: () => set((state) => ({ count: state.count + 1 })),
  reset: () => set(initialState)
}))
```
