## 一、方法的介绍

### 1. 什么是方法

方法是一段**具有独立功能的代码块**，可以被重复调用。

    方法 = 一个功能模块
         = 一个可以重复使用的工具
         = 一段封装好的代码

### 2. 为什么使用方法

```java
// ❌ 不使用方法：代码重复
System.out.println("====================");
System.out.println("   欢迎光临");
System.out.println("====================");

// ... 其他代码 ...

System.out.println("====================");
System.out.println("   欢迎光临");
System.out.println("====================");

// ✅ 使用方法：代码复用
public static void welcome() {
    System.out.println("====================");
    System.out.println("   欢迎光临");
    System.out.println("====================");
}

// 调用两次
welcome();
welcome();
```

### 3. 方法的好处

| 好处       | 说明         |
| -------- | ---------- |
| **代码复用** | 写一次，用多次    |
| **结构清晰** | 功能模块化，易于理解 |
| **便于维护** | 修改一处，全局生效  |
| **易于调试** | 问题定位更准确    |

## 二、方法的基本定义和调用

### 1. 最简单的方法定义

```java
public static void 方法名() {
    // 方法体（要执行的代码）
}
```

### 2. 方法的调用

```java
方法名();
```

### 3. 完整示例

```java
public class MethodDemo {
    public static void main(String[] args) {
        // 调用方法（可以多次调用）
        printLine();
        System.out.println("内容");
        printLine();
    }
    
    // 定义方法
    public static void printLine() {
        System.out.println("====================");
    }
}
```

**输出：**

    ====================
    内容
    ====================

### 4. 方法定义的位置

```java
public class MethodDemo {
    
    // 方法定义在类中，与 main 方法平级
    public static void method1() {
        System.out.println("方法1");
    }
    
    public static void main(String[] args) {
        method1();  // 调用方法
        method2();  // 可以调用后面定义的方法
    }
    
    // 方法的位置不影响调用
    public static void method2() {
        System.out.println("方法2");
    }
}
```

### 5. 注意事项

```java
// ❌ 错误：方法不能嵌套定义
public static void main(String[] args) {
    public static void test() {  // 编译错误！
    }
}

// ✅ 正确：方法与方法平级
public static void main(String[] args) {
    test();
}

public static void test() {
}
```

## 三、方法的执行流程

### 1. 执行流程图解

    main 方法开始
        ↓
    代码1
        ↓
    调用 method1() ──────→ 进入 method1()
        ↓                       ↓
    （暂停，等待返回）        执行 method1() 的代码
        ↓                       ↓
    ←────────────────── method1() 执行完毕，返回
        ↓
    代码2
        ↓
    main 方法结束

### 2. 代码示例

```java
public class FlowDemo {
    public static void main(String[] args) {
        System.out.println("1. main 开始");
        
        method1();  // 调用方法，跳转执行
        
        System.out.println("4. main 结束");
    }
    
    public static void method1() {
        System.out.println("2. method1 开始");
        System.out.println("3. method1 结束");
    }
}
```

**输出：**

    1. main 开始
    2. method1 开始
    3. method1 结束
    4. main 结束

### 3. 方法调用方法

```java
public class FlowDemo2 {
    public static void main(String[] args) {
        System.out.println("1. main 开始");
        methodA();
        System.out.println("6. main 结束");
    }
    
    public static void methodA() {
        System.out.println("2. methodA 开始");
        methodB();  // A 调用 B
        System.out.println("5. methodA 结束");
    }
    
    public static void methodB() {
        System.out.println("3. methodB 开始");
        System.out.println("4. methodB 结束");
    }
}
```

**输出：**

    1. main 开始
    2. methodA 开始
    3. methodB 开始
    4. methodB 结束
    5. methodA 结束
    6. main 结束

### 4. 方法调用栈

    调用栈变化（后进先出）：

    时刻1：[main]
    时刻2：[main, methodA]
    时刻3：[main, methodA, methodB]
    时刻4：[main, methodA]（methodB 执行完毕，出栈）
    时刻5：[main]（methodA 执行完毕，出栈）
    时刻6：[]（main 执行完毕，出栈）

### 5. 栈溢出（StackOverflowError）⭐

当方法调用层级过深，超出栈内存限制时，会发生栈溢出。

```java
// 无限递归导致栈溢出
public static void infiniteCall() {
    infiniteCall();  // 自己调用自己，永无止境
}

// 执行结果：
// Exception in thread "main" java.lang.StackOverflowError
```

**栈溢出过程：**

    调用栈不断增长：
    [main, infiniteCall]
    [main, infiniteCall, infiniteCall]
    [main, infiniteCall, infiniteCall, infiniteCall]
    ...（不断增长，直到超出栈内存限制）
    → StackOverflowError

**如何避免：**

*   确保递归有终止条件
*   避免过深的方法调用链
*   可以用循环代替递归

## 四、带参数的方法

### 1. 为什么需要参数

```java
// 问题：只能打印固定次数
public static void printHello() {
    for (int i = 0; i < 3; i++) {
        System.out.println("Hello");
    }
}

// 解决：使用参数，灵活控制次数
public static void printHello(int count) {
    for (int i = 0; i < count; i++) {
        System.out.println("Hello");
    }
}
```

### 2. 定义格式

```java
// 一个参数
public static void 方法名(数据类型 参数名) {
    // 方法体
}

// 多个参数
public static void 方法名(数据类型 参数名1, 数据类型 参数名2) {
    // 方法体
}
```

### 3. 调用格式

```java
方法名(参数值);
方法名(参数值1, 参数值2);
```

### 4. 形参与实参

| 概念     | 说明              | 示例                               |
| ------ | --------------- | -------------------------------- |
| **形参** | 定义方法时的参数（形式参数）  | `void add(int a, int b)` 中的 a, b |
| **实参** | 调用方法时传入的值（实际参数） | `add(10, 20)` 中的 10, 20          |

### 5. 使用示例

```java
public class ParamDemo {
    public static void main(String[] args) {
        // 调用带参数的方法
        sayHello("张三");
        sayHello("李四");
        
        // 多个参数
        add(10, 20);
        
        // 使用变量作为参数
        String name = "王五";
        sayHello(name);
    }
    
    // 一个参数
    public static void sayHello(String name) {
        System.out.println("你好，" + name);
    }
    
    // 两个参数
    public static void add(int a, int b) {
        int sum = a + b;
        System.out.println(a + " + " + b + " = " + sum);
    }
}
```

### 6. 参数传递的本质：值传递 ⭐

**Java 中所有参数传递都是值传递（传递的是值的副本）**

#### 基本类型：传递值的副本

```java
public static void main(String[] args) {
    int num = 10;
    change(num);
    System.out.println(num);  // 仍然是 10
}

public static void change(int num) {
    num = 20;  // 修改的是副本，不影响原值
}
```

    内存图解：
    main:  num = 10  ──→  change:  num = 10（副本）
                                        ↓
                                   num = 20（修改副本）
    main:  num = 10（不变）

#### 引用类型：传递地址值的副本

```java
public static void main(String[] args) {
    int[] arr = {10, 20, 30};
    change(arr);
    System.out.println(arr[0]);  // 100（被修改了）
}

public static void change(int[] arr) {
    arr[0] = 100;  // 通过地址修改堆中的数据
}
```

    内存图解：
    main:  arr = 0x001 ──→ 堆内存：[10, 20, 30]
              ↓
    change: arr = 0x001（地址副本）
              ↓
           arr[0] = 100 ──→ 堆内存：[100, 20, 30]

    main:  arr[0] = 100（堆中数据被修改）

#### 引用类型：重新赋值不影响原引用

```java
public static void main(String[] args) {
    int[] arr = {10, 20, 30};
    changeRef(arr);
    System.out.println(arr[0]);  // 10（没变）
}

public static void changeRef(int[] arr) {
    arr = new int[]{100, 200, 300};  // 新建对象，修改的是副本指向
}
```

    内存图解：
    main:  arr = 0x001 ──→ 堆内存 0x001：[10, 20, 30]
              ↓
    change: arr = 0x001（地址副本）
              ↓
           arr = 0x002 ──→ 堆内存 0x002：[100, 200, 300]（新对象）

    main:  arr = 0x001（仍指向原对象）

### 7. 参数传递总结

| 参数类型 | 传递内容  | 修改形参的影响      |
| ---- | ----- | ------------ |
| 基本类型 | 值的副本  | 不影响实参        |
| 引用类型 | 地址的副本 | 修改对象内容会影响原对象 |
| 引用类型 | 地址的副本 | 重新赋值不影响原引用   |

**记忆口诀**：Java 只有值传递，引用类型传地址值

## 五、带返回值的方法

### 1. 为什么需要返回值

```java
// 问题：计算结果只能在方法内部使用
public static void add(int a, int b) {
    int sum = a + b;
    System.out.println(sum);  // 只能打印，不能给外部使用
}

// 解决：使用返回值，把结果返回给调用者
public static int add(int a, int b) {
    int sum = a + b;
    return sum;  // 返回结果
}
```

### 2. 定义格式

```java
public static 返回值类型 方法名(参数列表) {
    // 方法体
    return 返回值;
}
```

**说明：**

*   `返回值类型`：方法返回的数据类型（int, double, String 等）
*   `return`：结束方法并返回数据
*   `返回值`：要返回的具体数据，必须与返回值类型匹配

### 3. 调用格式

```java
// 方式1：直接调用（不推荐，返回值被忽略）
方法名(参数);

// 方式2：赋值调用（推荐）
数据类型 变量名 = 方法名(参数);

// 方式3：输出调用
System.out.println(方法名(参数));

// 方式4：作为其他方法的参数
其他方法(方法名(参数));
```

### 4. 使用示例

```java
public class ReturnDemo {
    public static void main(String[] args) {
        // 方式1：直接调用（不推荐）
        add(10, 20);  // 返回值被丢弃
        
        // 方式2：赋值调用（推荐）
        int result = add(10, 20);
        System.out.println("结果：" + result);
        
        // 方式3：输出调用
        System.out.println("结果：" + add(30, 40));
        
        // 方式4：作为其他方法的参数
        int max = Math.max(add(10, 20), add(5, 15));
        
        // 返回值参与其他运算
        int sum1 = add(10, 20);
        int sum2 = add(30, 40);
        int total = sum1 + sum2;
        System.out.println("总和：" + total);
    }
    
    // 带返回值的方法
    public static int add(int a, int b) {
        int sum = a + b;
        return sum;
    }
}
```

### 5. return 的作用

```java
// 1. 返回数据
public static int getMax(int a, int b) {
    return (a > b) ? a : b;
}

// 2. 结束方法（void 方法中也可以使用）
public static void printPositive(int num) {
    if (num < 0) {
        return;  // 提前结束方法
    }
    System.out.println(num);
}

// 3. return 后面的代码不会执行
public static int test() {
    return 10;
    // System.out.println("不会执行");  // 编译错误
}
```

### 6. 各种返回类型示例

```java
// 返回 int
public static int getInt() {
    return 10;
}

// 返回 double
public static double getDouble() {
    return 3.14;
}

// 返回 boolean
public static boolean getBoolean() {
    return true;
}

// 返回 String
public static String getString() {
    return "Hello";
}

// 返回数组
public static int[] getArray() {
    return new int[]{1, 2, 3};
}

// 返回对象
public static User getUser() {
    return new User("张三", 18);
}
```

## 六、方法通用定义格式

### 1. 完整格式

```java
修饰符 返回值类型 方法名(参数类型 参数名1, 参数类型 参数名2, ...) {
    方法体;
    return 返回值;
}
```

### 2. 各部分说明

| 部分         | 说明                     | 示例                      |
| ---------- | ---------------------- | ----------------------- |
| **修饰符**    | 目前固定写 `public static`  | `public static`         |
| **返回值类型**  | 方法返回的数据类型，无返回值写 `void` | `int`, `void`, `String` |
| **方法名**    | 标识符，小驼峰命名              | `add`, `getUserName`    |
| **参数列表**   | 多个参数用逗号分隔，无参数留空        | `(int a, int b)`, `()`  |
| **方法体**    | 要执行的代码                 | `int sum = a + b;`      |
| **return** | 返回数据并结束方法              | `return sum;`           |

### 3. 方法分类

| 类型         | 定义                                        | 调用                       |
| ---------- | ----------------------------------------- | ------------------------ |
| **无参无返回值** | `void sayHello() { }`                     | `sayHello();`            |
| **有参无返回值** | `void sayHello(String name) { }`          | `sayHello("张三");`        |
| **无参有返回值** | `int getNumber() { return 10; }`          | `int n = getNumber();`   |
| **有参有返回值** | `int add(int a, int b) { return a + b; }` | `int sum = add(10, 20);` |

### 4. 方法定义技巧

**确定两点：**

1.  **明确返回值类型**：方法需要返回什么类型的数据？
2.  **明确参数列表**：方法需要什么数据才能完成功能？

```java
// 案例：判断两个数是否相等
// 分析：
// 1. 返回值类型：boolean（相等/不相等）
// 2. 参数列表：两个 int（要比较的数）

public static boolean isEqual(int a, int b) {
    return a == b;
}
```

### 5. 常见方法模板

```java
// 模板1：打印类（无返回值）
public static void print数据(数据类型 参数) {
    System.out.println(参数);
}

// 模板2：计算类（有返回值）
public static 返回类型 calculate(数据类型 参数1, 数据类型 参数2) {
    // 计算过程
    return 结果;
}

// 模板3：判断类（返回 boolean）
public static boolean is条件(数据类型 参数) {
    // 判断逻辑
    return true或false;
}

// 模板4：获取类（有返回值）
public static 返回类型 get数据() {
    // 获取数据
    return 数据;
}

// 模板5：转换类（有返回值）
public static 目标类型 toXxx(源类型 参数) {
    // 转换逻辑
    return 转换结果;
}
```

## 七、方法注意事项

### 1. 方法不调用不执行

```java
public static void main(String[] args) {
    // 没有调用 method，method 不会执行
    System.out.println("main 方法");
}

public static void method() {
    System.out.println("method 方法");  // 不会输出
}
```

### 2. 方法之间是平级关系

```java
// ❌ 错误：方法不能嵌套定义
public static void method1() {
    public static void method2() {  // 编译错误
    }
}

// ✅ 正确：方法平级定义
public static void method1() {
}

public static void method2() {
}
```

### 3. 方法的执行顺序与定义顺序无关

```java
public static void main(String[] args) {
    method2();  // 可以调用后面定义的方法
    method1();
}

public static void method1() { }
public static void method2() { }
```

### 4. 返回值类型必须匹配

```java
// ❌ 错误：返回值类型不匹配
public static int getNumber() {
    return 3.14;  // 编译错误：需要 int，提供了 double
}

// ❌ 错误：void 方法不能返回数据
public static void test() {
    return 10;  // 编译错误
}

// ✅ 正确
public static double getNumber() {
    return 3.14;
}
```

### 5. 有返回值的方法必须有 return

```java
// ❌ 错误：缺少 return 语句
public static int getNumber() {
    int num = 10;
    // 编译错误：缺少 return 语句
}

// ❌ 错误：不是所有路径都有 return
public static int getMax(int a, int b) {
    if (a > b) {
        return a;
    }
    // 编译错误：如果 a <= b，没有 return
}

// ✅ 正确：所有路径都有 return
public static int getMax(int a, int b) {
    if (a > b) {
        return a;
    } else {
        return b;
    }
}

// ✅ 更简洁的写法
public static int getMax(int a, int b) {
    return (a > b) ? a : b;
}
```

### 6. return 后面不能有代码

```java
// ❌ 错误：return 后面有代码
public static int test() {
    return 10;
    System.out.println("不会执行");  // 编译错误：无法访问的语句
}
```

### 7. 参数必须匹配

```java
public static void show(int a, String b) {
    System.out.println(a + ", " + b);
}

// 调用时参数必须匹配：数量、类型、顺序
show(10, "hello");     // ✅ 正确
// show(10);           // ❌ 错误：参数数量不对
// show("hello", 10);  // ❌ 错误：参数顺序不对
// show(10, 20);       // ❌ 错误：参数类型不对
```

### 8. 注意事项总结表

| 注意事项                | 说明              |
| ------------------- | --------------- |
| 方法不调用不执行            | 定义了不调用不会运行      |
| 方法不能嵌套定义            | 方法之间是平级关系       |
| 执行顺序与定义顺序无关         | 可以调用后面定义的方法     |
| void 方法不能有 return 值 | 可以有空的 `return;` |
| 有返回值必须有 return      | 所有分支都要有 return  |
| return 后不能有代码       | 无法访问的语句         |
| 参数必须匹配              | 数量、类型、顺序都要匹配    |

## 八、方法重载（Overload）

### 1. 什么是方法重载

方法重载是指在**同一个类中**，**方法名相同**，但**参数列表不同**的多个方法。

```java
// 这就是方法重载
public static int add(int a, int b) {
    return a + b;
}

public static int add(int a, int b, int c) {
    return a + b + c;
}

public static double add(double a, double b) {
    return a + b;
}
```

### 2. 方法重载的条件

**必须满足：**

1.  在同一个类中
2.  方法名相同
3.  参数列表不同（数量不同、类型不同、顺序不同）

**与以下无关：**

*   返回值类型
*   参数名称
*   访问修饰符

### 3. 参数列表不同的情况

```java
// 1. 参数数量不同
public static void show(int a) { }
public static void show(int a, int b) { }

// 2. 参数类型不同
public static void show(int a) { }
public static void show(double a) { }

// 3. 参数顺序不同
public static void show(int a, double b) { }
public static void show(double a, int b) { }
```

### 4. 不构成重载的情况

```java
// ❌ 只有参数名不同，不是重载
public static void show(int a) { }
public static void show(int b) { }  // 编译错误：重复定义

// ❌ 只有返回值不同，不是重载
public static void show(int a) { }
public static int show(int a) { return a; }  // 编译错误

// ❌ 只有修饰符不同，不是重载
public static void show(int a) { }
private static void show(int a) { }  // 编译错误
```

### 5. 重载方法的匹配规则 ⭐

当调用重载方法时，JVM 按以下顺序匹配：

```java
public static void test(int a) {
    System.out.println("int");
}

public static void test(double a) {
    System.out.println("double");
}

public static void test(Integer a) {
    System.out.println("Integer");
}

public static void test(int... a) {
    System.out.println("int...");
}

public static void main(String[] args) {
    test(10);    // int（精确匹配优先）
    test(10.0);  // double
    test(10L);   // double（自动类型提升：long → double）
}
```

**匹配优先级（从高到低）：**

1.  **精确匹配**：参数类型完全一致
2.  **自动类型提升**：byte → short → int → long → float → double
3.  **自动装箱/拆箱**：int ↔ Integer
4.  **可变参数**：最后匹配

```java
test(10);        // 优先匹配 test(int)
test((byte) 10); // 没有 test(byte)，提升为 int，匹配 test(int)
test(10L);       // 没有 test(long)，提升为 double，匹配 test(double)
```

## 九、方法递归（Recursion）

### 1. 什么是递归

递归是指**方法调用自身**的编程技巧。

```java
public static void method() {
    method();  // 自己调用自己
}
```

### 2. 递归的两个要素

| 要素       | 说明          | 缺失后果       |
| -------- | ----------- | ---------- |
| **终止条件** | 什么时候停止递归    | 无限递归 → 栈溢出 |
| **递推公式** | 大问题如何拆解为小问题 | 无法正确计算     |

### 3. 经典案例：计算阶乘

**数学定义**：n! = n × (n-1) × ... × 2 × 1

```java
// 递归实现
public static int factorial(int n) {
    // 终止条件
    if (n == 1) {
        return 1;
    }
    // 递推公式：n! = n × (n-1)!
    return n * factorial(n - 1);
}

// 调用
System.out.println(factorial(5));  // 120
```

**执行过程：**

    factorial(5)
    = 5 * factorial(4)
    = 5 * 4 * factorial(3)
    = 5 * 4 * 3 * factorial(2)
    = 5 * 4 * 3 * 2 * factorial(1)
    = 5 * 4 * 3 * 2 * 1
    = 120

### 4. 经典案例：斐波那契数列

**数学定义**：F(1)=1, F(2)=1, F(n)=F(n-1)+F(n-2)

```java
// 递归实现（效率较低）
public static int fib(int n) {
    if (n == 1 || n == 2) {
        return 1;
    }
    return fib(n - 1) + fib(n - 2);
}

// 优化：使用缓存（记忆化递归）
public static int fibMemo(int n, int[] memo) {
    if (n == 1 || n == 2) {
        return 1;
    }
    if (memo[n] != 0) {
        return memo[n];  // 已计算过，直接返回
    }
    memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
    return memo[n];
}
```

### 5. 经典案例：递归求和

```java
// 计算 1 + 2 + 3 + ... + n
public static int sum(int n) {
    if (n == 1) {
        return 1;
    }
    return n + sum(n - 1);
}

System.out.println(sum(100));  // 5050
```

### 6. 递归 vs 循环

```java
// ========== 计算阶乘 ==========

// 递归方式
public static int factorialRecursive(int n) {
    if (n == 1) return 1;
    return n * factorialRecursive(n - 1);
}

// 循环方式
public static int factorialLoop(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

| 对比项   | 递归          | 循环     |
| ----- | ----------- | ------ |
| 代码简洁性 | 简洁，接近数学定义   | 相对繁琐   |
| 效率    | 较低（有方法调用开销） | 较高     |
| 内存占用  | 高（调用栈）      | 低      |
| 栈溢出风险 | 有（递归过深）     | 无      |
| 适用场景  | 树、图遍历，分治算法  | 简单重复操作 |

### 7. 递归注意事项

```java
// ❌ 错误：没有终止条件
public static void bad() {
    bad();  // 无限递归 → StackOverflowError
}

// ❌ 错误：终止条件不可达
public static void bad2(int n) {
    if (n == 0) return;
    bad2(n + 1);  // n 永远不会变成 0
}

// ✅ 正确：有终止条件，且每次递归都向终止条件靠近
public static void good(int n) {
    if (n <= 0) return;  // 终止条件
    good(n - 1);         // 每次 n 减小，逐渐接近终止条件
}
```

### 8. 递归应用场景

| 场景     | 示例          |
| ------ | ----------- |
| 数学计算   | 阶乘、斐波那契、幂运算 |
| 数据结构遍历 | 树的遍历、图的 DFS |
| 分治算法   | 归并排序、快速排序   |
| 回溯算法   | 八皇后、迷宫问题    |
| 文件操作   | 遍历目录        |

```java
// 遍历目录（实际应用）
public static void listFiles(File dir) {
    if (dir == null || !dir.exists()) return;
    
    File[] files = dir.listFiles();
    if (files == null) return;
    
    for (File file : files) {
        if (file.isDirectory()) {
            listFiles(file);  // 递归遍历子目录
        } else {
            System.out.println(file.getAbsolutePath());
        }
    }
}
```

## 十、可变参数（Varargs）

### 1. 什么是可变参数

可变参数是 JDK 5 引入的特性，允许方法接收**数量不定**的参数。

**语法**：`数据类型...`（类型后面加三个点）

```java
public static int add(int... nums) {
    // nums 本质是一个数组
}
```

### 2. 为什么需要可变参数

```java
// ❌ 没有可变参数：需要定义多个重载方法
public static int add(int a, int b) {
    return a + b;
}

public static int add(int a, int b, int c) {
    return a + b + c;
}

public static int add(int a, int b, int c, int d) {
    return a + b + c + d;
}

// ✅ 使用可变参数：一个方法搞定
public static int add(int... nums) {
    int sum = 0;
    for (int num : nums) {
        sum += num;
    }
    return sum;
}
```

### 3. 基本使用

```java
public class VarargsDemo {
    public static void main(String[] args) {
        // 传递不同数量的参数
        System.out.println(add());           // 0
        System.out.println(add(10));         // 10
        System.out.println(add(10, 20));     // 30
        System.out.println(add(10, 20, 30)); // 60
        System.out.println(add(1, 2, 3, 4, 5)); // 15
    }
    
    public static int add(int... nums) {
        int sum = 0;
        for (int num : nums) {
            sum += num;
        }
        return sum;
    }
}
```

### 4. 可变参数的本质

**可变参数本质是数组**，编译后会转换为数组。

```java
public static void test(int... nums) {
    // nums 就是 int[] 数组
    System.out.println(nums.getClass().getName());  // [I（int 数组）
    System.out.println(nums.length);  // 数组长度
    
    // 可以像数组一样使用
    for (int i = 0; i < nums.length; i++) {
        System.out.println(nums[i]);
    }
}
```

### 5. 调用方式

```java
public static void show(int... nums) {
    for (int num : nums) {
        System.out.print(num + " ");
    }
    System.out.println();
}

// 调用方式1：传递多个参数
show(1, 2, 3);        // 1 2 3

// 调用方式2：不传参数（0 个）
show();               // （空）

// 调用方式3：直接传递数组
int[] arr = {10, 20, 30};
show(arr);            // 10 20 30
```

### 6. 注意事项

#### 6.1 可变参数必须放在参数列表最后

```java
// ✅ 正确：可变参数在最后
public static void method(String name, int... nums) { }

// ❌ 错误：可变参数不在最后
public static void method(int... nums, String name) { }
// 编译错误：Vararg parameter must be the last in the list
```

#### 6.2 一个方法只能有一个可变参数

```java
// ✅ 正确：只有一个可变参数
public static void method(int... nums) { }

// ❌ 错误：有两个可变参数
public static void method(int... nums, String... strs) { }
// 编译错误
```

#### 6.3 不传参数时是空数组，不是 null

```java
public static void test(int... nums) {
    System.out.println(nums == null);  // false
    System.out.println(nums.length);   // 0
}

test();  // nums 是空数组 []，不是 null
```

### 7. 与其他参数组合

```java
public class CombineDemo {
    public static void main(String[] args) {
        print("数字", 1, 2, 3, 4, 5);
        print("成绩", 90, 85, 95);
        print("空");
    }
    
    // 可变参数必须在最后
    public static void print(String prefix, int... nums) {
        System.out.print(prefix + ": ");
        for (int num : nums) {
            System.out.print(num + " ");
        }
        System.out.println();
    }
}
```

**输出：**

    数字: 1 2 3 4 5 
    成绩: 90 85 95 
    空: 

### 8. 可变参数 vs 数组参数

| 对比项      | 可变参数 `int... nums` | 数组参数 `int[] nums`            |
| -------- | ------------------ | ---------------------------- |
| **调用方式** | `method(1, 2, 3)`  | `method(new int[]{1, 2, 3})` |
| **不传参数** | `method()` ✅       | ❌ 必须传数组或 null                |
| **传递数组** | `method(arr)` ✅    | `method(arr)` ✅              |
| **灵活性**  | 更灵活                | 较繁琐                          |
| **本质**   | 编译后变成数组            | 就是数组                         |
