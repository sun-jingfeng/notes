## 一、面向对象思想

### 1. 什么是面向对象

面向对象编程（OOP，Object-Oriented Programming）是一种**以对象为核心**的编程思想。

### 2. 两种编程思想对比

| 编程思想     | 核心    | 思维方式          | 例子                   |
| -------- | ----- | ------------- | -------------------- |
| **面向过程** | 步骤、过程 | 亲力亲为，关注每一步怎么做 | 自己做饭：买菜→洗菜→切菜→炒菜→装盘  |
| **面向对象** | 对象、职责 | 找对象，让对象做事     | 点外卖：找商家（对象），调用"送餐"功能 |

### 3. 面向对象三大特征

| 特征     | 说明              | 关键字                     |
| ------ | --------------- | ----------------------- |
| **封装** | 隐藏内部细节，对外提供公共接口 | `private`、getter/setter |
| **继承** | 子类继承父类的属性和方法    | `extends`               |
| **多态** | 同一方法在不同对象上有不同表现 | 重写、父类引用指向子类对象           |

### 4. 面向对象的优点

| 优点       | 说明               |
| -------- | ---------------- |
| **可维护性** | 代码结构清晰，易于理解和修改   |
| **可扩展性** | 通过继承和多态，方便添加新功能  |
| **可重用性** | 类可以被多次实例化，代码可以复用 |
| **灵活性**  | 通过多态和接口，降低模块间耦合度 |

## 二、类和对象

### 1. 基本概念

| 概念             | 说明                 | 现实类比   |
| -------------- | ------------------ | ------ |
| **类（Class）**   | 对象的模板/蓝图，描述共同特征和行为 | 汽车设计图纸 |
| **对象（Object）** | 类的具体实例，真实存在的个体     | 你开的那辆车 |

**关系**：类是抽象的概念，对象是具体的实体。先定义类，再创建对象。

### 2. 类的定义

```java
public class Phone {
    // ========== 成员变量（属性）- 描述对象"有什么" ==========
    String brand;   // 品牌
    double price;   // 价格
    String color;   // 颜色
    
    // ========== 成员方法（行为）- 描述对象"能做什么" ==========
    public void call(String name) {
        System.out.println("给" + name + "打电话");
    }
    
    public void playGame() {
        System.out.println("玩游戏");
    }
}
```

### 3. 对象的创建和使用

```java
public class PhoneTest {
    public static void main(String[] args) {
        // 1. 创建对象：类名 对象名 = new 类名();
        Phone p = new Phone();
        
        // 2. 赋值：对象名.变量名 = 值;
        p.brand = "华为";
        p.price = 5999.0;
        p.color = "黑色";
        
        // 3. 取值：对象名.变量名
        System.out.println(p.brand);  // 华为
        
        // 4. 调用方法：对象名.方法名(参数);
        p.call("张三");    // 给张三打电话
        p.playGame();      // 玩游戏
    }
}
```

### 4. 对象的默认值

创建对象后，成员变量会自动赋默认值：

| 数据类型 | 默认值           |
| ---- | ------------- |
| 整数类型 | `0`           |
| 浮点类型 | `0.0`         |
| 布尔类型 | `false`       |
| 字符类型 | `\u0000`（空字符） |
| 引用类型 | `null`        |

### 5. null 引用

```java
Phone p = null;  // 引用不指向任何对象

// p.brand = "华为";  ❌ NullPointerException（空指针异常）

// 使用前建议判空
if (p != null) {
    p.call("张三");
}
```

### 6. 对象的比较

```java
Phone p1 = new Phone();
Phone p2 = new Phone();
Phone p3 = p1;

// == 比较的是地址值（是否是同一个对象）
System.out.println(p1 == p2);  // false（不同对象）
System.out.println(p1 == p3);  // true（同一个对象）

// equals() 默认也是比较地址，需要重写才能比较内容
System.out.println(p1.equals(p2));  // false（未重写）
```

## 三、类和对象内存图

### 1. JVM 内存区域

| 区域                   | 存储内容                  | JDK 8+ 变化 |
| -------------------- | --------------------- | --------- |
| **栈内存（Stack）**       | 方法执行、局部变量、对象引用（地址值）   | 无变化       |
| **堆内存（Heap）**        | new 出来的对象、成员变量        | 无变化       |
| **方法区（Method Area）** | .class 字节码文件、静态变量、常量池 | 变为**元空间** |

**JDK 8+ 变化**：方法区由 JVM 内存变为**元空间（Metaspace）**，使用本地内存，不再受 JVM 堆大小限制。

## 四、成员变量和局部变量的区别

| 区别       | 成员变量                        | 局部变量             |
| -------- | --------------------------- | ---------------- |
| **定义位置** | 类中，方法外                      | 方法中、方法参数、代码块中    |
| **内存位置** | 堆内存（随对象）                    | 栈内存（随方法）         |
| **生命周期** | 随对象创建而存在，随对象消失而消失           | 随方法调用而存在，方法结束就消失 |
| **默认值**  | 有（整数0，浮点0.0，布尔false，引用null） | 无，必须先赋值才能使用      |
| **作用范围** | 整个类中                        | 只在所属的 {} 内有效     |

```java
public class Student {
    // 成员变量（类中方法外）
    String name;          // 默认值 null
    int age;              // 默认值 0
    
    public void study() {
        // 局部变量（方法内）
        int hours = 8;    // 必须赋值！
        System.out.println(name + "学习了" + hours + "小时");
    }
    
    public void sleep() {
        // System.out.println(hours);  // 错误！hours 在这里不可见
    }
}
```

## 五、this 关键字

### 1. 什么是 this

`this` 代表**当前对象的引用**。谁调用这个方法，`this` 就代表谁。

### 2. 主要用途：区分同名变量

```java
public class Student {
    private String name;
    private int age;
    
    public void setName(String name) {
        // name = name;         // 错误！局部变量自己赋值给自己
        this.name = name;       // 正确！this.name 是成员变量
    }
    
    public void setAge(int age) {
        this.age = age;         // this.age 是成员变量，age 是参数
    }
}
```

### 3. this 调用构造方法

```java
public class Student {
    private String name;
    private int age;
    
    // this() 调用本类其他构造方法（必须放在第一行）
    public Student() {
        this("默认名", 0);   // 调用有参构造
    }
    
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

### 4. this 实现链式调用

```java
public class Builder {
    private String name;
    private int age;
    
    // 返回 this，支持链式调用
    public Builder setName(String name) {
        this.name = name;
        return this;  // 返回当前对象
    }
    
    public Builder setAge(int age) {
        this.age = age;
        return this;
    }
    
    public void print() {
        System.out.println(name + ", " + age);
    }
}

// 使用链式调用
new Builder()
    .setName("张三")
    .setAge(18)
    .print();  // 张三, 18
```

## 六、构造方法

### 1. 什么是构造方法

构造方法（Constructor）是一种特殊方法，在**创建对象时自动调用**，用于初始化对象。

### 2. 构造方法的特点

| 特点   | 说明                      |
| ---- | ----------------------- |
| 方法名  | **必须与类名完全相同**           |
| 返回值  | **没有返回值类型**（连 void 都不写） |
| 调用时机 | 创建对象时自动调用               |
| 默认提供 | 不写任何构造方法时，系统提供默认无参构造    |

### 3. 构造方法示例

```java
public class Student {
    private String name;
    private int age;
    
    // 无参构造方法
    public Student() {
        System.out.println("无参构造被调用");
    }
    
    // 带参构造方法
    public Student(String name) {
        this.name = name;
    }
    
    // 全参构造方法
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

### 4. 构造方法链（this 调用）

```java
public class Person {
    private String name;
    private int age;
    private String address;
    
    public Person() {
        this("未知");  // 调用一参构造
    }
    
    public Person(String name) {
        this(name, 0);  // 调用二参构造
    }
    
    public Person(String name, int age) {
        this(name, age, "未知");  // 调用三参构造
    }
    
    public Person(String name, int age, String address) {
        // 最终的初始化逻辑
        this.name = name;
        this.age = age;
        this.address = address;
    }
}
```

### 5. 重要注意事项

| 注意点    | 说明                      |
| ------ | ----------------------- |
| 默认构造   | 不写任何构造方法 → 系统提供无参构造     |
| 覆盖规则   | **写了有参构造 → 系统不再提供无参构造** |
| 推荐写法   | **无论是否使用，都手动写出无参构造**    |
| 方法重载   | 可以定义多个构造方法，参数不同即可       |
| this() | 必须放在构造方法第一行，且不能循环调用     |

## 七、封装

### 1. 什么是封装

封装是面向对象三大特征之一：**隐藏内部实现细节，对外提供公共访问方式**。

### 2. 封装的好处

| 好处  | 说明                 |
| --- | ------------------ |
| 安全性 | 防止外部随意访问和修改数据      |
| 可控性 | 可以在 setter 中加入数据校验 |
| 灵活性 | 内部实现可以随时修改，不影响外部调用 |

### 3. 封装的实现

**两步走**：

1.  `private` 修饰成员变量（私有化）
2.  提供 `public` 的 getter/setter 方法

```java
public class Student {
    // 1. private 私有化成员变量
    private String name;
    private int age;
    
    // 2. 提供 public 的 getter 方法（获取值）
    public String getName() {
        return name;
    }
    
    // 3. 提供 public 的 setter 方法（设置值）
    public void setName(String name) {
        this.name = name;
    }
    
    public int getAge() {
        return age;
    }
    
    // 可以在 setter 中加入校验逻辑
    public void setAge(int age) {
        if (age < 0 || age > 150) {
            System.out.println("年龄不合法！");
            return;
        }
        this.age = age;
    }
}
```

### 4. boolean 类型的 getter

```java
public class User {
    private boolean active;
    
    // boolean 类型的 getter 推荐用 isXxx 命名
    public boolean isActive() {
        return active;
    }
    
    public void setActive(boolean active) {
        this.active = active;
    }
}
```

## 八、JavaBean

### 1. 什么是 JavaBean

JavaBean 是符合特定规范的 Java 类，主要用于**封装数据**（实体类）。

### 2. JavaBean 标准规范

| 要求   | 说明                                             |
| ---- | ---------------------------------------------- |
| 成员变量 | 使用 **private** 修饰                              |
| 构造方法 | 提供**无参构造**（必须）+ 全参构造（建议）                       |
| 成员方法 | 提供 **getter 和 setter** 方法                      |
| 可选   | toString()、equals()、hashCode()、实现 Serializable |

### 3. 标准 JavaBean 示例

```java
import java.io.Serializable;
import java.util.Objects;

public class User implements Serializable {
    // 序列化版本号（建议添加）
    private static final long serialVersionUID = 1L;
    
    // ========== 1. 私有成员变量 ==========
    private String username;
    private String password;
    private int age;
    
    // ========== 2. 无参构造（必须） ==========
    public User() {
    }
    
    // ========== 3. 全参构造（建议） ==========
    public User(String username, String password, int age) {
        this.username = username;
        this.password = password;
        this.age = age;
    }
    
    // ========== 4. getter 和 setter ==========
    public String getUsername() {
        return username;
    }
    
    public void setUsername(String username) {
        this.username = username;
    }
    
    public String getPassword() {
        return password;
    }
    
    public void setPassword(String password) {
        this.password = password;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        this.age = age;
    }
    
    // ========== 5. toString()（建议） ==========
    @Override
    public String toString() {
        return "User{username='" + username + "', age=" + age + "}";
    }
    
    // ========== 6. equals() 和 hashCode()（建议） ==========
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        User user = (User) o;
        return age == user.age && 
               Objects.equals(username, user.username);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(username, age);
    }
}
```

## 九、Lombok

### 1. 什么是 Lombok

Lombok 是一个 Java 库，通过**注解**自动生成 getter/setter、构造方法、toString 等样板代码。

| 对比项           | 传统 JavaBean | 使用 Lombok                                    |
| ------------- | ----------- | -------------------------------------------- |
| getter/setter | 手动写几十行      | 一个 `@Data` 搞定                                |
| 构造方法          | 手动写         | `@NoArgsConstructor` / `@AllArgsConstructor` |
| toString      | 手动重写        | `@ToString` 自动生成                             |
| 代码量           | 几十行         | 几行                                           |

### 2. 添加 Lombok 依赖

```xml
<!-- Maven 依赖 -->
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.30</version>
    <scope>provided</scope>
</dependency>
```

**IDEA 配置**：

1.  安装 Lombok 插件：`Settings → Plugins → 搜索 Lombok → Install`
2.  开启注解处理：`Settings → Build → Compiler → Annotation Processors → Enable`

### 3. 常用注解

| 注解                         | 作用                                                                                  |
| -------------------------- | ----------------------------------------------------------------------------------- |
| `@Getter`                  | 生成所有字段的 getter 方法                                                                   |
| `@Setter`                  | 生成所有字段的 setter 方法                                                                   |
| `@ToString`                | 生成 toString() 方法                                                                    |
| `@EqualsAndHashCode`       | 生成 equals() 和 hashCode() 方法                                                         |
| `@NoArgsConstructor`       | 生成无参构造方法                                                                            |
| `@AllArgsConstructor`      | 生成全参构造方法                                                                            |
| `@RequiredArgsConstructor` | 生成包含 final 和 @NonNull 字段的构造方法                                                       |
| **`@Data`**                | **= @Getter + @Setter + @ToString + @EqualsAndHashCode + @RequiredArgsConstructor** |
| `@Value`                   | 不可变类（final 类 + final 字段 + 只有 getter）                                                |
| `@Builder`                 | 生成 Builder 模式代码                                                                     |
| `@Slf4j` / `@Log4j2`       | 生成日志对象 log                                                                          |
| `@Accessors`               | 配置 getter/setter 的风格                                                                |

### 4. 使用示例

#### 传统写法 vs Lombok

```java
// ❌ 传统 JavaBean：40+ 行代码
public class User {
    private String username;
    private String password;
    private int age;
    
    public User() {}
    public User(String username, String password, int age) { ... }
    
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    public String getPassword() { return password; }
    public void setPassword(String password) { this.password = password; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
    
    @Override public String toString() { ... }
}

// ✅ Lombok 写法：10 行代码
import lombok.*;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class User {
    private String username;
    private String password;
    private int age;
}
```

### 5. @Builder 使用

```java
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class User {
    private String username;
    private String password;
    @Builder.Default  // 设置默认值
    private int age = 18;
}

// Builder 方式创建对象（链式调用）
User user = User.builder()
    .username("zhangsan")
    .password("123456")
    .age(20)
    .build();
```

### 6. @Accessors 配置

```java
// chain = true：setter 返回 this，支持链式调用
@Data
@Accessors(chain = true)
public class User {
    private String username;
    private int age;
}

// 使用
User user = new User()
    .setUsername("张三")
    .setAge(18);

// fluent = true：getter/setter 去掉 get/set 前缀
@Data
@Accessors(fluent = true)
public class User {
    private String username;
}

// 使用
user.username("张三");   // setter
user.username();         // getter
```

### 8. @Value（不可变类）

```java
@Value  // 生成不可变类
public class Point {
```

### 10. Lombok 注意事项 ⚠️

| 问题             | 说明                        | 解决方案                                      |
| -------------- | ------------------------- | ----------------------------------------- |
| @Data 与继承      | 子类的 equals/hashCode 可能不正确 | 使用 `@EqualsAndHashCode(callSuper = true)` |
| @Builder 与无参构造 | @Builder 默认不生成无参构造        | 同时添加 `@NoArgsConstructor`                 |
| @ToString 循环引用 | 双向关联可能导致 StackOverflow    | 使用 `@ToString(exclude = "field")`         |
| IDE 不识别        | 编译通过但 IDE 报红              | 安装 Lombok 插件                              |

```java
// 继承时的正确写法
@Data
@EqualsAndHashCode(callSuper = true)  // 调用父类的 equals/hashCode
@ToString(callSuper = true)            // 包含父类的 toString
public class Dog extends Animal {
    private String breed;
}

// 排除字段
@Data
@ToString(exclude = {"password"})      // 不包含 password
@EqualsAndHashCode(exclude = {"id"})   // 不比较 id
public class User {
    private Long id;
    private String username;
    private String password;
}
```

## 十、static 关键字

### 1. static 的含义

`static` 表示**静态**，被它修饰的成员**属于类**，而不是属于某个对象。

### 2. 静态变量 vs 实例变量

```java
public class Student {
    String name;              // 实例变量：每个对象独有
    static String school;     // 静态变量：所有对象共享
}
```

| 区别   | 静态变量（类变量）    | 实例变量（对象变量） |
| ---- | ------------ | ---------- |
| 归属   | 属于类          | 属于对象       |
| 内存位置 | 方法区/元空间      | 堆内存        |
| 加载时机 | 类加载时         | 创建对象时      |
| 访问方式 | `类名.变量名`（推荐） | `对象名.变量名`  |
| 共享性  | 所有对象共享一份     | 每个对象独有一份   |

### 3. 静态变量内存图

```java
Student.school = "清华大学";
Student s1 = new Student();
s1.name = "张三";
Student s2 = new Student();
s2.name = "李四";
```

    ┌───────────────────────┐
    │     方法区/元空间       │
    │  ┌─────────────────┐  │
    │  │ Student.class   │  │
    │  │ school = "清华"  │←─┼──── 所有对象共享
    │  └─────────────────┘  │
    └───────────────────────┘

    ┌─────────────────┐        ┌─────────────────────────────────┐
    │     栈内存       │        │           堆内存                 │
    │  ┌───────────┐  │        │  ┌───────────┐  ┌───────────┐  │
    │  │s1 = 0x001 │──┼────────┼─→│name="张三" │  │name="李四" │←─┼── s2
    │  └───────────┘  │        │  └───────────┘  └───────────┘  │
    │  ┌───────────┐  │        │      @0x001         @0x002     │
    │  │s2 = 0x002 │──┼────────┼─────────────────────┘          │
    │  └───────────┘  │        └─────────────────────────────────┘
    └─────────────────┘

### 4. 静态变量的使用

```java
// 推荐：通过类名访问
Student.school = "清华大学";

// 不推荐：通过对象访问
Student s = new Student();
s.school = "清华大学";   // 能用但不推荐
```

### 5. 静态方法

```java
public class MathUtils {
    // 静态方法
    public static int add(int a, int b) {
        return a + b;
    }
    
    public static int max(int a, int b) {
        return a > b ? a : b;
    }
}

// 调用：类名.方法名()
int sum = MathUtils.add(10, 20);
int max = MathUtils.max(5, 8);
```

### 6. 静态方法的访问规则

| 在静态方法中   | 能否访问  | 原因                   |
| -------- | ----- | -------------------- |
| 静态变量     | ✅ 可以  | 都属于类                 |
| 静态方法     | ✅ 可以  | 都属于类                 |
| 实例变量     | ❌ 不可以 | 实例变量需要对象才存在          |
| 实例方法     | ❌ 不可以 | 实例方法需要对象才能调用         |
| this 关键字 | ❌ 不可以 | this 代表对象，静态方法可能没有对象 |

```java
public class Demo {
    int num = 10;              // 实例变量
    static int count = 0;      // 静态变量
    
    // 静态方法
    public static void staticMethod() {
        System.out.println(count);     // ✅ 可以访问静态变量
        // System.out.println(num);    // ❌ 不能访问实例变量
        // System.out.println(this);   // ❌ 不能使用 this
    }
    
    // 实例方法
    public void instanceMethod() {
        System.out.println(num);       // ✅ 可以访问实例变量
        System.out.println(count);     // ✅ 也可以访问静态变量
    }
}
```

### 7. 静态代码块

```java
public class Demo {
    static int count;
    
    // 静态代码块：类加载时执行，只执行一次
    static {
        System.out.println("静态代码块执行");
        count = 100;  // 初始化静态变量
    }
    
    public static void main(String[] args) {
        System.out.println("main 方法执行");
        System.out.println(Demo.count);  // 100
    }
}

// 输出：
// 静态代码块执行
// main 方法执行
// 100
```

**使用场景**：

```java
public class JDBCUtils {
    private static Connection conn;
    
    // 静态代码块：加载数据库驱动（只需执行一次）
    static {
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
            conn = DriverManager.getConnection(url, user, password);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### 8. 静态导入

```java
// 普通导入
import java.lang.Math;
double result = Math.sqrt(16);
double pi = Math.PI;

// 静态导入：直接导入静态成员
import static java.lang.Math.sqrt;
import static java.lang.Math.PI;
import static java.lang.Math.*;  // 导入所有静态成员

double result = sqrt(16);  // 直接使用，无需类名
double pi = PI;
```

### 9. 工具类的设计

```java
public class ArrayUtils {
    // 1. 私有构造方法，防止创建对象
    private ArrayUtils() {
    }
    
    // 2. 所有方法都是静态的
    public static int getMax(int[] arr) {
        int max = arr[0];
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] > max) max = arr[i];
        }
        return max;
    }
    
    public static int getMin(int[] arr) {
        int min = arr[0];
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] < min) min = arr[i];
        }
        return min;
    }
    
    // 3. 可以定义静态常量
    public static final int DEFAULT_CAPACITY = 10;
}

// 使用
int max = ArrayUtils.getMax(arr);
int capacity = ArrayUtils.DEFAULT_CAPACITY;
```

## 十一、权限修饰符

### 1. 四种权限修饰符

| 修饰符         | 同一个类 | 同一个包 | 不同包子类 | 不同包无关类 |
| ----------- | :--: | :--: | :---: | :----: |
| `private`   |   ✅  |   ❌  |   ❌   |    ❌   |
| `(默认)`      |   ✅  |   ✅  |   ❌   |    ❌   |
| `protected` |   ✅  |   ✅  |   ✅   |    ❌   |
| `public`    |   ✅  |   ✅  |   ✅   |    ✅   |

**记忆口诀**：权限从小到大：`private < 默认 < protected < public`

### 2. 权限范围图解

    ┌─────────────────────────────────────────────────────────────┐
    │                        public                                │
    │  ┌───────────────────────────────────────────────────────┐  │
    │  │                    protected                           │  │
    │  │  ┌─────────────────────────────────────────────────┐  │  │
    │  │  │                  默认（包访问）                   │  │  │
    │  │  │  ┌───────────────────────────────────────────┐  │  │  │
    │  │  │  │               private                      │  │  │  │
    │  │  │  │           （仅本类可见）                    │  │  │  │
    │  │  │  └───────────────────────────────────────────┘  │  │  │
    │  │  │              （同包可见）                        │  │  │
    │  │  └─────────────────────────────────────────────────┘  │  │
    │  │            （同包 + 不同包子类可见）                   │  │
    │  └───────────────────────────────────────────────────────┘  │
    │                    （任何地方都可见）                        │
    └─────────────────────────────────────────────────────────────┘

### 3. 使用建议

| 场景       | 推荐权限                          |
| -------- | ----------------------------- |
| 成员变量     | **private**（配合 getter/setter） |
| 成员方法     | 根据需求，通常是 **public**           |
| 构造方法     | 通常是 **public**                |
| 工具类构造方法  | **private**（防止创建对象）           |
| 只给子类用的方法 | **protected**                 |
| 单例模式构造方法 | **private**                   |

### 4. 修饰符的适用范围

| 修饰符         |  类  | 成员变量 | 成员方法 | 构造方法 | 局部变量 |
| ----------- | :-: | :--: | :--: | :--: | :--: |
| `public`    |  ✅  |   ✅  |   ✅  |   ✅  |   ❌  |
| `protected` |  ❌  |   ✅  |   ✅  |   ✅  |   ❌  |
| `(默认)`      |  ✅  |   ✅  |   ✅  |   ✅  |   ❌  |
| `private`   |  ❌  |   ✅  |   ✅  |   ✅  |   ❌  |

**注意**：外部类只能用 `public` 或默认修饰，不能用 `private` 或 `protected`

## 十二、总结脑图

    面向对象编程
        │
        ├── 类和对象
        │     ├── 类 = 模板（成员变量 + 成员方法）
        │     ├── 对象 = new 出来的实例
        │     ├── 内存：栈存引用，堆存对象
        │     ├── null 引用 → NullPointerException
        │     └── == 比较地址，equals 比较内容
        │
        ├── 变量区别
        │     ├── 成员变量：类中方法外，有默认值，堆内存
        │     └── 局部变量：方法中，无默认值，栈内存
        │
        ├── this 关键字
        │     ├── 代表当前对象，解决同名变量问题
        │     ├── this() 调用本类构造（必须第一行）
        │     └── return this 实现链式调用
        │
        ├── 构造方法
        │     ├── 方法名 = 类名，无返回值类型
        │     ├── 创建对象时自动调用
        │     ├── 构造方法链：this() 互相调用
        │     └── 建议：无参 + 全参都写
        │
        ├── 封装
        │     ├── private 修饰成员变量
        │     ├── public 的 getter/setter
        │     └── boolean 类型用 isXxx()
        │
        ├── JavaBean
        │     ├── 标准实体类 = private变量 + 构造方法 + getter/setter
        │     └── 可选：toString、equals、Serializable
        │
        ├── Lombok
        │     ├── @Data = getter/setter/toString/equals/hashCode
        │     ├── @NoArgsConstructor = 无参构造
        │     ├── @AllArgsConstructor = 全参构造
        │     ├── @RequiredArgsConstructor = final字段构造（Spring推荐）
        │     ├── @Builder = 建造者模式
        │     ├── @Value = 不可变类
        │     ├── @Accessors(chain=true) = 链式调用
        │     └── 注意：继承时用 callSuper=true
        │
        ├── static 关键字
        │     ├── 静态成员属于类，所有对象共享
        │     ├── 通过类名访问
        │     ├── 静态方法不能访问非静态成员
        │     ├── 静态代码块：类加载时执行一次
        │     └── 静态导入：import static
        │
        └── 权限修饰符
              ├── private：本类
              ├── 默认：本类 + 同包
              ├── protected：本类 + 同包 + 子类
              └── public：任意位置
