## 一、枚举（Enum）

### 1. 什么是枚举

枚举是一种特殊的类，用于表示**一组固定的常量**。

```java
// 没有枚举时：用常量表示
public class Season {
    public static final int SPRING = 1;
    public static final int SUMMER = 2;
    public static final int AUTUMN = 3;
    public static final int WINTER = 4;
}
// 问题：类型不安全，可以传入任意 int 值

// 使用枚举：类型安全
public enum Season {
    SPRING, SUMMER, AUTUMN, WINTER
}
```

***

### 2. 枚举的定义

#### 基本定义

```java
public enum Season {
    SPRING, SUMMER, AUTUMN, WINTER
}

// 使用
Season s = Season.SPRING;
System.out.println(s);  // SPRING
```

#### 带属性和方法的枚举

```java
public enum Season {
    // 枚举常量（必须在最前面）
    SPRING("春天", "温暖"),
    SUMMER("夏天", "炎热"),
    AUTUMN("秋天", "凉爽"),
    WINTER("冬天", "寒冷");
    
    // 属性
    private String name;
    private String desc;
    
    // 构造方法（默认 private，不能 public）
    Season(String name, String desc) {
        this.name = name;
        this.desc = desc;
    }
    
    // getter 方法
    public String getName() {
        return name;
    }
    
    public String getDesc() {
        return desc;
    }
}

// 使用
Season s = Season.SPRING;
System.out.println(s.getName());  // 春天
System.out.println(s.getDesc());  // 温暖
```

***

### 3. 枚举的常用方法

| 方法                     | 说明                      |
| ---------------------- | ----------------------- |
| `values()`             | 返回所有枚举常量的数组             |
| `valueOf(String name)` | 根据名称获取枚举常量              |
| `name()`               | 获取枚举常量的名称               |
| `ordinal()`            | 获取枚举常量的序号（从 0 开始）       |
| `compareTo(E o)`       | 比较两个枚举常量的顺序（基于 ordinal） |

```java
public class Demo {
    public static void main(String[] args) {
        // 获取所有枚举常量
        Season[] seasons = Season.values();
        for (Season s : seasons) {
            System.out.println(s.name() + " - " + s.ordinal());
        }
        // SPRING - 0
        // SUMMER - 1
        // AUTUMN - 2
        // WINTER - 3
        
        // 根据名称获取枚举
        Season spring = Season.valueOf("SPRING");
        System.out.println(spring);  // SPRING
        
        // 比较枚举顺序
        Season s1 = Season.SPRING;
        Season s2 = Season.WINTER;
        System.out.println(s1.compareTo(s2));  // -3（SPRING 在 WINTER 前面）
        System.out.println(s2.compareTo(s1));  // 3（WINTER 在 SPRING 后面）
    }
}
```

### 5. 枚举实现接口

#### 方式一：每个枚举常量单独实现

```java
public interface Describable {
    String getDescription();
}

public enum Season implements Describable {
    SPRING {
        @Override
        public String getDescription() {
            return "万物复苏";
        }
    },
    SUMMER {
        @Override
        public String getDescription() {
            return "烈日当空";
        }
    },
    AUTUMN {
        @Override
        public String getDescription() {
            return "硕果累累";
        }
    },
    WINTER {
        @Override
        public String getDescription() {
            return "银装素裹";
        }
    }
}
```

#### 方式二：枚举类统一实现

```java
public enum Season implements Describable {
    SPRING("万物复苏"),
    SUMMER("烈日当空"),
    AUTUMN("硕果累累"),
    WINTER("银装素裹");
    
    private String description;
    
    Season(String description) {
        this.description = description;
    }
    
    @Override
    public String getDescription() {
        return description;
    }
}
```

> 💡 **两种方式的选择**：
>
> *   方式一适合每个枚举常量的实现逻辑差异较大的场景
> *   方式二适合实现逻辑相似，只是数据不同的场景（更简洁）

### 6. 枚举的本质

```java
// 枚举本质上是一个类，继承自 java.lang.Enum
public enum Season {
    SPRING, SUMMER, AUTUMN, WINTER
}

// 编译后相当于
public final class Season extends Enum<Season> {
    public static final Season SPRING = new Season("SPRING", 0);
    public static final Season SUMMER = new Season("SUMMER", 1);
    public static final Season AUTUMN = new Season("AUTUMN", 2);
    public static final Season WINTER = new Season("WINTER", 3);
    
    private Season(String name, int ordinal) {
        super(name, ordinal);
    }
}
```

### 7. 枚举进阶用法

#### EnumSet - 高效的枚举集合

```java
import java.util.EnumSet;

public class Demo {
    public static void main(String[] args) {
        // 创建包含指定元素的 EnumSet
        EnumSet<Season> warmSeasons = EnumSet.of(Season.SPRING, Season.SUMMER);
        
        // 创建包含所有元素的 EnumSet
        EnumSet<Season> allSeasons = EnumSet.allOf(Season.class);
        
        // 创建空的 EnumSet
        EnumSet<Season> noSeasons = EnumSet.noneOf(Season.class);
        
        // 创建补集
        EnumSet<Season> coldSeasons = EnumSet.complementOf(warmSeasons);
        // coldSeasons 包含 AUTUMN, WINTER
        
        // 创建范围（基于 ordinal）
        EnumSet<Season> range = EnumSet.range(Season.SPRING, Season.AUTUMN);
        // range 包含 SPRING, SUMMER, AUTUMN
        
        // 遍历
        for (Season s : warmSeasons) {
            System.out.println(s);
        }
    }
}
```

#### EnumMap - 以枚举为键的高效 Map

```java
import java.util.EnumMap;

public class Demo {
    public static void main(String[] args) {
        EnumMap<Season, String> activities = new EnumMap<>(Season.class);
        
        activities.put(Season.SPRING, "踏青");
        activities.put(Season.SUMMER, "游泳");
        activities.put(Season.AUTUMN, "登高");
        activities.put(Season.WINTER, "滑雪");
        
        // 获取
        System.out.println(activities.get(Season.SUMMER));  // 游泳
        
        // 遍历
        activities.forEach((season, activity) -> {
            System.out.println(season + ": " + activity);
        });
    }
}
```

#### 枚举实现单例模式（最佳实践）

```java
// 枚举单例 - 线程安全、防止反射攻击、防止序列化破坏
public enum Singleton {
    INSTANCE;
    
    // 可以有属性
    private int count = 0;
    
    // 可以有方法
    public void doSomething() {
        count++;
        System.out.println("执行操作，当前计数: " + count);
    }
    
    public int getCount() {
        return count;
    }
}

// 使用
Singleton.INSTANCE.doSomething();
```

> 💡 **为什么枚举单例是最佳实践？**
>
> 1.  **线程安全**：枚举实例的创建由 JVM 保证线程安全
> 2.  **防止反射攻击**：反射无法创建枚举实例
> 3.  **防止序列化破坏**：枚举的序列化机制保证不会创建新实例

## 二、类加载器（ClassLoader）

### 1. 类加载过程

    ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
    │    加载     │────→│    链接     │────→│    初始化   │
    │  (Loading)  │     │  (Linking)  │     │(Initialization)│
    └─────────────┘     └──────┬──────┘     └─────────────┘
                               │
                  ┌────────────┼────────────┐
                  │            │            │
            ┌─────┴─────┐ ┌────┴────┐ ┌─────┴─────┐
            │   验证    │ │  准备   │ │   解析    │
            │(Verification)│ │(Preparation)│ │(Resolution)│
            └───────────┘ └─────────┘ └───────────┘

| 阶段      | 说明                       |
| ------- | ------------------------ |
| **加载**  | 读取 .class 文件，生成 Class 对象 |
| **验证**  | 验证字节码是否符合规范              |
| **准备**  | 为静态变量分配内存，赋默认值           |
| **解析**  | 将符号引用转为直接引用              |
| **初始化** | 执行静态代码块，为静态变量赋值          |

***

### 2. 三种类加载器

#### JDK 8 及之前

    ┌─────────────────────────────────────────────────────┐
    │           Bootstrap ClassLoader（启动类加载器）      │
    │           加载 rt.jar、resources.jar（jre/lib 目录） │
    │           C++ 实现，Java 中获取为 null               │
    └───────────────────────┬─────────────────────────────┘
                            │ 父加载器
    ┌───────────────────────┴─────────────────────────────┐
    │           Extension ClassLoader（扩展类加载器）       │
    │           加载 jre/lib/ext 目录下的类                │
    └───────────────────────┬─────────────────────────────┘
                            │ 父加载器
    ┌───────────────────────┴─────────────────────────────┐
    │           Application ClassLoader（应用类加载器）     │
    │           加载 classpath 下的类（我们写的类）         │
    └─────────────────────────────────────────────────────┘

#### JDK 9 及之后（模块化系统）

    ┌─────────────────────────────────────────────────────┐
    │           Bootstrap ClassLoader（启动类加载器）      │
    │           加载核心模块（java.base 等）               │
    │           C++ 实现，Java 中获取为 null               │
    └───────────────────────┬─────────────────────────────┘
                            │ 父加载器
    ┌───────────────────────┴─────────────────────────────┐
    │           Platform ClassLoader（平台类加载器）        │
    │           加载平台模块（java.sql、java.xml 等）       │
    │           ⚠️ 取代了 Extension ClassLoader            │
    └───────────────────────┬─────────────────────────────┘
                            │ 父加载器
    ┌───────────────────────┴─────────────────────────────┐
    │           Application ClassLoader（应用类加载器）     │
    │           加载 classpath 和应用模块                  │
    └─────────────────────────────────────────────────────┘

***

### 3. JDK 8 vs JDK 9+ 类加载器对比

| JDK 8 及之前                 | JDK 9 及之后                | 变化说明                 |
| ------------------------- | ------------------------ | -------------------- |
| Bootstrap ClassLoader     | Bootstrap ClassLoader    | 加载内容变化：rt.jar → 核心模块 |
| **Extension ClassLoader** | **Platform ClassLoader** | **名称和职责都变了**         |
| Application ClassLoader   | Application ClassLoader  | 基本不变                 |

#### 主要变化

| 变化点                      | 说明                                |
| ------------------------ | --------------------------------- |
| **rt.jar 被移除**           | 核心类库拆分成多个模块（java.base、java.sql 等） |
| **jre/lib/ext 目录移除**     | 扩展机制被废弃                           |
| **Extension → Platform** | 不再是"扩展"概念，而是"平台"概念                |
| **模块化**                  | 引入 module-info.java，模块之间有明确的依赖关系  |

### 5. 双亲委派机制

                            开始
                              |
                              ↓
        ┌─────────────────────────────────────────────┐
        │             AppClassLoader                   │
        │    是否加载过? → 是 → 已加载,不再加载         │
        │                 ↓ 否                         │
        │            委托给父加载器                     │
        └─────────────────────────────────────────────┘
                              |
                              ↓
        ┌─────────────────────────────────────────────┐
        │       Platform/Extension ClassLoader         │
        │    是否加载过? → 是 → 已加载,不再加载         │
        │                 ↓ 否                         │
        │            委托给父加载器                     │
        └─────────────────────────────────────────────┘
                              |
                              ↓
        ┌─────────────────────────────────────────────┐
        │          Bootstrap ClassLoader               │
        │    是否加载过? → 是 → 已加载,不再加载         │
        │                 ↓ 否                         │
        │    能加载?   → 是 → 返回(自己加载此类)        │
        │                 ↓ 否                         │
        └─────────────────────────────────────────────┘
                              |
                              ↓
        ┌─────────────────────────────────────────────┐
        │       Platform/Extension ClassLoader         │
        │    能加载?   → 是 → 返回(自己加载此类)        │
        │                 ↓ 否                         │
        └─────────────────────────────────────────────┘
                              |
                              ↓
        ┌─────────────────────────────────────────────┐
        │             AppClassLoader                   │
        │    能加载?   → 是 → 返回(自己加载此类)        │
        │                 ↓ 否                         │
        │         ClassNotFoundException               │
        └─────────────────────────────────────────────┘

**双亲委派机制**（JDK 8 和 JDK 9+ 都适用）：

1.  向上委托阶段：App → Platform → Bootstrap (检查是否已加载)
2.  向下加载阶段：Bootstrap → Platform → App (尝试加载)
3.  都无法加载：抛出 ClassNotFoundException

**好处**：

*   避免类的重复加载
*   保证核心类的安全（防止自定义 java.lang.String）

### 6. 打破双亲委派机制

某些场景需要打破双亲委派机制：

| 场景          | 说明                                   |
| ----------- | ------------------------------------ |
| **SPI 机制**  | JDBC、JNDI 等，核心类需要加载第三方实现（父加载器调用子加载器） |
| **Tomcat**  | 每个 Web 应用独立的类加载器，实现应用隔离              |
| **OSGi**    | 模块化热部署，每个模块独立的类加载器                   |
| **热部署/热更新** | 自定义类加载器实现代码热替换                       |

#### SPI 机制示例（以 JDBC 为例）

```java
// DriverManager 在 rt.jar 中，由 Bootstrap ClassLoader 加载
// 但具体的数据库驱动（如 MySQL Driver）在 classpath 中，由 App ClassLoader 加载
// 这就打破了双亲委派：父加载器加载的类需要调用子加载器加载的类

// 解决方案：线程上下文类加载器（Thread Context ClassLoader）
ClassLoader cl = Thread.currentThread().getContextClassLoader();
// 默认是 Application ClassLoader
```

#### 自定义类加载器

```java
public class MyClassLoader extends ClassLoader {
    
    private String classPath;
    
    public MyClassLoader(String classPath) {
        this.classPath = classPath;
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            // 读取 class 文件
            byte[] data = loadClassData(name);
            // 将字节数组转换为 Class 对象
            return defineClass(name, data, 0, data.length);
        } catch (Exception e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] loadClassData(String name) throws Exception {
        String fileName = classPath + "/" + name.replace(".", "/") + ".class";
        FileInputStream fis = new FileInputStream(fileName);
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        int len;
        byte[] buffer = new byte[1024];
        while ((len = fis.read(buffer)) != -1) {
            baos.write(buffer, 0, len);
        }
        fis.close();
        return baos.toByteArray();
    }
}

// 使用
MyClassLoader loader = new MyClassLoader("/path/to/classes");
Class<?> clazz = loader.loadClass("com.example.MyClass");
Object obj = clazz.getDeclaredConstructor().newInstance();
```

## 三、反射（Reflection）

### 1. 什么是反射

反射是在**运行时**动态获取类的信息、创建对象、调用方法的机制。

    编译时                            运行时
    ┌──────────────┐                ┌──────────────┐
    │  .java 文件   │ → 编译 →      │  .class 文件  │
    │  源代码      │                │  字节码       │
    └──────────────┘                └───────┬──────┘
                                            │
                                            ↓ 反射
                                    ┌──────────────┐
                                    │  Class 对象   │
                                    │  获取类信息    │
                                    │  创建对象     │
                                    │  调用方法     │
                                    └──────────────┘

***

### 2. 获取 Class 对象的三种方式

```java
// 方式1：类名.class
Class<Student> clazz1 = Student.class;

// 方式2：对象.getClass()
Student stu = new Student();
Class<? extends Student> clazz2 = stu.getClass();

// 方式3：Class.forName("全类名")  ← 最常用
Class<?> clazz3 = Class.forName("com.example.Student");

// 三种方式获取的是同一个 Class 对象
System.out.println(clazz1 == clazz2);  // true
System.out.println(clazz2 == clazz3);  // true
```

***

### 3. 反射获取构造方法

```java
public class Student {
    private String name;
    private int age;
    
    public Student() {}
    
    public Student(String name) {
        this.name = name;
    }
    
    private Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

```java
Class<?> clazz = Class.forName("com.example.Student");

// 获取所有 public 构造方法
Constructor<?>[] constructors = clazz.getConstructors();

// 获取所有构造方法（包括 private）
Constructor<?>[] allConstructors = clazz.getDeclaredConstructors();

// 获取指定构造方法
Constructor<?> con1 = clazz.getConstructor();  // 无参
Constructor<?> con2 = clazz.getConstructor(String.class);  // 一个参数
Constructor<?> con3 = clazz.getDeclaredConstructor(String.class, int.class);  // private

// 使用构造方法创建对象
Object obj1 = con1.newInstance();
Object obj2 = con2.newInstance("张三");

// 访问 private 构造方法
con3.setAccessible(true);  // 暴力反射
Object obj3 = con3.newInstance("李四", 20);
```

***

### 4. 反射获取成员变量

```java
Class<?> clazz = Class.forName("com.example.Student");

// 获取所有 public 成员变量
Field[] fields = clazz.getFields();

// 获取所有成员变量（包括 private）
Field[] allFields = clazz.getDeclaredFields();

// 获取指定成员变量
Field nameField = clazz.getDeclaredField("name");

// 设置/获取成员变量的值
Object obj = clazz.getDeclaredConstructor().newInstance();
nameField.setAccessible(true);  // 暴力反射
nameField.set(obj, "张三");     // 设置值
Object value = nameField.get(obj);  // 获取值
System.out.println(value);  // 张三
```

***

### 5. 反射获取成员方法

```java
public class Student {
    public void study() {
        System.out.println("学习中...");
    }
    
    public String sayHello(String name) {
        return "Hello, " + name;
    }
    
    private void secret() {
        System.out.println("私有方法");
    }
}
```

```java
Class<?> clazz = Class.forName("com.example.Student");

// 获取所有 public 方法（包括父类）
Method[] methods = clazz.getMethods();

// 获取本类所有方法（包括 private）
Method[] allMethods = clazz.getDeclaredMethods();

// 获取指定方法
Method studyMethod = clazz.getMethod("study");
Method sayHelloMethod = clazz.getMethod("sayHello", String.class);
Method secretMethod = clazz.getDeclaredMethod("secret");

// 调用方法
Object obj = clazz.getDeclaredConstructor().newInstance();
studyMethod.invoke(obj);  // 学习中...
Object result = sayHelloMethod.invoke(obj, "World");  // Hello, World

// 调用 private 方法
secretMethod.setAccessible(true);
secretMethod.invoke(obj);  // 私有方法
```

### 7. 反射性能优化

反射相比直接调用有性能开销，以下是优化建议：

| 优化方式                | 说明                              |
| ------------------- | ------------------------------- |
| **缓存反射对象**          | 缓存 Class、Method、Field 对象，避免重复获取 |
| **使用 MethodHandle** | Java 7+ 提供，性能接近直接调用             |
| **关闭安全检查**          | setAccessible(true) 可以提升性能      |
| **考虑代码生成**          | 对于高频调用，考虑使用字节码生成库               |

#### 缓存反射对象

```java
// ❌ 不推荐：每次都获取 Method
public void badExample(Object obj) throws Exception {
    Method method = obj.getClass().getMethod("doSomething");
    method.invoke(obj);
}

// ✅ 推荐：缓存 Method 对象
private static final Map<Class<?>, Method> METHOD_CACHE = new ConcurrentHashMap<>();

public void goodExample(Object obj) throws Exception {
    Method method = METHOD_CACHE.computeIfAbsent(obj.getClass(), clazz -> {
        try {
            Method m = clazz.getMethod("doSomething");
            m.setAccessible(true);  // 关闭安全检查，提升性能
            return m;
        } catch (NoSuchMethodException e) {
            throw new RuntimeException(e);
        }
    });
    method.invoke(obj);
}
```

#### MethodHandle 示例（Java 7+）

```java
import java.lang.invoke.MethodHandle;
import java.lang.invoke.MethodHandles;
import java.lang.invoke.MethodType;

public class MethodHandleDemo {
    public static void main(String[] args) throws Throwable {
        // 获取 MethodHandles.Lookup
        MethodHandles.Lookup lookup = MethodHandles.lookup();
        
        // 获取方法句柄：String.substring(int, int)
        MethodHandle substringHandle = lookup.findVirtual(
            String.class,                              // 目标类
            "substring",                               // 方法名
            MethodType.methodType(String.class, int.class, int.class)  // 返回类型, 参数类型
        );
        
        // 调用方法
        String result = (String) substringHandle.invoke("Hello World", 0, 5);
        System.out.println(result);  // Hello
        
        // 静态方法示例：Integer.parseInt(String)
        MethodHandle parseIntHandle = lookup.findStatic(
            Integer.class,
            "parseInt",
            MethodType.methodType(int.class, String.class)
        );
        
        int num = (int) parseIntHandle.invoke("123");
        System.out.println(num);  // 123
    }
}
```

> 💡 **MethodHandle vs 反射**：
>
> *   MethodHandle 在首次查找后，调用性能接近直接调用
> *   反射每次调用都需要进行安全检查
> *   MethodHandle 是 JVM 层面的优化，支持方法内联

### 8. 模块化对反射的影响（Java 9+）

Java 9 引入模块化系统后，对反射有以下影响：

| 影响              | 说明                                  |
| --------------- | ----------------------------------- |
| **默认无法反射非公开成员** | 模块内的非 public 类型默认无法被反射访问            |
| **需要显式开放**      | 在 module-info.java 中使用 opens 关键字开放包 |
| **非法反射警告**      | 非法反射访问会产生警告（未来版本可能报错）               |

```java
// module-info.java
module my.module {
    // 开放包给所有模块（允许反射）
    opens com.example.entity;
    
    // 开放包给指定模块
    opens com.example.internal to spring.core;
    
    // 导出包（只允许正常访问，不允许反射私有成员）
    exports com.example.api;
}
```

## 四、注解（Annotation）

### 1. 什么是注解

注解是一种**元数据**，用于为代码添加额外信息，可以被编译器或运行时读取。

```java
@Override           // 编译时检查是否正确重写
public String toString() {
    return "Student";
}

@Deprecated         // 标记已过时
public void oldMethod() {}

@SuppressWarnings("unchecked")  // 抑制警告
public void test() {}
```

***

### 2. 自定义注解

```java
// 定义注解
public @interface MyAnnotation {
    // 注解属性（本质是抽象方法）
    String value() default "";
    int age() default 0;
    String[] tags() default {};
}

// 使用注解
@MyAnnotation(value = "张三", age = 18, tags = {"学生", "程序员"})
public class Student {
}

// 只有 value 属性时可以省略 value=
@MyAnnotation("张三")
public class Teacher {
}
```

***

### 3. 元注解

元注解是**用于修饰注解的注解**。

| 元注解           | 说明                        |
| ------------- | ------------------------- |
| `@Target`     | 指定注解可以用在哪里（类、方法、字段等）      |
| `@Retention`  | 指定注解的保留策略（源码、class文件、运行时） |
| `@Documented` | 注解包含在 Javadoc 中           |
| `@Inherited`  | 注解可以被子类继承                 |
| `@Repeatable` | 注解可以重复使用（Java 8+）         |

#### @Target - 注解的作用位置

```java
@Target({ElementType.TYPE, ElementType.METHOD, ElementType.FIELD})
public @interface MyAnnotation {
}
```

| ElementType      | 说明               |
| ---------------- | ---------------- |
| `TYPE`           | 类、接口、枚举          |
| `FIELD`          | 字段               |
| `METHOD`         | 方法               |
| `PARAMETER`      | 方法参数             |
| `CONSTRUCTOR`    | 构造方法             |
| `LOCAL_VARIABLE` | 局部变量             |
| `TYPE_PARAMETER` | 类型参数（Java 8+）    |
| `TYPE_USE`       | 任何类型使用处（Java 8+） |

#### @Retention - 注解的保留策略

```java
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
}
```

| RetentionPolicy | 说明                     |
| --------------- | ---------------------- |
| `SOURCE`        | 只在源码中保留，编译后丢弃          |
| `CLASS`         | 保留到 class 文件，运行时丢弃（默认） |
| `RUNTIME`       | 运行时保留，可以通过反射获取         |

***

### 4. 完整的自定义注解示例

```java
import java.lang.annotation.*;

@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface MyTest {
    String value() default "";
    int priority() default 0;
}
```

### 5. 通过反射获取注解信息

```java
@MyTest(value = "学生类", priority = 1)
public class Student {
    
    @MyTest(value = "学习方法", priority = 2)
    public void study() {
        System.out.println("学习中...");

}
```

```java
public class Demo {
    public static void main(String[] args) throws Exception {
        Class<?> clazz = Student.class;
        
        // 判断类上是否有注解
        if (clazz.isAnnotationPresent(MyTest.class)) {
            // 获取类上的注解
            MyTest annotation = clazz.getAnnotation(MyTest.class);
            System.out.println("类注解 value: " + annotation.value());
            System.out.println("类注解 priority: " + annotation.priority());
        }
        
        // 获取方法上的注解
        Method method = clazz.getMethod("study");
        if (method.isAnnotationPresent(MyTest.class)) {
            MyTest annotation = method.getAnnotation(MyTest.class);
            System.out.println("方法注解 value: " + annotation.value());
            System.out.println("方法注解 priority: " + annotation.priority());
        }
    }
}
```

### 6. 可重复注解（Java 8+）

```java
// 定义可重复注解
@Repeatable(Roles.class)  // 指定容器注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Role {
    String value();
}

// 容器注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Roles {
    Role[] value();
}

// 使用：可以重复标注
@Role("admin")
@Role("user")
@Role("guest")
public class User {
}

// 获取重复注解
public class Demo {
    public static void main(String[] args) {
        Class<?> clazz = User.class;
        
        // 方式1：获取所有重复注解
        Role[] roles = clazz.getAnnotationsByType(Role.class);
        for (Role role : roles) {
            System.out.println("角色: " + role.value());
        }
        
        // 方式2：获取容器注解
        Roles rolesContainer = clazz.getAnnotation(Roles.class);
        if (rolesContainer != null) {
            for (Role role : rolesContainer.value()) {
                System.out.println("角色: " + role.value());
            }
        }
    }
}
```

### 7. TYPE\_USE 注解（Java 8+）

TYPE\_USE 允许注解用于任何类型使用处：

```java

}

@Retention(RetentionPolicy.RUNTIME)

}

    public @Nullable String getName() {
        return name;
    }

    }
    
    // 用于异常声明

    }
    
    // 用于类型转换
    public void cast(Object obj) {
        String str = (@NonNull String) obj;
    }
    
    // 用于 instanceof
    public boolean check(Object obj) {
        return obj instanceof @NonNull String;
    }
}
```

### 8. 注解的实际应用：模拟单元测试

```java

@Retention(RetentionPolicy.RUNTIME)

}

            }
        }
    }
}
```

## 五、动态代理

### 1. 什么是代理

代理是一种设计模式，通过代理对象**控制对原对象的访问**。

    ┌──────────┐         ┌──────────┐         ┌──────────┐
    │  调用者   │ ──────→ │  代理对象 │ ──────→ │  目标对象 │
    └──────────┘         └──────────┘         └──────────┘
                               │
                               ↓
                         增强功能
                       （日志、权限、事务）

***

### 2. 静态代理 vs 动态代理

| 对比      | 静态代理         | 动态代理            |
| ------- | ------------ | --------------- |
| **代理类** | 手动编写         | 运行时自动生成         |
| **灵活性** | 低，每个接口都要写代理类 | 高，一个代理类可以代理多个接口 |
| **应用**  | 简单场景         | 框架（Spring AOP）  |

***

### 3. JDK 动态代理

**要求**：目标对象必须实现接口

#### 步骤

```java
// 1. 定义接口
public interface UserService {
    void add();
    void delete();
    void update();
    void query();
}

// 2. 实现类（目标对象）
public class UserServiceImpl implements UserService {
    @Override
    public void add() {
        System.out.println("添加用户");
    }
    
    @Override
    public void delete() {
        System.out.println("删除用户");
    }
    
    @Override
    public void update() {
        System.out.println("更新用户");
    }
    
    @Override
    public void query() {
        System.out.println("查询用户");
    }
}
```

#### 创建代理对象

```java
import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public class ProxyFactory {
    
    // 目标对象
    private Object target;
    
    public ProxyFactory(Object target) {
        this.target = target;
    }
    
    // 生成代理对象
    public Object getProxyInstance() {
        return Proxy.newProxyInstance(
            target.getClass().getClassLoader(),  // 类加载器
            target.getClass().getInterfaces(),   // 目标对象实现的接口
            new InvocationHandler() {            // 调用处理器
                @Override
                public Object invoke(Object proxy, Method method, Object[] args) 
                        throws Throwable {
                    // 前置增强
                    System.out.println("【日志】开始执行 " + method.getName());
                    long start = System.currentTimeMillis();
                    
                    // 调用目标方法
                    Object result = method.invoke(target, args);
                    
                    // 后置增强
                    long end = System.currentTimeMillis();
                    System.out.println("【日志】执行完毕，耗时 " + (end - start) + "ms");
                    
                    return result;
                }
            }
        );
    }
}
```

#### 使用代理

```java
public class Demo {
    public static void main(String[] args) {
        // 创建目标对象
        UserService target = new UserServiceImpl();
        
        // 创建代理对象
        UserService proxy = (UserService) new ProxyFactory(target).getProxyInstance();
        
        // 通过代理调用方法
        proxy.add();
        // 【日志】开始执行 add
        // 添加用户
        // 【日志】执行完毕，耗时 0ms
        
        proxy.query();
        // 【日志】开始执行 query
        // 查询用户
        // 【日志】执行完毕，耗时 0ms
    }
}
```

### 4. 简化版工具类

```java
public class ProxyUtil {
    
    /**
     * 创建代理对象
     * @param target 目标对象
     * @return 代理对象
     */
    @SuppressWarnings("unchecked")
    public static <T> T createProxy(T target) {
        return (T) Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            (proxy, method, args) -> {
                // 前置处理
                System.out.println("=== 方法执行前 ===");
                
                // 执行目标方法
                Object result = method.invoke(target, args);
                
                // 后置处理
                System.out.println("=== 方法执行后 ===");
                
                return result;
            }
        );
    }
}

proxy.add();
```

### 5. JDK 动态代理 vs CGLIB

| 对比项           | JDK 动态代理      | CGLIB            |
| ------------- | ------------- | ---------------- |
| **原理**        | 基于接口，生成接口的实现类 | 基于继承，生成目标类的子类    |
| **要求**        | 目标类必须实现接口     | 目标类不能是 final     |
| **性能（创建）**    | 创建代理对象快       | 创建代理对象慢（需要生成字节码） |
| **性能（调用）**    | 调用稍慢          | 调用快（直接调用方法）      |
| **依赖**        | JDK 内置，无需额外依赖 | 需要引入 cglib 库     |
| **Spring 选择** | 有接口时默认使用      | 无接口时自动使用         |

#### CGLIB 示例（需要引入依赖）

```xml
<!-- Maven 依赖 -->
<dependency>
    <groupId>cglib</groupId>
    <artifactId>cglib</artifactId>
    <version>3.3.0</version>
</dependency>
```

```java
import net.sf.cglib.proxy.Enhancer;
import net.sf.cglib.proxy.MethodInterceptor;
import net.sf.cglib.proxy.MethodProxy;

// 目标类（不需要实现接口）
public class UserServiceImpl {
    public void add() {
        System.out.println("添加用户");
    }
}

// CGLIB 代理工厂
public class CglibProxyFactory {
    
    private Object target;
    
    public CglibProxyFactory(Object target) {
        this.target = target;
    }
    
    public Object getProxyInstance() {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(target.getClass());  // 设置父类
        enhancer.setCallback(new MethodInterceptor() {
            @Override
            public Object intercept(Object obj, Method method, Object[] args, 
                    MethodProxy proxy) throws Throwable {
                System.out.println("【CGLIB】方法执行前");
                Object result = proxy.invokeSuper(obj, args);  // 调用父类方法
                System.out.println("【CGLIB】方法执行后");
                return result;
            }
        });
        return enhancer.create();
    }
}

// 使用
UserServiceImpl proxy = (UserServiceImpl) new CglibProxyFactory(new UserServiceImpl())
        .getProxyInstance();
proxy.add();
```

### 7. 完整示例：权限校验

```java
// 接口
public interface OrderService {
    void createOrder();
    void deleteOrder();
}

// 实现类
public class OrderServiceImpl implements OrderService {
    @Override
    public void createOrder() {
        System.out.println("创建订单");
    }
    
    @Override
    public void deleteOrder() {
        System.out.println("删除订单");
    }
}

// 代理工厂（带权限校验）
public class AuthProxyFactory {
    
    private Object target;
    private String currentUser;
    
    public AuthProxyFactory(Object target, String currentUser) {
        this.target = target;
        this.currentUser = currentUser;
    }
    
    public Object getProxy() {
        return Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            (proxy, method, args) -> {
                // 权限校验
                if ("deleteOrder".equals(method.getName())) {
                    if (!"admin".equals(currentUser)) {
                        throw new RuntimeException("无权限执行此操作！");
                    }
                }
                
                return method.invoke(target, args);
            }
        );
    }
}

// 测试
public class Demo {
    public static void main(String[] args) {
        OrderService target = new OrderServiceImpl();
        
        // 普通用户
        OrderService proxy1 = (OrderService) new AuthProxyFactory(target, "user").getProxy();
        proxy1.createOrder();   // ✅ 成功
        // proxy1.deleteOrder(); // ❌ 抛异常：无权限执行此操作！
        
        // 管理员
        OrderService proxy2 = (OrderService) new AuthProxyFactory(target, "admin").getProxy();
        proxy2.createOrder();   // ✅ 成功
        proxy2.deleteOrder();   // ✅ 成功
    }
}
```

## 六、总结脑图

    Java 高级特性
        │
        ├── 枚举（Enum）
        │     ├── 定义：一组固定常量
        │     ├── 方法：values()、valueOf()、name()、ordinal()、compareTo()
        │     ├── 可以有属性、构造方法、方法
        │     ├── 本质：继承自 java.lang.Enum 的类
        │     └── 进阶：EnumSet、EnumMap、枚举单例
        │
        ├── 类加载器（ClassLoader）
        │     ├── 加载过程：加载 → 链接 → 初始化
        │     ├── JDK 8：Bootstrap → Extension → Application
        │     ├── JDK 9+：Bootstrap → Platform → Application
        │     │           （Extension 改名为 Platform）
        │     ├── 双亲委派：先委托父加载器，加载失败才自己加载
        │     └── 打破双亲委派：SPI、Tomcat、OSGi、自定义类加载器
        │
        ├── 反射（Reflection）
        │     ├── 获取 Class：类名.class、对象.getClass()、Class.forName()
        │     ├── 获取构造方法：getConstructor()、getDeclaredConstructor()
        │     ├── 获取成员变量：getField()、getDeclaredField()
        │     ├── 获取成员方法：getMethod()、getDeclaredMethod()
        │     ├── 暴力反射：setAccessible(true)
        │     ├── 性能优化：缓存反射对象、MethodHandle
        │     └── 模块化影响：Java 9+ 需要 opens 开放包
        │
        ├── 注解（Annotation）
        │     ├── 定义：@interface
        │     ├── 元注解：@Target、@Retention、@Documented、@Inherited、@Repeatable
        │     ├── 获取注解：isAnnotationPresent()、getAnnotation()
        │     ├── Java 8+：@Repeatable（可重复注解）、TYPE_USE
        │     └── 应用：配置、验证、测试框架
        │
        └── 动态代理
              ├── JDK 动态代理：Proxy.newProxyInstance()
              │     └── 要求：目标对象必须实现接口
              ├── CGLIB 动态代理：基于继承，无需接口
              ├── InvocationHandler：invoke(proxy, method, args)
              └── 应用：日志、权限、事务、Spring AOP
