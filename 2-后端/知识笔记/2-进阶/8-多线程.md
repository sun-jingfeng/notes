## 一、进程和线程

### 1. 基本概念

| 概念              | 说明                       | 类比     |
| --------------- | ------------------------ | ------ |
| **进程（Process）** | 操作系统中正在运行的一个程序           | 一个工厂   |
| **线程（Thread）**  | 进程中的一个执行单元，是 CPU 调度的最小单位 | 工厂里的工人 |

**关系**：

*   一个进程可以包含多个线程
*   一个进程至少有一个线程（主线程）
*   线程共享进程的资源（内存、文件等）

### 2. 图解

    ┌─────────────────────────────────────────────────────────────┐
    │                        操作系统                              │
    │  ┌─────────────────────┐    ┌─────────────────────┐         │
    │  │   进程1（QQ）        │    │   进程2（微信）      │         │
    │  │  ┌─────┐ ┌─────┐   │    │  ┌─────┐ ┌─────┐   │         │
    │  │  │线程1│ │线程2│   │    │  │线程1│ │线程2│   │         │
    │  │  └─────┘ └─────┘   │    │  └─────┘ └─────┘   │         │
    │  │  ┌─────┐           │    │  ┌─────┐           │         │
    │  │  │线程3│           │    │  │线程3│           │         │
    │  │  └─────┘           │    │  └─────┘           │         │
    │  └─────────────────────┘    └─────────────────────┘         │
    └─────────────────────────────────────────────────────────────┘

### 3. 并发与并行

| 概念                 | 说明                 | 图解               |
| ------------------ | ------------------ | ---------------- |
| **并发（Concurrent）** | 多个任务交替执行（CPU 快速切换） | 一个人同时做多件事（看起来同时） |
| **并行（Parallel）**   | 多个任务真正同时执行（多核 CPU） | 多个人同时做多件事（真正同时）  |

    并发（单核 CPU）：
    时间 →
    CPU:  [任务A][任务B][任务A][任务B][任务A]...
          交替执行，看起来像同时

    并行（多核 CPU）：
    时间 →
    CPU1: [任务A][任务A][任务A]...
    CPU2: [任务B][任务B][任务B]...
          真正同时执行

## 二、Java 开启线程的方式

### 方式一：继承 Thread 类

```java
// 1. 定义类继承 Thread
public class MyThread extends Thread {
    
    // 2. 重写 run() 方法
    @Override
    public void run() {
        for (int i = 0; i < 100; i++) {
            System.out.println(getName() + "：" + i);
        }
    }
}

// 3. 创建对象，调用 start() 启动线程
public class Demo {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        
        t1.setName("线程1");
        t2.setName("线程2");
        
        t1.start();  // ✅ 启动线程
        t2.start();
        
        // t1.run();  // ❌ 错误！直接调用 run() 不会开启新线程
    }
}
```

**注意**：

*   调用 `start()` 方法启动线程，不是 `run()`
*   `run()` 中的代码才是线程要执行的任务

***

### 方式二：实现 Runnable 接口（推荐）

```java
// 1. 定义类实现 Runnable 接口
public class MyRunnable implements Runnable {
    
    // 2. 重写 run() 方法
    @Override
    public void run() {
        for (int i = 0; i < 100; i++) {
            // Runnable 没有 getName()，需要通过 Thread.currentThread()
            System.out.println(Thread.currentThread().getName() + "：" + i);
        }
    }
}

// 3. 创建 Runnable 对象，传给 Thread
public class Demo {
    public static void main(String[] args) {
        MyRunnable mr = new MyRunnable();
        
        Thread t1 = new Thread(mr, "线程1");
        Thread t2 = new Thread(mr, "线程2");
        
        t1.start();
        t2.start();
    }
}
```

**Lambda 简化写法**：

```java
// 匿名内部类
new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("线程执行");
    }
}).start();

// Lambda 表达式（推荐）
new Thread(() -> {
    System.out.println("线程执行");
}).start();
```

***

### 方式三：实现 Callable 接口（可以获取返回值）

```java
import java.util.concurrent.Callable;
import java.util.concurrent.FutureTask;

// 1. 定义类实现 Callable 接口，指定返回值类型
public class MyCallable implements Callable<Integer> {
    
    // 2. 重写 call() 方法，可以有返回值
    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = 1; i <= 100; i++) {
            sum += i;
        }
        return sum;
    }
}

// 3. 使用 FutureTask 包装，获取返回值
public class Demo {
    public static void main(String[] args) throws Exception {
        MyCallable mc = new MyCallable();
        FutureTask<Integer> ft = new FutureTask<>(mc);
        
        Thread t = new Thread(ft);
        t.start();
        
        // 获取返回值（会阻塞，等待线程执行完毕）
        Integer result = ft.get();
        System.out.println("结果：" + result);  // 结果：5050
    }
}
```

***

### 4. 三种方式对比

| 方式              | 优点                          | 缺点         |
| --------------- | --------------------------- | ---------- |
| **继承 Thread**   | 编写简单，可直接使用 `this.getName()` | 单继承限制，扩展性差 |
| **实现 Runnable** | 扩展性强，可以继承其他类；适合多线程共享资源      | 无返回值       |
| **实现 Callable** | 可以有返回值，可以抛出异常               | 编写稍复杂      |

**推荐**：优先使用 **Runnable**（无返回值）或 **Callable**（有返回值）

## 三、线程常用方法与生命周期

### 1. 常用方法一览

| 方法                               | 说明                   |
| -------------------------------- | -------------------- |
| `void start()`                   | 启动线程                 |
| `void run()`                     | 线程要执行的任务             |
| `String getName()`               | 获取线程名称               |
| `void setName(String name)`      | 设置线程名称               |
| `static Thread currentThread()`  | 获取当前线程对象             |
| `static void sleep(long millis)` | 让当前线程休眠指定毫秒          |
| `void join()`                    | 等待该线程执行完毕            |
| `void join(long millis)`         | 等待该线程执行，最多等待指定毫秒     |
| `void setDaemon(boolean on)`     | 设置为守护线程              |
| `int getPriority()`              | 获取线程优先级              |
| `void setPriority(int priority)` | 设置线程优先级（1-10）        |
| `static void yield()`            | 让出 CPU 执行权           |
| `void interrupt()`               | 中断线程                 |
| `boolean isInterrupted()`        | 判断线程是否被中断            |
| `static boolean interrupted()`   | 判断当前线程是否被中断（会清除中断标志） |

***

### 2. 方法详解

#### sleep() - 线程休眠

```java
public class Demo {
    public static void main(String[] args) throws InterruptedException {
        System.out.println("开始");
        
        Thread.sleep(3000);  // 休眠 3 秒
        
        System.out.println("结束");
    }
}
```

***

#### join() - 等待线程结束

```java
public class Demo {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("子线程：" + i);
            }
        });
        
        t.start();
        t.join();  // 主线程等待 t 执行完毕
        
        System.out.println("主线程继续执行");
    }
}

// 输出：
// 子线程：0
// 子线程：1
// 子线程：2
// 子线程：3
// 子线程：4
// 主线程继续执行
```

***

#### setDaemon() - 守护线程

```java
public class Demo {
    public static void main(String[] args) {
        Thread t = new Thread(() -> {
            while (true) {
                System.out.println("守护线程运行中...");
            }
        });
        
        t.setDaemon(true);  // 设置为守护线程（必须在 start 之前）
        t.start();
        
        // 主线程结束后，守护线程也会结束
        System.out.println("主线程结束");
    }
}
```

**守护线程特点**：

*   当所有非守护线程结束时，守护线程自动结束
*   典型应用：垃圾回收线程

***

#### setPriority() - 线程优先级

```java
public class Demo {
    public static void main(String[] args) {
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 100; i++) {
                System.out.println("t1：" + i);
            }
        });
        
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 100; i++) {
                System.out.println("t2：" + i);
            }
        });
        
        // 优先级范围：1-10，默认是 5
        t1.setPriority(Thread.MIN_PRIORITY);  // 1
        t2.setPriority(Thread.MAX_PRIORITY);  // 10
        
        t1.start();
        t2.start();
    }
}
```

**注意**：优先级高只是表示**抢到 CPU 的概率更大**，不保证一定先执行

***

#### interrupt() - 线程中断

```java
public class Demo {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                System.out.println("线程运行中...");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    // sleep 被中断会抛出异常，并清除中断标志
                    System.out.println("线程被中断，退出循环");
                    break;  // 或者重新设置中断标志：Thread.currentThread().interrupt();
                }
            }
            System.out.println("线程结束");
        });
        
        t.start();
        
        Thread.sleep(3000);  // 主线程等待 3 秒
        t.interrupt();       // 中断线程
        System.out.println("发送中断信号");
    }
}
```

**中断机制说明**：

| 方法                     | 说明                   |
| ---------------------- | -------------------- |
| `interrupt()`          | 设置中断标志（不会强制停止线程）     |
| `isInterrupted()`      | 判断是否被中断（不清除中断标志）     |
| `Thread.interrupted()` | 判断当前线程是否被中断（会清除中断标志） |

> 💡 **注意**：`interrupt()` 只是设置中断标志，线程需要自己检查并响应中断。如果线程正在 `sleep()`、`wait()`、`join()` 等阻塞状态，会抛出 `InterruptedException` 并清除中断标志。

### 3. 线程生命周期（6 种状态）

Java 线程有 **6 种状态**，定义在 `Thread.State` 枚举中：

                                         ┌──────────────────┐
                                         │     BLOCKED      │
                                         │   （阻塞-等待锁）  │
                                         └────────┬─────────┘
                                                  ↑ 等待锁
                                                  │
    ┌─────────┐     ┌───────────────┐     ┌───────┴───────┐     ┌─────────────┐
    │   NEW   │────→│   RUNNABLE    │←───→│   获取到锁    │────→│ TERMINATED  │
    │  新建   │     │  就绪/运行中   │     │               │     │    终止     │
    └─────────┘     └───────┬───────┘     └───────────────┘     └─────────────┘
      new Thread()          │ wait()/join()/sleep()
           ↓ start()        ↓
                      ┌─────────────────┐
                      │    WAITING      │  无限等待
                      │  （等待-无超时）  │  wait()、join()
                      └─────────────────┘
                      ┌─────────────────┐
                      │  TIMED_WAITING  │  有限等待
                      │  （等待-有超时）  │  sleep(n)、wait(n)、join(n)
                      └─────────────────┘

| 状态                 | 说明           | 触发条件                                   |
| ------------------ | ------------ | -------------------------------------- |
| **NEW**            | 新建状态         | `new Thread()` 后，未调用 `start()`         |
| **RUNNABLE**       | 可运行状态（就绪+运行） | 调用 `start()` 后                         |
| **BLOCKED**        | 阻塞状态         | 等待获取 synchronized 锁                    |
| **WAITING**        | 无限等待状态       | `wait()`、`join()`、`LockSupport.park()` |
| **TIMED\_WAITING** | 有限等待状态       | `sleep(n)`、`wait(n)`、`join(n)`         |
| **TERMINATED**     | 终止状态         | 线程执行完毕或异常终止                            |

```java
// 获取线程状态
Thread t = new Thread(() -> {});
System.out.println(t.getState());  // NEW

t.start();
System.out.println(t.getState());  // RUNNABLE

// ... 线程执行完毕
System.out.println(t.getState());  // TERMINATED
```

## 四、线程安全和同步

### 1. 线程安全问题

当多个线程**同时访问共享资源**时，可能出现数据不一致的问题。

#### 问题演示：卖票

```java
public class Ticket implements Runnable {
    private int ticket = 100;  // 共享资源
    
    @Override
    public void run() {
        while (true) {
            if (ticket > 0) {
                try {
                    Thread.sleep(10);  // 模拟出票时间
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println(Thread.currentThread().getName() 
                    + "卖出第" + ticket + "张票");
                ticket--;
            } else {
                break;
            }
        }
    }
}

// 测试
public class Demo {
    public static void main(String[] args) {
        Ticket t = new Ticket();
        
        new Thread(t, "窗口1").start();
        new Thread(t, "窗口2").start();
        new Thread(t, "窗口3").start();
    }
}

// 可能出现的问题：
// 窗口1卖出第100张票
// 窗口2卖出第100张票  ← 重复卖票
// 窗口3卖出第-1张票   ← 卖出负数票
```

***

### 2. 解决方案一：synchronized 同步代码块

```java
public class Ticket implements Runnable {
    private int ticket = 100;
    private Object lock = new Object();  // 锁对象
    
    @Override
    public void run() {
        while (true) {
            // synchronized (锁对象) { 需要同步的代码 }
            synchronized (lock) {
                if (ticket > 0) {
                    try {
                        Thread.sleep(10);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    System.out.println(Thread.currentThread().getName() 
                        + "卖出第" + ticket + "张票");
                    ticket--;
                } else {
                    break;
                }
            }
        }
    }
}
```

**锁对象的选择**：

*   可以是任意对象，但必须是**同一个对象**
*   常用 `this`（当前对象）或 `类名.class`（类的字节码对象）

```java
// 使用 this 作为锁
synchronized (this) {
    // ...
}

// 使用类的字节码对象作为锁（静态方法中常用）
synchronized (Ticket.class) {
    // ...
}
```

***

### 3. 解决方案二：synchronized 同步方法

```java
public class Ticket implements Runnable {
    private int ticket = 100;
    
    @Override
    public void run() {
        while (true) {
            if (!sellTicket()) {
                break;
            }
        }
    }
    
    // 同步方法：锁对象是 this
    public synchronized boolean sellTicket() {
        if (ticket > 0) {
            try {
                Thread.sleep(10);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(Thread.currentThread().getName() 
                + "卖出第" + ticket + "张票");
            ticket--;
            return true;
        }
        return false;
    }
}
```

**同步方法的锁**：

*   非静态同步方法：锁是 `this`
*   静态同步方法：锁是 `类名.class`

***

### 4. 解决方案三：Lock 锁

```java
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class Ticket implements Runnable {
    private int ticket = 100;
    private Lock lock = new ReentrantLock();  // 创建锁对象
    
    @Override
    public void run() {
        while (true) {
            lock.lock();  // 加锁
            try {
                if (ticket > 0) {
                    Thread.sleep(10);
                    System.out.println(Thread.currentThread().getName() 
                        + "卖出第" + ticket + "张票");
                    ticket--;
                } else {
                    break;
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            } finally {
                lock.unlock();  // 释放锁（必须在 finally 中）
            }
        }
    }
}
```

***

### 5. synchronized vs Lock

| 对比项        | synchronized      | Lock                   |
| ---------- | ----------------- | ---------------------- |
| **类型**     | 关键字               | 接口                     |
| **加锁/释放锁** | 自动                | 手动（lock/unlock）        |
| **灵活性**    | 较低                | 较高                     |
| **可中断**    | 不可中断              | 可中断（lockInterruptibly） |
| **公平锁**    | 非公平               | 可以设置公平锁                |
| **条件变量**   | 只有一个（wait/notify） | 可以有多个 Condition        |
| **性能**     | JDK 1.6 后优化，差异不大  | 差异不大                   |

***

### 6. 死锁

**死锁**：两个或多个线程互相持有对方需要的锁，导致所有线程都无法继续执行。

```java
public class DeadLockDemo {
    private static Object lockA = new Object();
    private static Object lockB = new Object();
    
    public static void main(String[] args) {
        // 线程1：先获取 lockA，再获取 lockB
        new Thread(() -> {
            synchronized (lockA) {
                System.out.println("线程1获取了lockA");
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {}
                
                synchronized (lockB) {
                    System.out.println("线程1获取了lockB");
                }
            }
        }).start();
        
        // 线程2：先获取 lockB，再获取 lockA
        new Thread(() -> {
            synchronized (lockB) {
                System.out.println("线程2获取了lockB");
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {}
                
                synchronized (lockA) {
                    System.out.println("线程2获取了lockA");
                }
            }
        }).start();
    }
}

// 输出：
// 线程1获取了lockA
// 线程2获取了lockB
// （程序卡死，发生死锁）
```

**死锁产生的四个条件**：

1.  **互斥条件**：资源只能被一个线程持有
2.  **持有并等待**：持有资源的同时等待其他资源
3.  **不可剥夺**：资源只能由持有者主动释放
4.  **循环等待**：线程之间形成循环等待链

**避免死锁的方法**：

*   按相同顺序获取锁
*   使用 `tryLock()` 设置超时
*   使用 `Lock.lockInterruptibly()` 响应中断

### 7. volatile 关键字

**作用**：保证变量的**可见性**，禁止**指令重排序**。

#### 可见性问题

```java
public class VolatileDemo {
    private static boolean flag = true;  // 不加 volatile
    
    public static void main(String[] args) throws InterruptedException {
        new Thread(() -> {
            while (flag) {
                // 空循环
            }
            System.out.println("线程结束");
        }).start();
        
        Thread.sleep(1000);
        flag = false;  // 主线程修改 flag
        System.out.println("flag 已设置为 false");
        
        // 问题：子线程可能永远不会结束！
        // 原因：子线程读取的是自己工作内存中的 flag 副本
    }
}
```

#### 使用 volatile 解决

```java
private static volatile boolean flag = true;  // 加上 volatile
// 修改后，子线程能立即看到 flag 的变化
```

**volatile vs synchronized**：

| 对比项      | volatile | synchronized |
| -------- | -------- | ------------ |
| **可见性**  | ✅ 保证     | ✅ 保证         |
| **原子性**  | ❌ 不保证    | ✅ 保证         |
| **性能**   | 较高       | 较低           |
| **适用场景** | 一写多读     | 多写多读         |

> ⚠️ **注意**：`volatile` 不保证原子性！`count++` 这样的操作仍然需要同步。

## 五、线程通信

### 1. wait/notify 机制

| 方法                   | 说明           |
| -------------------- | ------------ |
| `wait()`             | 让当前线程等待，并释放锁 |
| `wait(long timeout)` | 等待指定毫秒后自动唤醒  |
| `notify()`           | 唤醒一个等待的线程    |
| `notifyAll()`        | 唤醒所有等待的线程    |

**注意**：这些方法必须在 **synchronized** 块中调用，且调用对象必须是**锁对象**。

***

### 2. 生产者-消费者模式

```java
public class ProducerConsumerDemo {
    public static void main(String[] args) {
        DataBox box = new DataBox();
        
        // 生产者
        new Thread(() -> {
            for (int i = 1; i <= 5; i++) {
                box.put(i);
            }
        }, "生产者").start();
        
        // 消费者
        new Thread(() -> {
            for (int i = 1; i <= 5; i++) {
                box.take();
            }
        }, "消费者").start();
    }
}

class DataBox {
    private int data;
    private boolean hasData = false;  // 是否有数据
    
    // 生产数据
    public synchronized void put(int data) {
        // 有数据则等待
        while (hasData) {
            try {
                wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        
        this.data = data;
        hasData = true;
        System.out.println(Thread.currentThread().getName() + " 生产了：" + data);
        
        notify();  // 唤醒消费者
    }
    
    // 消费数据
    public synchronized void take() {
        // 没有数据则等待
        while (!hasData) {
            try {
                wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        
        System.out.println(Thread.currentThread().getName() + " 消费了：" + data);
        hasData = false;
        
        notify();  // 唤醒生产者
    }
}
```

**输出**：

    生产者 生产了：1
    消费者 消费了：1
    生产者 生产了：2
    消费者 消费了：2
    ...

> 💡 **为什么用 `while` 而不是 `if`？**
> 防止虚假唤醒（spurious wakeup），线程可能在没有被 `notify` 的情况下醒来，需要重新检查条件。

## 六、线程池

### 1. 为什么需要线程池

**问题**：频繁创建和销毁线程，开销大，效率低。

**解决**：预先创建一组线程，放入"池"中，需要时取出使用，用完归还。

    ┌────────────────────────────────────────────────────┐
    │                      线程池                         │
    │  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐           │
    │  │线程1 │  │线程2 │  │线程3 │  │线程4 │ ...       │
    │  └──────┘  └──────┘  └──────┘  └──────┘           │
    │                                                    │
    │              任务队列：[任务A] [任务B] [任务C]       │
    └────────────────────────────────────────────────────┘

    工作流程：
    1. 提交任务到任务队列
    2. 线程池中的线程从队列取任务执行
    3. 执行完毕后，线程归还到池中，等待下一个任务

### 2. 线程池的好处

| 好处         | 说明               |
| ---------- | ---------------- |
| **降低资源消耗** | 复用线程，减少创建和销毁的开销  |
| **提高响应速度** | 任务到达时，无需等待线程创建   |
| **便于管理**   | 统一管理线程，可以控制最大并发数 |

***

### 3. Executors 创建线程池（简单方式）

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class Demo {
    public static void main(String[] args) {
        // 1. 创建固定大小的线程池
        ExecutorService pool = Executors.newFixedThreadPool(3);
        
        // 2. 提交任务
        pool.submit(() -> {
            System.out.println(Thread.currentThread().getName() + "执行任务1");
        });
        
        pool.submit(() -> {
            System.out.println(Thread.currentThread().getName() + "执行任务2");
        });
        
        pool.submit(() -> {
            System.out.println(Thread.currentThread().getName() + "执行任务3");
        });
        
        // 3. 关闭线程池
        pool.shutdown();
    }
}
```

***

### 4. Executors 提供的四种线程池

| 方法                          | 说明      | 特点           |
| --------------------------- | ------- | ------------ |
| `newFixedThreadPool(n)`     | 固定大小线程池 | 固定 n 个线程     |
| `newCachedThreadPool()`     | 缓存线程池   | 线程数量不固定，按需创建 |
| `newSingleThreadExecutor()` | 单线程线程池  | 只有 1 个线程     |
| `newScheduledThreadPool(n)` | 定时任务线程池 | 可以执行定时任务     |

```java

// 缓存线程池（线程数量可变）

// 单线程线程池

// 定时任务线程池
ScheduledExecutorService pool4 = Executors.newScheduledThreadPool(3);
pool4.schedule(() -> {
    System.out.println("延迟 3 秒执行");
}, 3, TimeUnit.SECONDS);
```

### 5. submit() vs execute()

| 方法                          | 返回值    | 异常处理         | 参数类型                |
| --------------------------- | ------ | ------------ | ------------------- |
| `execute(Runnable)`         | 无      | 直接抛出         | 只能是 Runnable        |
| `submit(Runnable/Callable)` | Future | 封装在 Future 中 | Runnable 或 Callable |

```java
// execute：无返回值

    System.out.println("execute 执行");
});

// submit：有返回值

    return "submit 执行结果";
});
String result = future.get();  // 获取返回值
```

### 6. shutdown() vs shutdownNow()

| 方法                                | 说明                          |
| --------------------------------- | --------------------------- |
| `shutdown()`                      | 平缓关闭：不接受新任务，等待已提交任务执行完毕     |
| `shutdownNow()`                   | 立即关闭：尝试中断正在执行的任务，返回未执行的任务列表 |
| `isShutdown()`                    | 判断是否已调用 shutdown            |
| `isTerminated()`                  | 判断所有任务是否都已完成                |
| `awaitTermination(timeout, unit)` | 等待线程池终止，超时返回 false          |

```java
pool.shutdown();  // 平缓关闭

// 等待线程池完全终止，最多等 60 秒
if (pool.awaitTermination(60, TimeUnit.SECONDS)) {
    System.out.println("线程池已完全终止");
} else {
    System.out.println("超时，强制关闭");
    pool.shutdownNow();
}
```

### 7. 自定义线程池（推荐）

#### 为什么需要自定义线程池

`Executors` 创建的线程池存在风险：

*   `newFixedThreadPool` 和 `newSingleThreadExecutor`：任务队列无限大，可能导致 OOM
*   `newCachedThreadPool`：线程数量无限大，可能导致 OOM

**阿里巴巴开发规范**：**禁止使用 Executors 创建线程池，推荐使用 ThreadPoolExecutor**

#### 工作流程图解

    提交任务
        │
        ↓
    ┌─────────────────────────────────────────────────────────────────┐
    │                        线程池                                    │

    │  ┌─────────────────────┐    ┌─────────────────────┐             │
    │  │    核心线程区        │    │    任务队列          │             │
    │  │  ┌────┐ ┌────┐     │    │  [任务] [任务] ...   │             │
    │  │  │线程│ │线程│ ... │    │                      │             │

    │  └─────────────────────┘              │                          │
    │                                       ↓                          │
    │  ┌─────────────────────┐        队列满了？                       │
    │  │    临时线程区        │        ┌──────┐                        │
    │  │  ┌────┐ ┌────┐     │←───────│ 创建 │ 临时线程                │
    │  │  │线程│ │线程│ ... │        └──────┘                        │

    └─────────────────────────────────────────────────────────────────┘

**任务处理流程**：

1.  提交任务，优先使用核心线程执行
2.  核心线程满了 → 放入任务队列等待
3.  任务队列满了 → 创建临时线程执行
4.  达到最大线程数且队列满 → 执行拒绝策略

#### 自定义线程池示例

```java
import java.util.concurrent.*;

public class Demo {
    public static void main(String[] args) {
        // 创建自定义线程池
        ThreadPoolExecutor pool = new ThreadPoolExecutor(
            2,                                  // 核心线程数
            5,                                  // 最大线程数
            60,                                 // 空闲线程存活时间
            TimeUnit.SECONDS,                   // 时间单位
            new ArrayBlockingQueue<>(10),       // 任务队列（容量 10）
            Executors.defaultThreadFactory(),   // 默认线程工厂
            new ThreadPoolExecutor.AbortPolicy()// 拒绝策略
        );
        
        // 提交任务
        for (int i = 0; i < 15; i++) {
            int taskNum = i;
            pool.submit(() -> {
                System.out.println(Thread.currentThread().getName() 
                    + " 执行任务 " + taskNum);
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            });
        }
        
        // 关闭线程池
        pool.shutdown();
    }
}
```

| `DiscardPolicy`       | 直接丢弃任务，不抛异常                               |
| `DiscardOldestPolicy` | 丢弃队列中最老的任务，然后重新提交                         |

```java

// 丢弃最老的任务

```

### 1. 什么是原子类

原子类提供**无锁**的线程安全操作，比 synchronized 性能更高。

```java
import java.util.concurrent.atomic.AtomicInteger;

public class Demo {

        t2.start();
        t1.join();
        t2.join();
        
        System.out.println("count = " + count.get());  // 20000（正确）
    }
}
```

## 八、ThreadLocal 线程本地变量

### 1. 什么是 ThreadLocal

`ThreadLocal` 为每个线程提供**独立的变量副本**，实现线程隔离。每个线程访问的都是自己的副本，互不干扰。

```
┌─────────────────────────────────────────────────────────────┐
│  主内存                                                      │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  ThreadLocal<String> name = new ThreadLocal<>();    │    │
│  └─────────────────────────────────────────────────────┘    │
│                          │                                   │
│          ┌───────────────┼───────────────┐                  │
│          ↓               ↓               ↓                  │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐             │
│   │  线程A   │    │  线程B   │    │  线程C   │             │
│   │ name="张三"│    │ name="李四"│    │ name="王五"│             │
│   └──────────┘    └──────────┘    └──────────┘             │
│   各线程拥有独立副本，互不干扰                                │
└─────────────────────────────────────────────────────────────┘
```

**ThreadLocal vs 普通变量：**

| 类型 | 线程访问 | 数据隔离 |
| ---- | -------- | -------- |
| 普通成员变量 | 所有线程共享同一个变量 | ❌ 不隔离 |
| ThreadLocal | 每个线程独立副本 | ✅ 完全隔离 |

***

### 2. 基本使用

#### 常用方法

| 方法 | 说明 |
| ---- | ---- |
| `set(T value)` | 设置当前线程的变量值 |
| `get()` | 获取当前线程的变量值 |
| `remove()` | 移除当前线程的变量值（防止内存泄漏）|
| `initialValue()` | 设置初始值（重写此方法或使用 withInitial）|

#### 基本示例

```java
public class ThreadLocalDemo {
    // 创建 ThreadLocal 变量
    private static ThreadLocal<String> threadLocal = new ThreadLocal<>();
    
    public static void main(String[] args) {
        // 线程1
        new Thread(() -> {
            threadLocal.set("线程1的数据");
            System.out.println(Thread.currentThread().getName() + "：" + threadLocal.get());
            threadLocal.remove();  // 用完记得移除
        }, "Thread-1").start();
        
        // 线程2
        new Thread(() -> {
            threadLocal.set("线程2的数据");
            System.out.println(Thread.currentThread().getName() + "：" + threadLocal.get());
            threadLocal.remove();
        }, "Thread-2").start();
        
        // 主线程
        threadLocal.set("主线程的数据");
        System.out.println(Thread.currentThread().getName() + "：" + threadLocal.get());
        threadLocal.remove();
    }
}

// 输出（顺序可能不同）：
// Thread-1：线程1的数据
// Thread-2：线程2的数据
// main：主线程的数据
```

#### 设置初始值

```java
// 方式一：重写 initialValue() 方法
ThreadLocal<Integer> count = new ThreadLocal<>() {
    @Override
    protected Integer initialValue() {
        return 0;
    }
};

// 方式二：使用 withInitial()（推荐，更简洁）

// 使用
System.out.println(count.get());  // 0（未 set 时返回初始值）
```

***

### 3. 实现原理

#### 数据结构

每个线程（Thread）内部维护一个 `ThreadLocalMap`，存储该线程的所有 ThreadLocal 变量：

```
Thread 对象
    └── ThreadLocalMap（类似 HashMap）
            ├── Entry(ThreadLocal<?> key, Object value)
            ├── Entry(ThreadLocal<?> key, Object value)
            └── ...

示例：
Thread-1
    └── ThreadLocalMap
            ├── Entry(nameLocal, "张三")
            └── Entry(ageLocal, 18)

Thread-2
    └── ThreadLocalMap
            ├── Entry(nameLocal, "李四")
            └── Entry(ageLocal, 20)
```

#### get() 方法原理（简化）

```java
public T get() {
    // 1. 获取当前线程
    Thread t = Thread.currentThread();
    // 2. 获取当前线程的 ThreadLocalMap
    ThreadLocalMap map = t.threadLocals;
    // 3. 以当前 ThreadLocal 对象为 key，获取 value
    if (map != null) {
        Entry e = map.getEntry(this);
        if (e != null) {
            return (T) e.value;
        }
    }
    // 4. 没有则返回初始值
    return setInitialValue();
}
```

***

### 4. 内存泄漏问题

#### 问题原因

`ThreadLocalMap` 的 Entry 使用**弱引用**指向 ThreadLocal 对象：

```java
static class Entry extends WeakReference<ThreadLocal<?>> {
    Object value;  // 强引用
    Entry(ThreadLocal<?> k, Object v) {
        super(k);  // key 是弱引用
        value = v; // value 是强引用
    }
}
```

**问题场景：**

```
ThreadLocal 对象 ←── 弱引用 ── Entry.key
                              Entry.value ──→ 实际数据（强引用）

当 ThreadLocal 对象没有外部强引用时：
1. GC 回收 ThreadLocal 对象
2. Entry.key 变成 null
3. 但 Entry.value 仍然存在（强引用）
4. 如果线程长期存活（如线程池），value 无法被回收 → 内存泄漏
```

#### 解决方案

**使用完毕后调用 `remove()` 方法：**

```java
ThreadLocal<User> userLocal = new ThreadLocal<>();

try {
    userLocal.set(currentUser);
    // 业务逻辑...
} finally {
    userLocal.remove();  // ✅ 必须调用，防止内存泄漏
}
```

> ⚠️ **特别注意**：在使用线程池时，线程会被复用，如果不调用 `remove()`，下次使用该线程时可能会获取到上次的数据！

### 5. Web 开发中的应用

在 Web 应用中，每个 HTTP 请求由一个线程处理。使用 ThreadLocal 存储用户信息，可以在整个请求链路中方便地获取当前用户。

#### UserContext 工具类

```java
public class UserContext {

    private static final ThreadLocal<String> USERNAME = new ThreadLocal<>();

        USER_ID.set(userId);
    }
    
    public static Long getUserId() {
        return USER_ID.get();
    }
    
    public static void setUsername(String username) {
        USERNAME.set(username);
    }
    
    public static String getUsername() {
        return USERNAME.get();
    }
    
    /** 清理上下文（必须在请求结束时调用，防止内存泄漏）*/
    public static void clear() {
        USER_ID.remove();
        USERNAME.remove();
    }
}
```

### 6. InheritableThreadLocal

普通 `ThreadLocal` 在父子线程间**不共享**数据。如果需要子线程继承父线程的数据，使用 `InheritableThreadLocal`：

```java
// 普通 ThreadLocal：子线程无法获取父线程的值
ThreadLocal<String> threadLocal = new ThreadLocal<>();
threadLocal.set("父线程数据");

new Thread(() -> {
    System.out.println(threadLocal.get());  // null
}).start();

// InheritableThreadLocal：子线程可以继承父线程的值
InheritableThreadLocal<String> inheritableLocal = new InheritableThreadLocal<>();
inheritableLocal.set("父线程数据");

new Thread(() -> {
    System.out.println(inheritableLocal.get());  // 父线程数据
}).start();
```

> ⚠️ **注意**：`InheritableThreadLocal` 在线程池场景下可能失效，因为线程池中的线程是复用的，不是每次都新建子线程。

***

### 7. ThreadLocal 总结

| 特性 | 说明 |
| ---- | ---- |
| **作用** | 线程隔离，每个线程独立副本 |
| **原理** | 每个 Thread 持有一个 ThreadLocalMap |
| **内存泄漏** | 用完必须调用 `remove()` |
| **典型场景** | Web 请求上下文、数据库连接、日期格式化 |
| **子线程继承** | 使用 `InheritableThreadLocal` |
