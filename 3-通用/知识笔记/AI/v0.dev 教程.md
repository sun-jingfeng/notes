# v0.dev 教程

**v0.dev** 是 Vercel 推出的 **AI 驱动的前端/全栈生成工具**，通过自然语言描述、截图或 Figma 链接生成可运行代码，并支持在网页内迭代、连接数据库/API、一键部署到 Vercel。适合快速原型、从设计稿出代码、以及不依赖设计稿的「描述即出 UI」场景。

***

## 一、核心能力与定位

### 1.1 能做什么

| 能力 | 说明 |
| ----------------------- | ---------------------------------------------------------------------- |
| **文本生成 UI** | 用自然语言描述界面或功能，v0 生成 React/Next.js + Tailwind + shadcn/ui 等现代栈代码 |
| **截图/文件转代码** | 上传设计截图、草图或 Figma 链接，提取视觉与结构信息并生成对应代码 |
| **对话式迭代** | 在聊天中继续提需求（如「加搜索框」「改成暗色」），实时预览并更新代码 |
| **可视化编辑** | Design Mode 下点选元素修改样式，所见即所得 |
| **全栈与集成** | 可连接数据库（Supabase、Neon、Upstash 等）、AI 模型、Stripe、外部 API，生成带后端的应用 |
| **一键部署** | 直接发布到 Vercel，支持自定义域名、GitHub 同步 |

### 1.2 技术栈与输出

- **前端**：React、TypeScript、Tailwind CSS、shadcn/ui，兼顾响应式与可访问性（a11y）。
- **全栈**：Next.js App Router，可配合 Vercel 生态（Server Actions、AI SDK 等）。
- **输出**：生产可用代码，可复制到本地项目或通过 GitHub 同步、在 v0 内直接改。

### 1.3 适用角色

| 角色 | 典型用法 |
| -------------- | ------------------------------------------------------------------------------------ |
| **产品/PM** | 快速做原型、对齐需求、收集反馈，再交工程实现 |
| **设计师** | 将 Figma 稿或截图转为可交互组件，便于与开发 handoff |
| **前端/工程师** | 搭脚手架、写通用组件、接数据库/API，把 v0 当 AI 结对编程 |
| **创业者/无代码** | 用描述或设计快速做出 MVP，再决定是否投入开发 |

## 二、快速上手

### 2.1 入口与登录

1. 打开 [v0.app](https://v0.app)，使用 Vercel / GitHub 等账号登录。
2. 新建对话或新建项目（Project），在同一个项目下多轮对话共享上下文。

### 2.2 基本流程

```
用文字/截图/Figma 描述要做的界面或功能
        ↓
v0 生成可运行应用（常给出多版供选）
        ↓
在对话中继续提修改，或在 Code / Design 模式下直接改
        ↓
满意后复制代码到本地，或一键发布到 Vercel
```

### 2.3 入门提示词示例

- 「一个联系表单，提交后发邮件」
- 「带图表的用户分析看板」
- 「SaaS 产品落地页：hero、功能区块、定价表」
- 「待办应用：增删改、完成状态、筛选」

从简单、具体的描述开始，再在对话里补充细节（布局、配色、技术栈等）。

### 2.4 迭代方式

| 方式 | 说明 |
| ---------- | ---------------------------------------------------------------------- |
| **对话** | 自然语言描述变更，如「在商品列表上加一个搜索框」「改成深色主题」 |
| **Code 模式** | 切到 Code 标签页，直接编辑生成的代码，实时预览 |
| **Design 模式** | 在预览里点选元素，用可视化面板改样式（颜色、间距、字体等） |

## 三、提示与使用技巧

### 3.1 提示词原则

- **具体优于笼统**：说明功能、布局、交互、技术偏好，而不是只说「做个网站」。
- **复杂项目先规划**：多功能、多角色系统可先让 v0 出 PRD 或拆任务，再分步实现。
- **标明技术栈**：若需 React、Next.js、Tailwind、shadcn/ui、某数据库，在首条或后续提示里写明。
- **分步生成**：大页面先拆成区块（导航、表单、列表等），分别生成再组合，减少布局错乱。

### 3.2 从 Figma 出代码

- 在对话里用**附件**插入 Figma 链接（或按 v0 当前支持的 Figma 导入方式）。
- v0 会解析视觉布局和设计 token（颜色、间距等），生成质量通常优于纯截图。
- **建议**：在 Figma 里按 UI 模块分 frame（导航、侧栏、表单、区块），一次导一个或少量 frame，再在 v0 里用提示词指导如何排列、组合成完整页面。

### 3.3 与 Cursor + Figma MCP 的差异

| 维度 | v0.dev | Cursor + Figma MCP |
| ---------- | ------------------------------------------- | --------------------------------------------------------- |
| **设计来源** | 文字、截图、Figma 链接均可 | 必须有 Figma 设计链接，读设计稿源数据 |
| **工作环境** | 浏览器内聊天 + 预览 + 部署 | 在 Cursor IDE 内，代码直接落在当前仓库 |
| **与现有项目结合** | 复制代码或通过 GitHub 同步到仓库 | 可根据当前项目组件、规范做适配 |
| **无设计稿时** | 纯文字/截图即可出 UI，适合快速试方案 | 无 Figma 时 MCP 无法参与 |

有 Figma 且希望代码长在既有项目里时，Cursor + Figma MCP 更贴项目；无设计稿或想在网页里快速试多种 UI 时，v0 更合适。

## 四、定价与用量（截至文档更新时）

### 4.1 套餐概览

| 套餐 | 月费 | 月度额度 | 适用 |
| ---------- | ------------ | -------------------- | ------------------------------ |
| **Free** | $0 | $5 额度 | 体验、个人小项目 |
| **Premium** | $20/月 | $20 额度 | 个人高频、更高限额 |
| **Team** | $30/用户/月 | 每用户 $30，可共享池 | 团队协作、共享项目与额度 |
| **Business** | $100/用户/月 | 每用户 $30，可共享池 | 注重数据与隐私的团队 |
| **Enterprise** | 定制 | 定制 | 大企业、SSO、高级管控 |

具体额度与限制以 [v0 定价页](https://v0.dev/docs/pricing) 为准。

### 4.2 额度与 Token

- 额度类似预付费余额，按 **Token** 消耗（输入 + 输出）。
- 未用完的月度额度可滚存，通常约 30 天后过期；额外购买的额度过期时间以官方说明为准。
- 提供多种模型档位（如 v0 Mini / Pro / Max），输入输出单价不同，可按需选择以控制成本。

### 4.3 技术限制（参考官方文档）

| 项目 | 典型值 |
| -------------- | ------------------------- |
| 上下文窗口 | 约 128,000 token（输入 + 历史 + 当前提示） |
| 单张图片 | 约按 1,000 token 计 |
| 单次输出 | 约 32,000 token |

> 💡 大项目拆成小组件、提示写具体，可减少反复生成，节省额度。

## 五、集成与嵌入

### 5.1 在 Cursor / 其他编辑器里用 v0

v0 支持嵌入到现有开发流程：

- **Cursor**：通过 [v0 × Cursor 集成](https://v0.dev/docs/cursor) 在编辑器内调用 v0 能力。
- 另有 Cline、Zed、OpenAI Codex 等集成，见 v0 文档的 Embed 章节。

### 5.2 API 与自定义工具

- **v0 Model API**：Premium / Team 等计划可创建 API Key，用兼容 OpenAI 的接口或 Vercel AI SDK 调用 v0 模型，集成到自建工具或流水线。
- Key 与用量管理：在 v0 的 [Billing](https://v0.app/chat/settings/billing)、[Usage](https://v0.app/chat/settings/usage) 中查看。

## 六、学习价值小结

| 场景 | 是否值得学 |
| ----------------------- | ---------- |
| 快速从想法/设计出可运行 UI，且不依赖本地项目 | ✅ 价值高 |
| 团队已有 Figma，但希望代码直接进仓库、贴合现有组件 | 可优先用 Cursor + Figma MCP，v0 作补充 |
| 产品/设计/创业需要低成本验证界面与流程 | ✅ 价值高 |
| 前端/全栈想用 AI 做脚手架、组件、接库与 API | ✅ 价值高 |

**结论**：若你经常需要「无设计稿也能出 UI」或「在网页里快速试多种方案」，v0.dev 学习价值很高；若主要工作流是「Figma 定稿 → 在既有项目里写代码」，可把 v0 当作补充工具，与 Cursor + Figma MCP 搭配使用。
