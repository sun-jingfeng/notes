# Git 配置

## 用户信息

### 命令

```bash
# 设置
git config --global user.name "<用户名>"
git config --global user.email "<邮箱>"

# 查询
git config --global user.name
git config --global user.email

# 查看所有配置
git config --list
git config --list --show-origin    # 显示配置来源
```

### 说明

- `--local`（默认值）：适用于当前仓库
- `--global`：适用于用户在其操作系统账户下的所有仓库
- `--system`：适用于系统中所有用户的所有仓库

## SSH 密钥

### 命令

```bash
ssh-keygen -t ed25519 -C "<注释>"        # 以 ED25519 算法生成密钥（推荐）
ssh-keygen -t rsa -b 4096 -C "<注释>"    # 以 4096-bit RSA 算法生成密钥
```

### 配置流程

1. 选择一种算法对应的命令执行，连续按回车生成密钥
2. 在用户根目录的 `.ssh` 隐藏文件夹中找到后缀为 `.pub` 的文件，复制其中的全部内容（此内容为公钥）
3. 将公钥添加到远程仓库（GitHub、GitLab、Gitee 等）

### 测试连接

```bash
ssh -T git@github.com    # 测试 GitHub 连接
ssh -T git@gitlab.com    # 测试 GitLab 连接
ssh -T git@gitee.com     # 测试 Gitee 连接
```

> 同一台电脑的公钥可以添加给多个远程仓库，同一个远程仓库也可以添加多台电脑的公钥

## 常用配置项

```bash
# 设置默认编辑器
git config --global core.editor "vim"
git config --global core.editor "code --wait"    # VS Code

# 设置默认分支名（新建仓库时）
git config --global init.defaultBranch main

# 拉取时默认使用变基
git config --global pull.rebase true

# 自动处理换行符（跨平台协作推荐）
git config --global core.autocrlf input    # macOS/Linux
git config --global core.autocrlf true     # Windows

# 启用颜色输出
git config --global color.ui auto

# 设置凭证缓存（避免重复输入密码）
git config --global credential.helper cache                  # 缓存 15 分钟
git config --global credential.helper 'cache --timeout=3600' # 缓存 1 小时
```

## Git 别名

通过配置别名可以简化常用命令，提高效率。

### 配置方法

```bash
git config --global alias.<别名> '<命令>'
```

### 常用别名示例

```bash
# 简化常用命令
git config --global alias.co 'checkout'
git config --global alias.br 'branch'
git config --global alias.ci 'commit'
git config --global alias.st 'status'

# 美化日志输出
git config --global alias.lg "log --oneline --graph --all"
git config --global alias.ll "log --oneline -n 10"

# 查看最后一次提交
git config --global alias.last 'log -1 HEAD'

# 撤销暂存
git config --global alias.unstage 'reset HEAD --'
```

### 使用

```bash
git co main       # 等同于 git checkout main
git st            # 等同于 git status
git lg            # 查看美化的提交历史图
```

## .gitignore

### 说明

`.gitignore` 文件用于指定 Git 应该忽略的文件和目录，不纳入版本控制。

### 常用规则

```gitignore
# 忽略指定文件
config.local.js
.env

# 忽略指定目录
node_modules/
dist/
build/
.idea/

# 忽略指定后缀
*.log
*.tmp
*.class

# 忽略所有 .a 文件，但保留 lib.a
*.a
!lib.a

# 忽略根目录下的 TODO 文件，但不忽略子目录的 TODO
/TODO

# 忽略 doc 目录下所有 .pdf 文件
doc/**/*.pdf
```

### 已追踪文件的处理

如果文件已被追踪，添加到 `.gitignore` 后不会自动忽略，需要先取消追踪：

```bash
git rm --cached <文件名>           # 取消追踪单个文件
git rm -r --cached <目录名>        # 取消追踪整个目录
```
