# Git 基础操作

## 获取仓库

### 初始化本地仓库

```bash
git init
```

- 在需要初始化为 Git 仓库的目录中执行
- 初始化后生成一个 `.git` 子目录（隐藏文件），用于存储 Git 的版本控制信息和配置

### 克隆远程仓库

```bash
git clone <远程地址>                          # 克隆仓库
git clone <远程地址> <目录名>                  # 克隆到指定目录
git clone --depth 1 <远程地址>                # 浅克隆（只获取最新提交，适合大仓库）
git clone -b <分支名> <远程地址>               # 克隆指定分支
git clone --recurse-submodules <远程地址>     # 克隆仓库并初始化子模块
```

## 基本工作流

### 查看状态

```bash
git status       # 查看当前分支的状态
git status -s    # 简洁模式输出
```

### 暂存更改

```bash
git add <文件名>              # 添加单个文件
git add *.<文件后缀>          # 添加所有指定后缀的文件
git add .                    # 添加当前目录下的所有改动
git add -p                   # 交互式暂存，按代码块选择
```

### 取消暂存

```bash
git reset HEAD <文件名>       # 取消暂存（保留工作区修改）
git restore --staged <文件名> # 取消暂存（Git 2.23+ 推荐）
```

### 提交更改

```bash
git commit -m "<提交信息>"           # 创建新的提交
git commit -am "<提交信息>"          # 跳过暂存直接提交（仅限已追踪文件）
git commit --amend                  # 修改上一次提交的提交信息 / 将新内容追加到上一次提交
```

> `git commit --amend` 会重写提交历史，如果这个提交已经被推送到远程仓库，需要使用 `git push --force` 强制推送

## 查看历史

### 命令

```bash
# 整体
git log                        # 查看当前分支的提交历史
git log -n 5                   # 查看当前分支指定数量的提交历史
git log --all                  # 查看所有分支的提交历史
git log --oneline              # 将每个提交历史压缩到一行
git log --graph                # 以图形方式查看提交历史
git log --author="<用户名>"     # 搜索特定用户的提交历史
git log -p                     # 显示每次提交的差异
git log --stat                 # 显示每次提交的文件统计

# 单个文件
git log <文件路径>                       # 查看指定文件的提交历史
git log -p <文件路径>                    # 查看文件每次修改的详细差异
git log -- <文件路径（以-开头）>          # 查看指定文件的提交历史（文件路径以-开头）

# 查看某次提交
git show <哈希值>              # 查看某次提交的完整差异
git show --stat <哈希值>       # 查看某次提交修改了哪些文件

# 查看每行代码的修改者
git blame <文件路径>           # 查看文件每一行的最后修改者
```

### 说明

- 需使用 less 视图快捷键（见「5-进阶技巧」）
- `commit` 后面的一长串字符是哈希值，可以用来回退、比较等，使用时只需复制其一部分
- `--` 用于明确指示命令参数的结束，当文件路径不以 `-` 开头时可以省略

## 比较差异

```bash
git diff                              # 比较工作区与暂存区的差异
git diff --cached                     # 比较暂存区与上一次提交的差异
git diff HEAD                         # 比较工作区与上一次提交的差异
git diff <旧哈希值> <新哈希值>          # 比较旧提交到新提交的差异
git diff <分支名字1> <分支名字2>        # 比较分支1到分支2的差异
git diff HEAD~<n> HEAD                # 比较前 n 次提交到当前提交的差异
```

## 撤销与恢复

### 撤销工作区修改

```bash
git checkout -- <文件路径>             # 撤销工作区的修改
git restore <文件路径>                 # 撤销工作区的修改（Git 2.23+ 推荐）
```

### 回退提交

```bash
git reset --soft <哈希值>      # 回退到指定提交，将更改存入暂存区
git reset <哈希值>             # 回退到指定提交，将更改存入工作区
git reset --hard <哈希值>      # 回退到指定提交，丢弃所有更改（慎用）
git revert <哈希值>            # 撤销某次提交的更改并创建新提交（不改变历史，推荐用于已推送的提交）
```

### 恢复单个文件到指定版本

```bash
git checkout <哈希值> <文件路径>                   # 恢复指定文件，变更存入暂存区
git checkout <哈希值> -- <文件路径（以-开头）>      # 恢复指定文件（文件路径以-开头）
git restore --source <哈希值> <文件路径>           # 恢复指定文件，变更存入工作区（Git 2.23+）
```

### 清理未追踪的文件

```bash
git clean -n     # 预览将被删除的文件
git clean -f     # 删除未追踪的文件
git clean -fd    # 删除未追踪的文件和目录
```
