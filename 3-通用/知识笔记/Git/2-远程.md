# Git 远程操作

## 密钥

### 命令

```bash
ssh-keygen -t rsa -b 2048 -C "<注释>"    # 在用户根目录以 2048-bit RSA 算法生成密钥
ssh-keygen -t ed25519 -C "<注释>"        # 在用户根目录以 ED25519 算法生成密钥
```

### 说明

**流程：**
1. 选择一种算法对应的命令执行，连续按回车生成密钥
2. 在用户根目录的名字为 `.ssh` 的隐藏文件夹中找到后缀为 `.pub` 的文件，复制其中的全部内容（此内容为公钥）
3. 将公钥添加到远程仓库（GitHub、GitLab、Gitee 等）

> 同一台电脑的公钥可以添加给多个远程仓库，同一个远程仓库也可以添加多台电脑的公钥

## 远程仓库

### 命令

```bash
git remote -v                                    # 查看所有远程仓库
git remote add <远程名字> <远程地址>              # 添加远程仓库
git remote remove <远程名字>                      # 删除远程仓库
git remote set-url <远程名字> <远程地址>          # 更改远程仓库地址
git remote show <远程名字>                        # 查看远程仓库的详细信息
git remote rename <旧远程名字> <新远程名字>        # 重命名远程仓库
```

### 说明

- 远程名字任意，习惯为 `origin`
- 远程地址需要从远程仓库复制，一般有 https 和 ssh（需配置密钥）两种格式

## 初始化远程仓库

### 全局设置

```bash
git config --global user.name "<用户名>"
git config --global user.email "<邮箱>"
```

### 创建新项目

```bash
git init
git add .
git commit -m "<提交信息>"
git remote add <远程名字> <远程地址>
git push -u origin "<分支名字>"
```

### 已有项目

```bash
git remote add <远程名字> <远程地址>
git push -u origin "<分支名字>"
```

## 远程分支

### 命令

```bash
git checkout <分支名字>                                      # 拉取指定远程分支并切换到它
git push --all                                              # 推送本地所有分支到默认远程仓库
git push --all <远程名字>                                    # 推送本地所有分支到指定远程仓库
git push -u <远程名字> <分支名字>                             # 推送当前分支到指定远程分支，并自动关联
git push <远程名字> --delete <分支名字>                       # 删除远程分支
git branch -vv                                              # 查看本地分支关联的远程分支
git branch --set-upstream-to=<远程名字>/<分支名字>            # 将当前分支与远程分支关联
git branch --unset-upstream                                 # 取消当前分支与远程分支的关联
```

### 说明

"关联" 指本地分支与远程分支之间的跟踪关系，也称为 "上游" 关系。可以简化与远程仓库的交互操作，包括但不限于推送、拉取时无需指定远程仓库和分支名字。

---

## 推送、获取、拉取

### 命令

```bash
git push                                   # 推送当前分支到关联的远程分支
git push -f                                # 强制推送当前分支到关联的远程分支
git push <远程名字>                         # 推送当前分支到指定远程仓库的关联分支或同名分支
git push <远程名字> <分支名字>               # 推送当前分支到指定远程分支

git fetch                                  # 获取默认远程仓库的提交信息
git fetch --all                            # 获取所有远程仓库的提交信息
git fetch <远程名字>                        # 获取指定远程仓库的提交信息
git fetch <远程名字> <分支名字>              # 获取指定远程分支的提交信息

git pull                                   # 获取当前分支关联的远程分支的提交信息，并合并到当前分支
git pull <远程名字> <分支名字>               # 获取指定远程分支的提交信息，并合并到当前分支
```

### 说明

- `fetch` 操作只获取远程的最新提交信息，不合并
- `pull` 操作相当于 `fetch`、`merge` 操作的结合
