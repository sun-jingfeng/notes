# Git 分支与合并

## 分支操作

### 查看分支

```bash
git branch           # 查看本地分支
git branch -r        # 查看远程分支
git branch -a        # 查看本地和远程分支
git branch -av       # 查看本地和远程分支，并显示最后一次提交信息
git branch -vv       # 查看本地分支关联的远程分支
```

### 创建分支

```bash
git branch <新分支名字>                   # 从当前分支创建新分支
git branch <新分支名字> <旧分支名字>       # 从指定分支创建新分支
```

### 切换分支

```bash
git checkout <分支名字>                   # 切换到指定本地分支
git checkout -b <新分支名字>              # 创建新分支并切换到它
git switch <分支名字>                     # 切换到指定本地分支（Git 2.23+ 推荐）
git switch -c <新分支名字>                # 创建新分支并切换到它（Git 2.23+ 推荐）
```

### 重命名分支

```bash
git branch -m <新名字>                    # 重命名当前分支
git branch -m <旧名字> <新名字>            # 重命名指定分支
```

### 删除分支

```bash
git branch -d <分支名字>                  # 删除本地分支（已合并）
git branch -D <分支名字>                  # 强制删除本地分支（未合并也可删除）
```

## 合并分支

### 命令

```bash
git merge <分支名字>                                      # 将指定分支合并到当前分支，自动提交
git merge --no-commit <分支名字>                          # 将指定分支合并到当前分支，不自动提交
git merge --squash <分支名字>                             # 将指定分支的更改压缩为一个提交应用到当前分支
git merge <分支名字1> <分支名字2> ... <分支名字n>          # 将多个分支合并到当前分支，自动提交
git merge --abort                                        # 当合并遇到冲突时，终止当前合并
```

### 解决冲突

当合并时遇到冲突：

1. 手动编辑冲突文件，解决冲突标记：
   - `<<<<<<<` 当前分支的内容
   - `=======` 分隔线
   - `>>>>>>>` 要合并分支的内容
2. 添加解决后的文件：`git add <文件>`
3. 继续合并：`git commit` 或 `git merge --continue`

## 变基

### 命令

```bash
git rebase <分支名字>       # 将当前分支变基到指定分支
git rebase -i HEAD~<n>      # 交互式变基，基于前 n 次提交
git rebase -i <哈希值>       # 交互式变基，基于指定哈希值的提交
git rebase --continue       # 解决冲突后继续变基
git rebase --abort          # 放弃变基，恢复到变基前的状态
```

### 交互式变基用法

**修改历史提交信息：**
1. 将要修改提交信息的每个提交的 `pick` 更改为 `r`（reword），保存退出
2. 逐个修改提交信息，保存退出

**将多个历史提交合并为一个：**
1. 将要合并的每个提交的 `pick` 更改为 `s`（squash），保存退出
2. 修改提交信息，保存退出

### 说明

- 变基会重写提交历史，**不要对已推送到远程的提交进行变基**
- 当前分支的变基操作不会影响其他分支

## Cherry-pick

选择性地将某些提交应用到当前分支。

### 命令

```bash
git cherry-pick <commit>                     # 将指定的某个提交应用到当前分支
git cherry-pick <commit1> <commit2>          # 依次应用多个指定的提交
git cherry-pick <commit1>..<commit2>         # 应用从 commit1（不含）到 commit2（含）的提交
git cherry-pick <commit1>^..<commit2>        # 应用从 commit1（含）到 commit2（含）的提交
git cherry-pick -n <commit>                  # 不自动提交，而是把修改内容放入暂存区
git cherry-pick --skip                       # 冲突时跳过当前 cherry-pick 的提交
git cherry-pick --continue                   # 解决冲突后继续 cherry-pick 过程
git cherry-pick --abort                      # 放弃 cherry-pick 操作，恢复到操作前的状态
```

## 误删分支恢复

```bash
git reflog                           # 查看操作记录，找到删除前的 commit
git checkout -b <分支名> <哈希值>     # 从该 commit 创建新分支
```
