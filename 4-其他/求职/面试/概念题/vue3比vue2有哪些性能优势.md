# Vue3 相比 Vue2 的主要性能提升点

### 1. 响应式系统从 `Object.defineProperty` 升级为 `Proxy`
- **Vue2** 对每个属性做 `defineProperty` 拦截，初始化和深层对象开销大（尤其是大对象、深层嵌套）。
- **Vue3** 用 `Proxy`，对整个对象拦截：
  - 不需要在初始化阶段遍历所有属性，创建成本更低。
  - 访问到的子属性才会被「追踪」，懒跟踪，减少不必要的依赖收集。
  - 对象动态新增/删除属性也能被响应式系统正确捕获（不用再 `Vue.set`）。

### 2. 编译器优化：静态提升、Patch Flag、Block Tree
- **静态提升（hoistStatic）**：模板里不依赖响应式数据的节点只创建一次，后续更新直接复用，减少虚拟 DOM 创建与 diff。
- **Patch Flag**：编译阶段给动态节点打标记，运行时只对“标过记”的地方做 diff，避免整棵子树遍历。
- **Block Tree**：将更新范围精确缩小到真正可能变化的「区块」，减少无效比较，更新更快。

### 3. 更好的 Tree-shaking，体积更小
- Vue3 内部按功能拆成更多独立模块（尤其是 Composition API 风格）。
- 构建工具（Webpack/Rollup/Vite）可以更好地进行 tree-shaking，只打包用到的 API，最终产物体积更小，加载更快。

### 4. 虚拟 DOM 重写与多根 / Fragment 支持
- Vue3 重新实现了虚拟 DOM 实现，更精简、对 Patch Flag 友好。
- 支持 Fragment，减少不必要的包裹标签，DOM 结构更扁平，渲染、更新开销略低。

### 5. 更高效的组件初始化和销毁
- 组件实例结构更简单（基于 `setup`），减少初始化时的一些冗余逻辑。
- Effect / 副作用跟踪与清理机制更明确，组件销毁时对 watcher、依赖的清理更高效，内存占用更可控。

### 6. SSR 和 Hydration 优化
- Vue3 的 SSR 流程和运行时打通得更好，支持更高效的同构 hydration。
- 编译器针对 SSR 也做了优化，服务端渲染 + 客户端激活整体速度和体验优于 Vue2。
