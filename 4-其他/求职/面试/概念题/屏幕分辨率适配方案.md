# 前端屏幕分辨率适配方案详解

## 一、基础概念（必须先理解）

### 1. 物理像素（Physical Pixel）
- 设备屏幕实际拥有的像素点
- 屏幕出厂就固定了，无法改变
- 例：iPhone 14 Pro 的物理分辨率是 2556 × 1179

### 2. 逻辑像素 / CSS 像素（CSS Pixel）
- 前端开发中使用的像素单位（`px`）
- 是一个抽象单位，与物理像素不一定 1:1 对应
- 例：iPhone 14 Pro 的逻辑分辨率是 852 × 393

### 3. 设备像素比 DPR（Device Pixel Ratio）
`DPR = 物理像素 / 逻辑像素`

| 设备 | 物理分辨率 | 逻辑分辨率 | DPR |
| :--- | :--- | :--- | :--- |
| iPhone SE | 1334 × 750 | 667 × 375 | 2 |
| iPhone 14 Pro | 2556 × 1179 | 852 × 393 | 3 |
| 普通 PC 显示器 | 1920 × 1080 | 1920 × 1080 | 1 |
| 4K 显示器（缩放 200%） | 3840 × 2160 | 1920 × 1080 | 2 |

获取方式：
```javascript
window.devicePixelRatio // 返回当前设备的 DPR
```

### 4. 视口（Viewport）
- **布局视口（Layout Viewport）**：网页的实际布局宽度，移动端默认通常是 980px（为了兼容 PC 网页）
- **视觉视口（Visual Viewport）**：用户当前看到的区域，可通过缩放改变
- **理想视口（Ideal Viewport）**：设备屏幕的逻辑宽度，通过 `<meta name="viewport">` 设置

```html
<!-- 让布局视口 = 理想视口（移动端必加） -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

| 属性 | 说明 |
| :--- | :--- |
| `width=device-width` | 布局视口宽度 = 设备宽度 |
| `initial-scale=1.0` | 初始缩放比例 |
| `maximum-scale=1.0` | 最大缩放比例 |
| `minimum-scale=1.0` | 最小缩放比例 |
| `user-scalable=no` | 禁止用户缩放 |

### 5. 设计稿尺寸
| 场景 | 常用设计稿宽度 |
| :--- | :--- |
| 移动端 H5 | 375px（iPhone 6/7/8）或 750px（2 倍图） |
| PC 端 | 1440px / 1920px |
| 大屏可视化 | 1920 × 1080 |

## 二、适配方案详解

### 方案一：百分比布局（Percentage）

**原理**
- 使用 `%` 作为单位，相对于父元素计算
- 宽度百分比相对于父元素宽度
- 高度百分比相对于父元素高度
- `padding` / `margin` 百分比相对于父元素宽度

**示例**
```css
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
}

.sidebar {
  width: 25%;
  float: left;
}

.main {
  width: 75%;
  float: left;
}
```

**优点**
- 简单直观
- 兼容性好

**缺点**
- 高度难以用百分比控制
- 计算复杂，难以精确还原设计稿
- `padding` / `margin` 参照物不统一（都是父元素宽度）
- 字体大小无法用百分比适配

**适用场景**
- 简单的两栏/三栏布局
- 流式布局基础

### 方案二：媒体查询（Media Query）

**原理**
- 根据屏幕宽度设置不同的样式
- 常用于响应式布局，在特定断点切换布局

**常用断点**
```css
/* 超小屏幕（手机） */
@media (max-width: 575.98px) { }

/* 小屏幕（平板竖屏） */
@media (min-width: 576px) and (max-width: 767.98px) { }

/* 中等屏幕（平板横屏） */
@media (min-width: 768px) and (max-width: 991.98px) { }

/* 大屏幕（桌面） */
@media (min-width: 992px) and (max-width: 1199.98px) { }

/* 超大屏幕（大桌面） */
@media (min-width: 1200px) and (max-width: 1399.98px) { }

/* 超超大屏幕 */
@media (min-width: 1400px) { }
```

**Bootstrap 5 断点**
| 断点 | 类名前缀 | 尺寸 |
| :--- | :--- | :--- |
| X-Small | 无 | < 576px |
| Small | `sm` | ≥ 576px |
| Medium | `md` | ≥ 768px |
| Large | `lg` | ≥ 992px |
| X-Large | `xl` | ≥ 1200px |
| XX-Large | `xxl` | ≥ 1400px |

**完整示例**
```css
/* 移动优先：默认样式 */
.container {
  width: 100%;
  padding: 16px;
}

.nav {
  display: none; /* 移动端隐藏导航 */
}

.nav-mobile {
  display: block;
}

/* 平板及以上 */
@media (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }

  .nav {
    display: flex;
  }

  .nav-mobile {
    display: none;
  }
}

/* 桌面端 */
@media (min-width: 992px) {
  .container {
    width: 970px;
  }
}

/* 大桌面 */
@media (min-width: 1200px) {
  .container {
    width: 1170px;
  }
}
```

**优点**
- 可以针对不同屏幕做完全不同的布局
- 兼容性好
- 配合 Flex/Grid 效果更佳

**缺点**
- 需要写多套样式，工作量大
- 只能在断点处"跳变"，不是平滑过渡
- 维护成本高

**适用场景**
- PC + 移动端响应式网站
- 需要在不同设备上有完全不同布局的场景

### 方案三：rem 适配

**原理**
- `rem`（root em）：相对于根元素（`<html>`）的 `font-size`
- 动态设置 `html` 的 `font-size`，所有用 `rem` 的元素就会跟着缩放
- 实现等比缩放效果

**核心公式**
```
设计稿元素尺寸（px） / 基准值 = 实际书写（rem）
```
例：设计稿 375px，基准值 100（即 1rem = 100px），设计稿上 200px 的元素 → `200 / 100 = 2rem`

**实现步骤**
1. 动态设置根字体
```javascript
// flexible.js（简化版）
(function(doc, win) {
  const docEl = doc.documentElement
  const dpr = win.devicePixelRatio || 1

  function setRemUnit() {
    // 设计稿宽度 375，分成 10 份，1rem = 37.5px
    const rem = docEl.clientWidth / 10
    docEl.style.fontSize = rem + 'px'
  }

  setRemUnit()

  // 监听窗口变化
  win.addEventListener('resize', setRemUnit)
  win.addEventListener('pageshow', function(e) {
    if (e.persisted) {
      setRemUnit()
    }
  })
})(document, window)
```

2. PostCSS 自动转换（推荐）
```bash
npm install postcss-pxtorem -D
```
配置 `postcss.config.js`：
```javascript
module.exports = {
  plugins: {
    autoprefixer: {},
    'postcss-pxtorem': {
      rootValue: 37.5,    // 设计稿宽度 375 / 10
      propList: ['*'],    // 所有属性都转换
      unitPrecision: 5,   // 精度
      minPixelValue: 1,   // 小于 1px 不转换
      selectorBlackList: ['.no-rem'], // 忽略的选择器
      exclude: /node_modules/i,
    },
  },
}
```

3. 写样式时直接用 px
```css
/* 按设计稿写，自动转成 rem */
.header {
  height: 88px;    /* 编译后：2.34667rem */
  font-size: 32px; /* 编译后：0.85333rem */
  padding: 24px;   /* 编译后：0.64rem */
}

/* 不想转换的加类名 */
.no-rem .border {
  border: 1px solid #ccc; /* 保持 1px */
}
```

**lib-flexible 方案（淘宝方案）**
```javascript
// 核心逻辑
(function flexible(window, document) {
  const docEl = document.documentElement
  const dpr = window.devicePixelRatio || 1

  // 设置 body 字体大小
  function setBodyFontSize() {
    if (document.body) {
      document.body.style.fontSize = (12 * dpr) + 'px'
    } else {
      document.addEventListener('DOMContentLoaded', setBodyFontSize)
    }
  }
  setBodyFontSize()

  // 设置 1rem = viewWidth / 10
  function setRemUnit() {
    const rem = docEl.clientWidth / 10
    docEl.style.fontSize = rem + 'px'
  }
  setRemUnit()

  // 监听 resize
  window.addEventListener('resize', setRemUnit)
  window.addEventListener('pageshow', function(e) {
    if (e.persisted) {
      setRemUnit()
    }
  })

  // 检测 0.5px 支持
  if (dpr >= 2) {
    const fakeBody = document.createElement('body')
    const testElement = document.createElement('div')
    testElement.style.border = '.5px solid transparent'
    fakeBody.appendChild(testElement)
    docEl.appendChild(fakeBody)
    if (testElement.offsetHeight === 1) {
      docEl.classList.add('hairlines')
    }
    docEl.removeChild(fakeBody)
  }
}(window, document))
```

**计算方式对照表**
| 设计稿宽度 | 划分份数 | rootValue | 1rem |
| :--- | :--- | :--- | :--- |
| 375px | 10 | 37.5 | 37.5px |
| 750px | 10 | 75 | 75px |
| 375px | 100 | 3.75 | 3.75px |

**优点**
- 等比缩放，适配效果好
- 开发时直接写 px，PostCSS 自动转换
- 成熟方案，业界广泛使用

**缺点**
- 需要引入 JS 计算根字体
- 在超大/超小屏幕可能需要限制缩放范围
- 字体大小可能会过大/过小，需要特殊处理
- iframe 内无法继承根字体

**适用场景**
- 移动端 H5 页面
- 需要等比缩放的活动页

### 方案四：vw/vh 适配

**原理**
- `vw`：视口宽度的 1%（100vw = 视口宽度）
- `vh`：视口高度的 1%（100vh = 视口高度）
- `vmin`：`vw` 和 `vh` 中较小的
- `vmax`：`vw` 和 `vh` 中较大的

**核心公式**
```
设计稿元素尺寸（px） / 设计稿宽度（px） × 100 = vw 值
```
例：设计稿 375px，元素宽 200px → `200 / 375 × 100 = 53.33vw`

**实现步骤**
1. PostCSS 自动转换
```bash
npm install postcss-px-to-viewport -D
```
配置 `postcss.config.js`：
```javascript
module.exports = {
  plugins: {
    autoprefixer: {},
    'postcss-px-to-viewport': {
      unitToConvert: 'px',    // 要转换的单位
      viewportWidth: 375,     // 设计稿宽度
      unitPrecision: 5,       // 精度
      propList: ['*'],        // 要转换的属性
      viewportUnit: 'vw',     // 转换成的单位
      fontViewportUnit: 'vw', // 字体使用的单位
      selectorBlackList: ['.ignore', '.hairlines'], // 忽略的类名
      minPixelValue: 1,       // 小于等于 1px 不转换
      mediaQuery: false,      // 是否转换媒体查询中的 px
      replace: true,          // 是否直接替换
      exclude: [/node_modules/], // 忽略的文件
      landscape: false,       // 是否处理横屏
      landscapeUnit: 'vw',    // 横屏单位
      landscapeWidth: 667,    // 横屏宽度
    },
  },
}
```

2. 升级版：`postcss-px-to-viewport-8-plugin`（原插件不维护了，推荐使用）
```bash
npm install postcss-px-to-viewport-8-plugin -D
```

**手动计算示例**
```css
/* 设计稿 375px */
.header {
  /* 88px / 375 * 100 = 23.4667vw */
  height: 23.4667vw;

  /* 32px / 375 * 100 = 8.5333vw */
  font-size: 8.5333vw;
}
```

**配合 SCSS 函数**
```scss
// 设计稿宽度
$design-width: 375;

// px 转 vw 函数
@function vw($px) {
  @return ($px / $design-width) * 100vw;
}

// 使用
.header {
  height: vw(88);
  font-size: vw(32);
  padding: vw(24);
}
```

**优点**
- 纯 CSS 方案，不依赖 JS
- 天然支持等比缩放
- 兼容性好（iOS 8+，Android 4.4+）
- 配合 PostCSS 开发体验好

**缺点**
- 在超大/超小屏幕会无限放大/缩小
- `vh` 在移动端有坑（浏览器地址栏影响）
- 部分浏览器对 `vw` 计算有精度问题

**解决超大/超小屏问题**
```css
/* 使用 clamp 限制范围 */
.title {
  /* 最小 14px，理想 4.267vw，最大 24px */
  font-size: clamp(14px, 4.267vw, 24px);
}

/* 或者用媒体查询限制 */
@media (min-width: 750px) {
  html {
    font-size: 100px; /* 固定不再放大 */
  }
}
```

**适用场景**
- 移动端 H5（主流推荐）
- 不想依赖 JS 的项目

### 方案五：rem + vw 混合方案

**原理**
- 用 `vw` 设置根字体大小
- 页面元素使用 `rem`
- 结合两者优点

**实现**
```css
/* 设计稿 750px，1rem = 100px */
/* 100 / 750 * 100 = 13.333vw */
html {
  font-size: 13.333vw;
}

/* 限制最大最小值 */
@media (max-width: 320px) {
  html {
    font-size: 42.667px; /* 320 * 13.333% */
  }
}

@media (min-width: 750px) {
  html {
    font-size: 100px;
  }
}
```

**更优雅的写法（使用 clamp）**
```css
html {
  /* 最小 42.667px（320px 屏），理想 13.333vw，最大 100px（750px 屏） */
  font-size: clamp(42.667px, 13.333vw, 100px);
}
```

**优点**
- 不需要 JS
- 可以用 PostCSS 转换 `rem`
- 可以限制缩放范围

### 方案六：Flex + 百分比 + 媒体查询（PC 端推荐）

**原理**
- 使用 Flexbox 做弹性布局
- 配合媒体查询做断点切换
- 不追求"等比缩放"

**示例**
```css
/* 容器限制宽度 */
.container {
  max-width: 1200px;
  min-width: 1024px;
  margin: 0 auto;
  padding: 0 24px;
}

/* Flex 布局 */
.layout {
  display: flex;
  gap: 24px;
}

.sidebar {
  width: 240px;
  flex-shrink: 0;
}

.main {
  flex: 1;
  min-width: 0; /* 防止内容撑开 */
}

/* 小屏幕适配 */
@media (max-width: 1024px) {
  .layout {
    flex-direction: column;
  }

  .sidebar {
    width: 100%;
  }
}
```

**适用场景**
- PC 端后台管理系统
- 企业官网

### 方案七：Scale 等比缩放（大屏可视化）

**原理**
- 以设计稿尺寸（如 1920 × 1080）为基准
- 计算当前屏幕与设计稿的缩放比
- 使用 `transform: scale()` 缩放整个容器

**实现**
```html
<div id="screen-container">
  <!-- 大屏内容 -->
</div>
```
```css
#screen-container {
  position: fixed;
  left: 50%;
  top: 50%;
  width: 1920px;
  height: 1080px;
  transform-origin: left top;
  overflow: hidden;
}
```
```javascript
const DESIGN_WIDTH = 1920
const DESIGN_HEIGHT = 1080

function setScale() {
  const container = document.getElementById('screen-container')
  const { clientWidth, clientHeight } = document.documentElement

  // 方式一：宽度优先（可能上下留白）
  const scaleX = clientWidth / DESIGN_WIDTH
  container.style.transform = `scale(${scaleX}) translate(-50%, -50%)`

  // 方式二：高度优先（可能左右留白）
  // const scaleY = clientHeight / DESIGN_HEIGHT
  // container.style.transform = `scale(${scaleY}) translate(-50%, -50%)`

  // 方式三：完全填充（可能变形）
  // container.style.transform = `scale(${scaleX}, ${scaleY}) translate(-50%, -50%)`

  // 方式四：保持比例居中（推荐，可能留白）
  // const scale = Math.min(scaleX, scaleY)
  // container.style.transform = `scale(${scale}) translate(-50%, -50%)`
}

window.addEventListener('resize', setScale)
setScale()
```

**几种缩放模式对比**
| 模式 | 代码 | 效果 |
| :--- | :--- | :--- |
| 宽度铺满 | `scale(scaleX)` | 宽度撑满，高度可能超出或不足 |
| 高度铺满 | `scale(scaleY)` | 高度撑满，宽度可能超出或不足 |
| 保持比例 | `scale(min(scaleX, scaleY))` | 完整显示，可能留白 |
| 完全铺满 | `scale(scaleX, scaleY)` | 无留白，可能变形 |

**封装成 Vue 组件**
```vue
<template>
  <div ref="containerRef" class="scale-screen">
    <slot></slot>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted, onUnmounted } from 'vue'

interface Props {
  width?: number
  height?: number
  mode?: 'fit' | 'fill' | 'width' | 'height'
}

const props = withDefaults(defineProps<Props>(), {
  width: 1920,
  height: 1080,
  mode: 'fit',
})

const containerRef = ref<HTMLElement>()

function resize() {
  if (!containerRef.value) return

  const { clientWidth, clientHeight } = document.documentElement
  const scaleX = clientWidth / props.width
  const scaleY = clientHeight / props.height

  let scale: string
  switch (props.mode) {
    case 'fit':
      scale = `scale(${Math.min(scaleX, scaleY)})`
      break
    case 'fill':
      scale = `scale(${scaleX}, ${scaleY})`
      break
    case 'width':
      scale = `scale(${scaleX})`
      break
    case 'height':
      scale = `scale(${scaleY})`
      break
  }

  containerRef.value.style.transform = `${scale} translate(-50%, -50%)`
}

onMounted(() => {
  resize()
  window.addEventListener('resize', resize)
})

onUnmounted(() => {
  window.removeEventListener('resize', resize)
})
</script>

<style scoped>
.scale-screen {
  position: fixed;
  left: 50%;
  top: 50%;
  width: v-bind('props.width + "px"');
  height: v-bind('props.height + "px"');
  transform-origin: left top;
  overflow: hidden;
}
</style>
```

**优点**
- 完美还原设计稿比例
- 开发时直接按设计稿尺寸写 px
- 适配简单暴力

**缺点**
- 文字可能模糊（缩放导致）
- echarts、地图等组件事件坐标可能错位
- 弹窗、下拉框等定位可能出问题
- 不适合有大量交互/表单的场景

**适用场景**
- 大屏数据可视化展示
- 固定尺寸的展示型页面

### 方案八：clamp() 函数（现代方案）

**原理**
- `clamp(min, preferred, max)`：限制值在 min 和 max 之间
- 结合 `vw` 实现有边界的流体布局

**示例**
```css
.container {
  /* 宽度：最小 320px，理想 90vw，最大 1200px */
  width: clamp(320px, 90vw, 1200px);
}

.title {
  /* 字号：最小 16px，理想 5vw，最大 48px */
  font-size: clamp(16px, 5vw, 48px);
}

.section {
  /* 间距：最小 16px，理想 4vw，最大 64px */
  padding: clamp(16px, 4vw, 64px);
}
```

**计算公式**
```
preferred = (maxSize - minSize) / (maxWidth - minWidth) * 100vw + 偏移量
```
例：320px~1200px 屏幕，字号 16px~48px
- 斜率 = `(48 - 16) / (1200 - 320) = 0.03636`
- preferred ≈ `3.636vw + 调整值`

**优点**
- 纯 CSS，无需 JS
- 天然有最大最小限制
- 流体过渡，观感好

**缺点**
- 兼容性要求较高（IE 不支持）
- 计算较复杂

## 三、各方案对比总结

| 方案 | 原理 | 优点 | 缺点 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| 百分比 | 相对父元素 | 简单 | 不够精确 | 简单流式布局 |
| 媒体查询 | 断点切换样式 | 可完全不同布局 | 工作量大 | 响应式网站 |
| rem | 相对根字体 | 等比缩放、成熟 | 需要 JS | 移动端 H5 |
| vw/vh | 相对视口 | 纯 CSS、等比缩放 | 无限缩放问题 | 移动端 H5（推荐） |
| rem + vw | 混合 | 不需要 JS | 配置略复杂 | 移动端 H5 |
| Flex 响应式 | 弹性布局 | 灵活、现代 | 不等比缩放 | PC 端 |
| Scale 缩放 | transform 缩放 | 完美还原设计稿 | 交互/清晰度问题 | 大屏可视化 |
| clamp() | 限制范围 | 流体、有边界 | 兼容性 | 现代项目 |

## 四、移动端 1px 边框问题

**问题原因**
- 设计稿是 2 倍/3 倍图
- CSS 的 `1px` 在 DPR=2 的屏幕上实际显示 2 个物理像素
- 看起来比设计稿"粗"

**解决方案**

**方案一：0.5px（有兼容问题）**
```css
.border {
  border: 0.5px solid #ccc;
}
```

**方案二：transform 缩放（推荐）**
```css
.border-1px {
  position: relative;
  /* 配合 rem 方案，整体缩小再放大。 */
}

.border-1px::after {
  content: '';
  position: absolute;
  left: 0;
  bottom: 0;
  width: 100%;
  height: 1px;
  background: #ccc;
  transform: scaleY(0.5);
  transform-origin: 0 0;
}

/* DPR = 3 */
@media (-webkit-min-device-pixel-ratio: 3) {
  .border-1px::after {
    transform: scaleY(0.333);
  }
}
```

**方案三：viewport 缩放**
```html
<!-- DPR = 2 -->
<meta name="viewport" content="width=device-width, initial-scale=0.5, maximum-scale=0.5">
```

**方案四：SVG**
```css
.border {
  border: 1px solid transparent;
  border-image: url("data:image/svg+xml,...") 1;
}
```

## 五、安全区域适配（刘海屏）

**问题**
- iPhone X 及以后的刘海屏
- 底部有 Home Indicator 区域
- 内容可能被遮挡

**解决方案**
```html
<!-- viewport 开启安全区域 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
```

```css
/* 安全区域变量 */
.bottom-bar {
  padding-bottom: constant(safe-area-bottom); /* iOS < 11.2 */
  padding-bottom: env(safe-area-bottom);      /* iOS >= 11.2 */
}

/* 完整的安全区域变量 */
padding-top: env(safe-area-inset-top);
padding-right: env(safe-area-inset-right);
padding-bottom: env(safe-area-inset-bottom);
padding-left: env(safe-area-inset-left);
```

## 六、面试高频考点

| 考点 | 关键答案 |
| :--- | :--- |
| **rem 原理？** | 相对根元素 font-size，JS 动态设置根字体实现等比缩放 |
| **vw 原理？** | 1vw = 视口宽度的 1%，纯 CSS 方案 |
| **rem vs vw？** | rem 需要 JS；vw 纯 CSS 但无限缩放；推荐 vw + clamp 限制 |
| **DPR 是什么？** | 物理像素 / 逻辑像素，用于适配 Retina 屏 |
| **1px 问题？** | DPR > 1 时 1px 显示多个物理像素，用 transform: scale 解决 |
| **大屏适配？** | scale 等比缩放，注意交互坐标和清晰度问题 |
| **PostCSS 配置？** | postcss-pxtorem（rem）或 postcss-px-to-viewport（vw） |
| **安全区域？** | viewport-fit=cover + env(safe-area-inset-*) |
| **移动端 viewport？**| width=device-width, initial-scale=1.0 |
| **clamp() 作用？** | 限制值的范围，实现有边界的流体布局 |
