# 浏览器的本地存储

浏览器的本地存储是指在用户浏览器端保存数据的机制，主要有以下几种方式：

## 1. Cookie
- **容量**：约 4KB
- **生命周期**：可设置过期时间，不设置则为会话级别（关闭浏览器即失效）
- **特点**：
  - 每次 HTTP 请求都会自动携带发送到服务器
  - 可设置 `HttpOnly`、`Secure`、`SameSite` 等安全属性
  - 适合存储需要服务端读取的小数据（如身份验证 token）

## 2. localStorage
- **容量**：约 5-10MB
- **生命周期**：永久存储，除非手动清除
- **特点**：
  - 同源策略限制（同协议、域名、端口）
  - 仅在客户端存储，不会随请求发送
  - 适合存储长期数据（如用户偏好设置）

**用法示例：**
```javascript
// 存储
localStorage.setItem('key', 'value')
// 读取
localStorage.getItem('key')
// 删除
localStorage.removeItem('key')
// 清空
localStorage.clear()
```

## 3. sessionStorage
- **容量**：约 5-10MB
- **生命周期**：会话级别，关闭标签页/浏览器即失效
- **特点**：
  - API 与 localStorage 完全相同
  - 同一页面在不同标签页中 sessionStorage 不共享
  - 适合存储临时会话数据

## 4. IndexedDB
- **容量**：取决于磁盘空间，通常可达数百MB甚至更多
- **生命周期**：永久存储
- **特点**：
  - 完整的数据库系统，支持事务、索引、游标
  - 异步 API，不会阻塞主线程
  - 可存储结构化数据（包括文件/Blob）
  - 适合存储大量数据或离线应用

## 对比总结

| 特性 | Cookie | localStorage | sessionStorage | IndexedDB |
| :--- | :--- | :--- | :--- | :--- |
| **容量** | ~4KB | ~5-10MB | ~5-10MB | 无限制 |
| **过期** | 可设置 | 永久 | 会话 | 永久 |
| **随请求发送** | ✅ | ❌ | ❌ | ❌ |
| **API** | 字符串操作 | 简单同步 | 简单同步 | 复杂异步 |
| **跨标签共享** | ✅ | ✅ | ❌ | ✅ |
