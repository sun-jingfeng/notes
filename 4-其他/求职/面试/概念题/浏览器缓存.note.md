# 浏览器缓存的概念

浏览器缓存是一种将网页资源（如 HTML、CSS、JavaScript、图片等）存储在本地的机制，目的是在用户再次访问同一网站时，减少网络请求，加快页面加载速度。

## 🗂 缓存的分类

### 1. 强缓存（Strong Cache）
浏览器直接从本地缓存读取资源，不会向服务器发送请求。

| 响应头 | 说明 |
| :--- | :--- |
| `Expires` | HTTP/1.0 产物，设置过期的绝对时间（容易受本地时间影响） |
| `Cache-Control` | HTTP/1.1 产物，优先级更高，常用值：<br>• `max-age=3600`：缓存 3600 秒<br>• `no-cache`：需要协商缓存验证<br>• `no-store`：不缓存 |

### 2. 协商缓存（Negotiation Cache）
浏览器向服务器发送请求，由服务器判断资源是否更新。

| 请求头 / 响应头 | 说明 |
| :--- | :--- |
| `Last-Modified` / `If-Modified-Since` | 基于资源**修改时间**判断（精度只到秒） |
| `ETag` / `If-None-Match` | 基于资源**内容哈希**判断（更精确，优先级更高） |

- 若资源未变化，返回 `304 Not Modified`，使用本地缓存
- 若资源已变化，返回 `200 OK` 和新资源

## 🔄 缓存流程图

```text
请求资源
   ↓
是否有缓存？ ─── 否 ──→ 请求服务器 → 返回资源并缓存
   ↓ 是
强缓存是否过期？ ─── 否 ──→ 直接使用缓存（200 from cache）
   ↓ 是
发送请求（携带 ETag/Last-Modified）
   ↓
服务器验证资源是否变化？
   ↓ 否                ↓ 是
返回 304，使用缓存    返回 200 + 新资源
```

## 💾 缓存位置

| 位置 | 说明 |
| :--- | :--- |
| **Memory Cache** | 存在内存中，速度快，关闭标签页即失效 |
| **Disk Cache** | 存在硬盘中，容量大，持久保存 |
| **Service Worker** | 可编程控制缓存，支持离线访问（PWA） |

## 🎯 实际应用建议

1. **静态资源（JS/CSS/图片）**：使用强缓存 + 文件名哈希
   - `Cache-Control: max-age=31536000`
   - 文件内容变化时，文件名也会变化（如 `app.abc123.js`）
2. **HTML 文件**：使用协商缓存或不缓存
   - `Cache-Control: no-cache`
   - 确保用户总能获取最新的入口文件
3. **API 请求**：通常不缓存或短时间缓存
   - `Cache-Control: no-store`

## 个人补充

- 响应头中的 `Etag` 会被后续请求的请求头的 `If-None-Match` 携带，用于验证资源是否改变。
- 响应头中的 `Last-Modified` 会被后续请求的请求头的 `If-Modified-Since` 携带，用于验证资源是否改变。
- 客户端请求头影响本次请求行为，服务器响应头决定后续缓存策略。
- 响应头的 `Cache-Control` 控制的是后续所有请求的行为，直到有新的响应头的 `Cache-Control` 值覆盖之前的。
