# == 和 === 的区别

## 一、核心区别

| 运算符 | 名称 | 比较规则 |
| --- | --- | --- |
| **`==`** | 宽松相等 / 抽象相等 | 先做**隐式类型转换**，再比较值 |
| **`===`** | 严格相等 | **不转换类型**，类型不同直接返回 `false`；类型相同再比较值 |

**结论**：`===` 行为可预期、无隐式转换，实际开发应默认使用 `===`；ESLint 的 `eqeqeq` 规则可强制禁止 `==`。

***

## 二、隐式类型转换规则（== 时）

### 1. 特殊规则：null 与 undefined

```js
null == undefined  // true（规范规定）
null === undefined  // false
```

`null`、`undefined` 与除彼此外的任何值用 `==` 比较都为 `false`。

### 2. 数字与字符串

比较时**字符串会转为数字**；若一方是数字一方是字符串，按数字比较。

```js
1 == '1'    // true  （'1' → 1）
1 == '1.0'  // true
'' == 0     // true  （'' → 0）
'  \n  ' == 0  // true
```

### 3. 布尔与其它类型

**布尔先转为数字**：`true` → 1，`false` → 0，再按上面规则比较。

```js
true == 1   // true
false == 0  // true
true == '1' // true  （true→1, '1'→1）
false == '' // true  （false→0, ''→0）
```

### 4. 对象与基本类型

对象会先尝试 **ToPrimitive**：优先 `valueOf()`，再 `toString()`，得到基本类型后再按上面规则比较。

```js
const obj = { valueOf() { return 1 } }
obj == 1  // true
obj == '1' // true

[1] == 1   // true  （[1].toString() → '1'，'1' → 1）
[] == 0    // true  （[].toString() → ''，'' → 0）
```

### 5. 两边均为对象

**不做转换**，只比较是否同一引用。

```js
[] == []   // false
{} == {}   // false
```

## 三、常见“坑”与正确写法

| 写法 | 结果 | 建议 |
| --- | --- | --- |
| `x == null` | 同时匹配 `null` 和 `undefined` | 若需同时判断两者，可保留 |
| `x == 0` | `''`、`false`、`[]`、`'0'` 都会为 true | 若只要“数值为 0”应用 `x === 0` |
| `if (value)` | 会排除 0、''、false、null、undefined | 根据业务区分“假值”与“未设置” |

```js
// ✅ 明确意图时用 ===
if (count === 0) { /* 数量为 0 */ }
if (value === null || value === undefined) { /* 未设置 */ }

// ❌ 易混淆
if (value == 0) { /* value 可能是 0、''、false 等 */ }
```

***

## 四、面试答题要点

- **==**：会隐式类型转换；`null == undefined` 为 true；数字/字符串/布尔之间会转数字再比；对象通过 `valueOf`/`toString` 转成基本类型再比。
- **===**：类型不同必为 false，类型相同再比值，无转换。
- **实践**：一律用 `===`，仅在需要同时判断 `null`/`undefined` 时可用 `x == null`。
