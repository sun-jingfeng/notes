# 箭头函数与普通函数的区别

## 一、核心区别概览

| 区别项 | **普通函数** | **箭头函数** |
| --- | --- | --- |
| **this** | 由调用方式决定（默认、隐式、显式、new） | 由定义时外层作用域决定，且不可改 |
| **arguments** | 有 `arguments` 对象 | 无，需用 `...rest` |
| **作为构造函数** | 可用 `new` 调用 | 不可用 `new`（无 [[Construct]]） |
| **prototype** | 有 `prototype` | 无 `prototype` |
| **yield** | 可作为 generator 函数 | 不能作为 generator（无 `function*` 形式） |

***

## 二、this 指向（最重要）

### 1. 普通函数的 this

由**调用时**的调用方式决定：谁调用、是否 `new`、是否 `call/apply/bind`。

```js
function fn() { console.log(this) }

fn()                    // 非严格模式：window / global；严格模式：undefined
obj.fn()                // obj
new fn()                // 新创建的对象
fn.call({ id: 1 })     // { id: 1 }
```

### 2. 箭头函数的 this

**在定义时**就确定了：继承**外层（词法）作用域的 this**，且之后无法通过 `call`、`apply`、`bind` 或作为方法调用改变。

```js
const obj = {
  name: 'obj',
  normal: function() { console.log(this.name) },       // 普通函数
  arrow: () => console.log(this.name)                   // 箭头函数
}
obj.normal()  // 'obj'
obj.arrow()   // 外层若是全局，this 为 window，可能为 undefined（严格）
```

典型用法：在需要保留“外部 this”的回调里用箭头函数。

```js
class Demo {
  value = 1
  run() {
    setTimeout(() => {
      console.log(this.value)  // 箭头函数，this 固定为 Demo 实例
    }, 0)
  }
}
```

## 三、arguments 与剩余参数

普通函数内部有 **arguments**（类数组，包含所有实参）；箭头函数没有。

```js
function normal(a, b) {
  console.log(arguments)  // Arguments(2) [1, 2, ...]
}
normal(1, 2)

const arrow = (a, b) => {
  console.log(arguments)  // ReferenceError 或拿到外层的 arguments
}
arrow(1, 2)

// ✅ 箭头函数用剩余参数
const arrow2 = (...rest) => console.log(rest)  // [1, 2, 3]
arrow2(1, 2, 3)
```

***

## 四、不能作为构造函数

箭头函数没有 `[[Construct]]`，没有 `prototype`，因此不能 `new`。

```js
const F = () => {}
new F()  // TypeError: F is not a constructor

const G = function() {}
console.log(G.prototype)  // 有 prototype
console.log(F.prototype)  // undefined
```

***

## 五、何时用箭头、何时用普通函数

| 场景 | 建议 |
| --- | --- |
| 需要动态 this（对象方法、回调里要 this 指向调用者） | 普通函数 |
| 需要 `arguments` | 普通函数，或箭头内用 `...rest` |
| 需要 `new`、generator | 普通函数 |
| 需要固定 this 为外层（如 React 类中、setTimeout 回调） | 箭头函数 |
| 纯工具函数、无 this/arguments 依赖 | 两者皆可，箭头更简洁 |

> **注意**：对象字面量里的“方法”若用箭头函数，`this` 不会指向该对象，而是外层；Vue 选项里 `methods` 中一般用普通函数以保证 `this` 为组件实例。

***

## 六、面试答题要点

- **this**：普通函数看调用方式；箭头函数看定义时外层 this，且不可被改。
- **arguments**：普通函数有；箭头函数无，用 `...rest`。
- **new / prototype**：只有普通函数可 `new` 且有 `prototype`。
- **使用**：需要“固定外层 this”或简短回调用箭头；需要 this 指向调用者、构造函数、arguments 时用普通函数。
