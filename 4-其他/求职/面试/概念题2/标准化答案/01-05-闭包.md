# 闭包

## 一、什么是闭包

**闭包（Closure）**：函数与其**词法作用域**的组合。当函数能够访问并“记住”其定义时所处作用域中的变量，且在该作用域之外被使用，就形成了闭包。

更直白地说：**闭包 = 函数 + 它能访问的外层变量**。通常表现为“函数被返回或传到外部，仍持有对外层变量的引用”。

```js
function createCounter() {
  let count = 0
  return function () {
    count += 1
    return count
  }
}
const counter = createCounter()
counter()  // 1
counter()  // 2
// createCounter 已执行完，但返回的函数仍能访问并修改 count
```

***

## 二、闭包如何产生

1. **函数被 return 出去**：外部拿到的是内部函数，该函数引用外层变量。
2. **函数被当作参数传递**：如传给 `setTimeout`、事件监听、数组方法回调等，回调在别处执行时仍能访问定义时的外层变量。
3. **函数被赋给外部变量或对象属性**：与 1 类似，函数在别处被调用，作用域链上仍保留定义时的环境。

只要**内部函数在外部被持有并调用**，且它引用了外层变量，就会形成闭包。

***

## 三、常见应用场景

### 1. 数据封装（模块模式）

用闭包实现“私有变量”，只暴露有限接口。

```js
function createCache() {
  const cache = Object.create(null)
  return {
    get(key) { return cache[key] },
    set(key, value) { cache[key] = value },
    has(key) { return key in cache }
  }
}
const c = createCache()
c.set('a', 1)
c.get('a')  // 1；无法直接访问 cache
```

### 2. 函数柯里化（Currying）

固定部分参数，返回新函数，新函数闭包保存已传入参数。

```js
function add(a) {
  return function (b) {
    return a + b
  }
}
const add2 = add(2)
add2(3)  // 5
```

### 3. 防抖与节流

通过闭包保存 `timer`、`last` 等状态，在多次触发时共用同一变量。

```js
function debounce(fn, delay) {
  let timer = null
  return function (...args) {
    clearTimeout(timer)
    timer = setTimeout(() => fn.apply(this, args), delay)
  }
}
```

### 4. React Hooks

`useState` 等 Hook 依赖闭包保存组件的状态；每次渲染时函数组件重新执行，但 Hook 通过内部机制把状态与当次渲染绑定，本质也是“函数 + 持久化数据”。

***

## 四、注意事项：内存与性能

闭包会**保持对外层作用域变量的引用**，这些变量在闭包存活期间不会被 GC 回收。若闭包很多或引用了大对象，可能增加内存占用；在不需要时解除对闭包的引用（如移除事件监听、清空引用），有助于回收。

```js
function attachHandler() {
  const bigData = new Array(1000000).fill('x')
  element.onclick = function () {
    console.log(bigData.length)  // 闭包引用 bigData，一直不释放
  }
}
// 若 element 长期存在，bigData 也会长期存在
```

## 五、面试答题要点

- **定义**：闭包是函数与其词法作用域的绑定；内部函数在外部被使用时仍能访问定义时的外层变量。
- **形成条件**：内部函数被 return、传递或赋给外部，且引用了外层变量。
- **应用**：数据封装、柯里化、防抖节流、部分设计模式、React Hooks 等。
- **注意**：闭包会延长外层变量生命周期，需注意内存与及时解除引用。
