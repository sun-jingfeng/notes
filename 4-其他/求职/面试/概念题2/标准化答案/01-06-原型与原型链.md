# 原型与原型链

## 一、基本概念

**原型（prototype）** 是 JavaScript 实现继承与属性共享的机制。每个**函数**在创建时都会有一个 `prototype` 属性，指向一个对象（原型对象）；每个**对象**都有一个内部链接 `[[Prototype]]`（多数环境通过 `__proto__` 暴露），指向其“构造时的原型”。

### 1. 两条线

| 对象 | 属性 | 指向 |
| --- | --- | --- |
| **函数** | `prototype` | 该函数作为构造函数时，实例的默认原型 |
| **对象** | `__proto__`（即 [[Prototype]]） | 创建该对象时使用的构造函数的 `prototype` |

### 2. 关系式

- `实例.__proto__ === 构造函数.prototype`
- `构造函数.prototype.__proto__ === Object.prototype`
- `Object.prototype.__proto__ === null`（原型链终点）

***

## 二、原型链结构

属性查找时，先查对象自身，再沿 `__proto__` 向上查，直到 `Object.prototype` 或找不到为止，这条路径就是**原型链**。

```
    实例
      │ __proto__
      ▼
    构造函数.prototype
      │ __proto__
      ▼
    Object.prototype
      │ __proto__
      ▼
    null
```

```js
function F() {}
const obj = new F()

obj.__proto__ === F.prototype           // true
F.prototype.__proto__ === Object.prototype  // true
Object.prototype.__proto__ === null     // true
```

***

## 三、instanceof 的原理

`instanceof` 判断的是：在对象的原型链上，**是否存在**某构造函数的 `prototype` 对象。

```js
obj instanceof F
// 等价于：沿 obj -> obj.__proto__ -> ... 查找
// 是否有 === F.prototype 的节点
```

因此 `obj instanceof Object` 一般为 true（Object.prototype 在链上）；基本类型（如 `1 instanceof Number`）为 false，因为基本类型不是对象，没有原型链。

***

## 四、属性遮蔽与“修改”原型

子对象自身有某属性时，会遮蔽原型上的同名的属性；若没有，则用原型上的。给对象赋值只会改自身或原型上已有属性，不会在链上“新建”一层。

```js
function F() {}
F.prototype.x = 1
const a = new F()
const b = new F()

a.x = 2        // 在 a 自身上创建 x，不改 prototype
console.log(a.x)  // 2
console.log(b.x)  // 1（仍来自 prototype）
```

***

## 五、面试答题要点

- **prototype**：函数的 `prototype` 是其实例的默认原型对象。
- **__proto__**：对象的 `__proto__` 指向构造其的函数的 `prototype`，形成链。
- **原型链**：查找属性沿 `__proto__` 向上直到 `Object.prototype.__proto__ === null`。
- **instanceof**：沿实例的原型链查找是否存在某构造函数的 `prototype`。
