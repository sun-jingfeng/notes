# this 指向

## 一、this 是什么

**this** 是函数执行时的**上下文对象**，在**普通函数**里由**调用方式**决定，在**箭头函数**里由**定义时的外层 this** 决定。同一函数不同调用方式，this 可能不同。

***

## 二、普通函数中 this 的规则

### 1. 默认调用（独立调用）

函数单独执行，未挂载到对象上、未用 new、未用 call/apply/bind。

| 环境 | this |
| --- | --- |
| 非严格模式 | 全局对象（浏览器中为 `window`，Node 中为 `global`） |
| 严格模式 | `undefined` |

```js
function fn() { console.log(this) }
fn()  // window（非严格） / undefined（严格）
```

### 2. 方法调用（obj.fn()）

**谁调用，this 指向谁**（指向调用该方法的对象）。

```js
const obj = { name: 'obj', fn: function () { console.log(this.name) } }
obj.fn()  // 'obj'，this 为 obj
const f = obj.fn
f()       // 独立调用，this 为 window 或 undefined
```

### 3. 构造函数调用（new Fn()）

**this 指向新创建的对象**。

```js
function F() { this.x = 1 }
const o = new F()
console.log(o.x)  // 1，this 即 o
```

### 4. call / apply / bind

**显式指定 this**。`call`、`apply` 立即执行并指定 this；`bind` 返回一个固定 this 的新函数。

```js
function fn() { console.log(this.id) }
fn.call({ id: 1 })   // 1
fn.apply({ id: 2 })  // 2
const bound = fn.bind({ id: 3 })
bound()               // 3
```

### 5. 事件处理

绑定在 DOM 上的事件处理函数，在事件触发时通常 **this 指向绑定事件的元素**（除非用箭头函数或手动改了 this）。

```js
button.addEventListener('click', function () {
  console.log(this)  // button 元素
})
```

***

## 三、箭头函数的 this

箭头函数**没有自己的 this**，在定义时**继承外层（词法）作用域的 this**，之后无论如何调用都不会改变。

```js
const obj = {
  name: 'obj',
  normal: function () { return this.name },
  arrow: () => this.name  // 外层若是全局，this 为 window
}
obj.normal()  // 'obj'
obj.arrow()   // 非 obj，取决于定义时外层 this
```

***

## 四、小结表

| 调用方式 | **this 指向** |
| --- | --- |
| **独立调用** `fn()` | 非严格 → `window`；严格 → `undefined` |
| **方法调用** `obj.fn()` | `obj` |
| **new** `new Fn()` | 新创建的实例 |
| **call / apply / bind** | 传入的第一个参数（指定对象） |
| **箭头函数** | 定义时外层的 this |
| **DOM 事件处理** | 绑定事件的 DOM 元素（普通函数时） |

***

## 五、面试答题要点

- this 在**普通函数**中由**调用方式**决定：独立调用、方法调用、new、call/apply/bind、事件绑定各有规则。
- **箭头函数**的 this 在**定义时**就固定为外层 this，调用方式不影响。
- 记牢：独立调用容易丢 this；需要固定 this 时用箭头或 bind。
