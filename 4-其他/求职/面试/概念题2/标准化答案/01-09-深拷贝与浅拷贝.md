# 深拷贝与浅拷贝

## 一、浅拷贝与深拷贝

| 类型 | 含义 |
| --- | --- |
| **浅拷贝** | 只复制“第一层”；若属性是引用类型，复制的是引用，新旧对象共享嵌套对象 |
| **深拷贝** | 递归复制每一层，新对象与原对象完全独立，修改互不影响 |

```js
const origin = { a: 1, b: { c: 2 } }
const shallow = { ...origin }
shallow.b.c = 99
console.log(origin.b.c)  // 99，被改动了

// 深拷贝后改 shallow 不会影响 origin
```

***

## 二、常见实现方式

### 1. 浅拷贝

| 方式 | 说明 |
| --- | --- |
| **Object.assign(target, ...sources)** | 将源对象可枚举属性拷贝到 target，只拷贝第一层 |
| **展开运算符** `{...obj}`、`[...arr]` | 只拷贝第一层，嵌套仍是引用 |

### 2. 深拷贝

| 方式 | 优点 | 缺点/注意 |
| --- | --- | --- |
| **JSON.parse(JSON.stringify(x))** | 简单 | 不支持函数、undefined、Symbol、Date（变字符串）、RegExp、循环引用；会忽略不可枚举属性 |
| **structuredClone(value)** | 原生、支持循环引用、支持部分内置类型 | 不支持函数、Symbol、某些特殊对象；需环境支持 |
| **手写递归** | 可处理各种类型、可加循环引用检测 | 需自己处理 Date/RegExp/Map/Set 等，用 WeakMap 防循环引用 |

```js
// 循环引用示例
const a = { name: 'a' }
a.self = a
JSON.parse(JSON.stringify(a))  // 报错或丢属性
structuredClone(a)             // 可得到 { name: 'a', self: [Circular] }
```

***

## 三、手写深拷贝要点

- 区分基本类型与引用类型：基本类型直接返回。
- 引用类型：按 **Object / Array / Date / RegExp / Map / Set** 等分别构造新对象并递归拷贝内容。
- **循环引用**：用 **WeakMap** 记录“原对象 → 新对象”，若已拷贝过直接返回新对象，避免栈溢出。

```js
function deepClone(val, map = new WeakMap()) {
  if (val === null || typeof val !== 'object') return val
  if (map.has(val)) return map.get(val)
  if (val instanceof Date) return new Date(val)
  if (val instanceof RegExp) return new RegExp(val)
  const copy = Array.isArray(val) ? [] : {}
  map.set(val, copy)
  for (const k of Object.keys(val)) {
    copy[k] = deepClone(val[k], map)
  }
  return copy
}
```

***

## 四、面试答题要点

- **浅拷贝**：只复制第一层，嵌套引用共享；可用 `Object.assign`、展开运算。
- **深拷贝**：层层复制，完全独立；`JSON` 方式简单但有诸多限制；`structuredClone` 原生且支持循环引用；手写需处理类型与循环引用（WeakMap）。
