# 事件循环（Event Loop）

## 一、为什么需要事件循环

JavaScript 是**单线程**的，同一时间只能执行一段代码。浏览器需要处理脚本、事件、渲染、网络等，通过 **事件循环** 在“执行一段代码 → 处理其他任务”之间轮转，实现“非阻塞”的并发效果。

***

## 二、相关概念

### 1. 调用栈（Call Stack）

同步代码按顺序入栈、执行、出栈。函数调用时入栈，返回时出栈。若栈一直不空（如死循环），后续任务就无法执行。

### 2. 任务队列

异步任务在完成后会把**回调**放入任务队列，等待被调度执行。任务分为：

| 类型 | 来源示例 |
| --- | --- |
| **宏任务** | `setTimeout`、`setInterval`、I/O、UI 渲染、`setImmediate`（Node） |
| **微任务** | `Promise.then/catch/finally`、`queueMicrotask`、`MutationObserver` |

### 3. 微任务优先

在一轮循环中，**微任务会在当前宏任务之后、下一个宏任务之前被清空**。因此微任务会早于“下一个”宏任务执行。

***

## 三、事件循环流程（简化）

```
    1. 执行当前调用栈中的同步代码（可能产生微任务、宏任务）
        ↓
    2. 同步代码执行完毕，将当前产生的所有微任务依次执行并清空
        ↓
    3. 如需渲染（浏览器），则进行渲染
        ↓
    4. 从宏任务队列中取一个任务进入调用栈执行（回到步骤 1）
        ↓
    循环 1～4
```

可记为：**同步 → 清空微任务 →（渲染）→ 取一个宏任务 → 再同步 → 清空微任务 → …**

***

## 四、图示（概念）

```
    ┌─────────────────────────────────┐
    │          调用栈（Call Stack）      │ ← 同步代码在此执行
    └─────────┬───────────────────────┘
              │ 同步执行完毕
              ▼
    ┌─────────────────────────────────┐
    │        微任务队列（Microtask）     │ ← 全部清空
    └─────────┬───────────────────────┘
              │ 微任务清空后
              ▼
    ┌─────────────────────────────────┐
    │     宏任务队列（取一个 Macrotask）  │ ← 取一个执行
    └─────────┬───────────────────────┘
              │ 执行完毕后回到微任务
              ▼
           循环往复
```

***

## 五、例题理解

```js
console.log(1)
setTimeout(() => console.log(2), 0)
Promise.resolve().then(() => console.log(3))
console.log(4)
// 1 4 3 2
// 同步 1、4 → 微任务 3 → 宏任务 2
```

***

## 六、面试答题要点

- **事件循环**：在单线程下协调同步与异步；同步先执行，再清空微任务，再取一个宏任务，如此循环。
- **微任务先于下一宏任务**：Promise.then、queueMicrotask 等会在当前同步和当前宏任务之后、下一个宏任务之前全部执行完。
- **宏任务**：setTimeout、setInterval、I/O、渲染等；**微任务**：Promise 回调、queueMicrotask、MutationObserver。
