# 防抖与节流

## 一、为什么需要防抖和节流

高频事件（如 `resize`、`scroll`、`input`、`mousemove`）会在一段时间内触发很多次，若每次触发都执行较重逻辑，会造成性能问题或重复请求。**防抖**和**节流**都是“限制执行频率”的手段，但策略不同。

***

## 二、防抖（debounce）

**含义**：在连续触发时，**只执行最后一次**；每次新触发都会重新计时，只有“停顿超过设定时间”后才真正执行。

| 项目 | 说明 |
| --- | --- |
| **典型场景** | 搜索框输入（等用户停输再发请求）、窗口 resize（等调整完再算布局） |
| **效果** | 多次触发合并为一次执行 |

```js
function debounce(fn, delay) {
  let timer = null
  return function (...args) {
    clearTimeout(timer)
    timer = setTimeout(() => fn.apply(this, args), delay)
  }
}

const onSearch = debounce(() => fetchSuggestions(keyword), 300)
input.addEventListener('input', onSearch)
```

***

## 三、节流（throttle）

**含义**：在连续触发时，**在固定时间间隔内最多执行一次**；到了间隔就执行一次，然后重新计时。

| 项目 | 说明 |
| --- | --- |
| **典型场景** | 滚动加载更多、按钮防重复点击、mousemove 更新位置 |
| **效果** | 执行频率被限制为“每 interval 一次” |

**时间戳版**：用“上次执行时间”判断是否已过间隔。

```js
function throttle(fn, interval) {
  let last = 0
  return function (...args) {
    const now = Date.now()
    if (now - last >= interval) {
      last = now
      fn.apply(this, args)
    }
  }
}
```

**定时器版**：第一次可立即执行或延迟执行，之后按间隔执行，取决于实现。

***

## 四、对比小结

| 对比项 | **防抖** | **节流** |
| --- | --- | --- |
| **执行时机** | 连续触发停止后，延迟一次执行 | 固定间隔内至少执行一次 |
| **典型场景** | 搜索联想、resize | 滚动、按钮连点、mousemove |
| **语义** | “停下来了再执行” | “每隔一段时间执行一次” |

***

## 五、面试答题要点

- **防抖**：连续触发只执行最后一次，重新计时；用于搜索、resize 等“等用户停下来的场景”。
- **节流**：固定时间间隔内最多执行一次；用于滚动、点击、mousemove 等“需要周期性执行”的场景。
- **实现**：防抖用 `setTimeout` + `clearTimeout`；节流可用时间戳或定时器，注意 `this` 与参数用 `apply` 保持。
