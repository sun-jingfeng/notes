# 自定义 Hook

## 一、什么是自定义 Hook

**自定义 Hook** 是以 **`use` 开头**、内部可调用其它 Hook 的**函数**，用于把**可复用的状态逻辑**从组件里抽离出来，实现逻辑复用而不改变组件树结构。

***

## 二、命名与规则

- 函数名必须以 **use** 开头，便于 ESLint 等工具识别并检查 Hooks 规则。
- 内部可调用 **useState、useEffect、useContext** 等任意 Hook；遵守“只在顶层调用”等 Hooks 规则。

***

## 三、常见自定义 Hook 示例

| Hook | 作用 |
| --- | --- |
| **useDebounce / useThrottle** | 对输入值或回调做防抖/节流，减少请求或计算次数 |
| **useLocalStorage** | 把 state 同步到 localStorage，刷新不丢失 |
| **useFetch** | 封装请求 + loading/error/data 状态，可选依赖触发重新请求 |
| **useClickOutside** | 判断点击是否在目标元素外，用于关闭下拉、弹层等 |
| **useMediaQuery** | 根据媒体查询结果返回布尔（如是否移动端、是否暗色） |

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value)
  useEffect(() => {
    const id = setTimeout(() => setDebouncedValue(value), delay)
    return () => clearTimeout(id)
  }, [value, delay])
  return debouncedValue
}
```

***

## 四、设计原则

- **单一职责**：一个 Hook 只做一类事，便于复用和测试。
- **可组合**：内部可调用其它自定义 Hook，形成组合逻辑。
- **返回值语义清晰**：返回对象时用有意义的属性名（如 `{ data, loading, error }`），便于调用方使用。

***

## 五、面试答题要点

- **定义**：以 use 开头、内部可调其它 Hook 的函数，用于抽离可复用状态逻辑。
- **常见**：useDebounce、useThrottle、useLocalStorage、useFetch、useClickOutside、useMediaQuery 等。
- **原则**：单一职责、可组合、返回值清晰。
