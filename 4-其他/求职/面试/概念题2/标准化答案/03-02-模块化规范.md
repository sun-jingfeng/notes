# 模块化规范

## 一、常见规范

| 规范 | 语法 | 加载方式 | 典型环境 |
| --- | --- | --- | --- |
| **CommonJS (CJS)** | `require` / `module.exports` | 同步加载 | Node.js 默认 |
| **ES Module (ESM)** | `import` / `export` | 静态分析、可静态加载 | 浏览器原生、Node 新版本、打包器 |
| **AMD** | `define` / `require` | 异步加载 | 已过时（RequireJS） |
| **UMD** | 条件判断 | 兼容 CJS/AMD/全局 | 库发布 |

***

## 二、ESM 与 CJS 核心区别

| 对比项 | **ESM** | **CJS** |
| --- | --- | --- |
| **加载时机** | 编译/解析阶段可确定依赖（静态） | 运行时执行到 require 才加载（动态） |
| **导出** | 导出的是**引用**（live binding），模块内改值会影响导入方 | 导出的是**拷贝**，修改不会反向影响 |
| **Tree Shaking** | 可静态分析未使用导出并剔除 | 难以静态分析，一般无法 Tree Shaking |
| **顶层 await** | 支持 | 不支持 |
| **this** | 顶层为 undefined | 顶层为 module.exports |

```js
// ESM：live binding
// a.js
export let count = 0
export function inc() { count++ }
// b.js
import { count, inc } from './a'
inc()
console.log(count)  // 1，拿到的是引用

// CJS：拷贝
// a.js
let count = 0
module.exports = { count, inc: () => count++ }
// b.js
const { count, inc } = require('./a')
inc()
console.log(count)  // 0，拿到的是当时的拷贝
```

***

## 三、面试答题要点

- **CJS**：require/module.exports、同步、Node 默认、导出为拷贝。
- **ESM**：import/export、静态分析、导出为引用、支持 Tree Shaking 与顶层 await。
- **UMD**：兼容 CJS + AMD + 全局，常用于库。
