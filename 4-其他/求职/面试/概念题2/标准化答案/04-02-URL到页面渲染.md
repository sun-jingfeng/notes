# 从输入 URL 到页面渲染

## 一、整体流程

```
URL 解析 → DNS 解析 → TCP 三次握手 → (HTTPS 则 TLS 握手) → 发 HTTP 请求
    → 服务器处理并返回响应
    → 浏览器解析 HTML，构建 DOM 树
    → 解析 CSS，构建 CSSOM 树
    → 结合 DOM + CSSOM 生成渲染树（Render Tree）
    → 布局（Layout）计算几何信息
    → 绘制（Paint）成位图
    → 合成（Composite）到图层并显示
```

***

## 二、各阶段简述

| 阶段 | 说明 |
| --- | --- |
| **URL 解析** | 解析协议、主机、路径等，决定用 HTTP/HTTPS 等 |
| **DNS** | 域名 → IP，可能走缓存、本地 hosts、递归查询 |
| **TCP** | 三次握手建立连接；HTTPS 再走 TLS 握手 |
| **HTTP 请求/响应** | 请求行、头、体；响应状态、头、体 |
| **解析 HTML** | 边下载边解析，生成 **DOM 树**；遇到 script 可能阻塞解析 |
| **解析 CSS** | 生成 **CSSOM**；CSS 不会阻塞 DOM 解析，但会阻塞渲染 |
| **渲染树** | 只包含可见节点，结合 DOM + 计算后的样式 |
| **布局** | 计算节点位置、大小（重排会触发） |
| **绘制** | 转成像素（重绘会触发） |
| **合成** | 多层合成后输出到屏幕 |

***

## 三、关键优化点

- **DNS**：预解析（`<link rel="dns-prefetch">`）、减少域名数。
- **连接**：Keep-Alive、HTTP/2 多路复用、减少握手。
- **资源**：CSS 放头部、JS 放底部或 defer/async、关键资源预加载。
- **渲染**：减少重排重绘、用 transform/opacity 做动画、虚拟列表等。

***

## 四、面试答题要点

- **流程**：URL → DNS → TCP（+TLS）→ HTTP → 解析 HTML/CSS → DOM+CSSOM → 渲染树 → 布局 → 绘制 → 合成 → 显示。
- **优化**：DNS 预解析、资源顺序与异步、减少重排重绘、长列表优化。
