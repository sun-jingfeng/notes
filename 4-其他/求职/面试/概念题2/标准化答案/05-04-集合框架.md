# 集合框架

## 一、体系结构

```
    Collection
        ├── List（有序、可重复）→ ArrayList（数组）、LinkedList（双向链表）
        ├── Set（无序、不可重复）→ HashSet、LinkedHashSet、TreeSet
        └── Queue → LinkedList、PriorityQueue 等

    Map（键值对）
        ├── HashMap、LinkedHashMap、TreeMap、ConcurrentHashMap
```

***

## 二、List / Set 选型要点

| 实现 | 底层 | 特点 |
| --- | --- | --- |
| **ArrayList** | 数组 | 下标访问 O(1)；尾部增删 O(1)；中间增删需移动，慢 |
| **LinkedList** | 双向链表 | 头尾增删 O(1)；按下标访问需遍历，O(n) |
| **HashSet** | 基于 HashMap | 无序、去重；O(1) 增删查 |
| **LinkedHashSet** | 链表+HashMap | 保持插入顺序 |
| **TreeSet** | 红黑树 | 有序（自然或 Comparator）；O(log n) |

***

## 三、HashMap 高频考点

| 问题 | 要点 |
| --- | --- |
| **结构** | 数组 + 链表；链表长度≥8 且数组长度≥64 时链表转红黑树 |
| **容量与扩容** | 默认容量 16，负载因子 0.75，超过阈值扩容为 2 倍 |
| **hash** | `(h = key.hashCode()) ^ (h >>> 16)` 高低位混合，再 `(n-1) & hash` 得下标 |
| **容量为 2 的幂** | `(n-1) & hash` 等价于取模，位运算更快；扩容时便于 rehash |
| **线程安全** | 非线程安全；多线程用 **ConcurrentHashMap**（JDK8 后 CAS + synchronized 桶） |

***

## 四、面试答题要点

- **List**：ArrayList 查快增删慢，LinkedList 头尾增删快、按索引查慢。
- **Set**：HashSet 无序去重；LinkedHashSet 保序；TreeSet 有序。
- **HashMap**：数组+链表/红黑树、负载因子 0.75、2 的幂容量、多线程用 ConcurrentHashMap。
