# 多线程

## 一、创建线程的方式

| 方式 | 说明 |
| --- | --- |
| 继承 **Thread** | 重写 `run()`，`new MyThread().start()` |
| 实现 **Runnable** | 实现 `run()`，`new Thread(runnable).start()`；无返回值 |
| 实现 **Callable** | 实现 `call()`，通过 **FutureTask** 或线程池提交，可获返回值、异常 |
| **线程池** | `ExecutorService` 提交任务，复用线程，推荐生产使用 |

***

## 二、线程状态

**NEW** → **RUNNABLE**（就绪/运行）→ **BLOCKED**（等 monitor 锁）/ **WAITING**（无时限等待）/ **TIMED_WAITING**（有时限等待）→ **TERMINATED**。

***

## 三、synchronized 与 volatile

| 关键字 | 作用 |
| --- | --- |
| **synchronized** | 保证同一时刻只有一个线程执行同步块；可见性 + 有序性（锁内）；可重入；JDK6 后锁升级（偏向→轻量→重量） |
| **volatile** | 保证可见性、禁止指令重排；**不保证复合操作的原子性**（如 i++） |

***

## 四、线程池

**ThreadPoolExecutor** 核心参数：核心线程数、最大线程数、空闲存活时间、时间单位、工作队列、线程工厂、拒绝策略。

| 拒绝策略 | 行为 |
| --- | --- |
| **AbortPolicy** | 抛 RejectedExecutionException |
| **CallerRunsPolicy** | 调用者线程执行任务 |
| **DiscardPolicy** | 静默丢弃 |
| **DiscardOldestPolicy** | 丢弃队列最旧任务，再提交 |

***

## 五、面试答题要点

- **创建**：Thread、Runnable、Callable+Future、线程池；生产用线程池。
- **synchronized**：互斥、可见、可重入；锁升级。
- **volatile**：可见、禁重排；不保证原子性。
- **线程池**：核心/最大线程数、队列、拒绝策略（Abort/CallerRuns/Discard/DiscardOldest）。
