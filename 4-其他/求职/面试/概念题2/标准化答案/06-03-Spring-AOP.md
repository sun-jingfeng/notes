# Spring AOP

## 一、AOP 是什么

**AOP（面向切面编程）**把与业务无关的横切逻辑（日志、事务、权限、统计等）从业务代码中抽离，集中到“切面”里，通过**动态代理**在运行时织入到目标方法上，实现解耦与复用。

***

## 二、核心概念

| 概念 | 说明 |
| --- | --- |
| **切面（Aspect）** | 横切逻辑的封装，通常是一个带 `@Aspect` 的类 |
| **切入点（Pointcut）** | 用表达式定义在哪些连接点上生效，如 `execution(* com.xxx.service.*.*(..))` |
| **通知（Advice）** | 切面在连接点执行的动作：@Before、@After、@AfterReturning、@AfterThrowing、**@Around** |
| **连接点（JoinPoint）** | 可被拦截的点，如方法调用 |
| **目标对象（Target）** | 被代理的 Bean |

**@Around** 最强大：可决定是否执行目标方法、修改参数与返回值、统一处理异常和耗时统计。

***

## 三、代理方式

| 条件 | 代理方式 |
| --- | --- |
| 目标实现了接口 | **JDK 动态代理**（基于接口） |
| 目标未实现接口 | **CGLIB 代理**（继承子类） |

Spring 默认：有接口用 JDK，没有用 CGLIB；可配置强制 CGLIB。

***

## 四、面试答题要点

- **AOP**：横切逻辑与业务解耦，通过动态代理织入。
- **概念**：切面、切入点、通知（Before/After/Around 等）、连接点。
- **@Around** 可完全控制目标方法执行与返回值。
- **代理**：有接口用 JDK 动态代理，无接口用 CGLIB。
