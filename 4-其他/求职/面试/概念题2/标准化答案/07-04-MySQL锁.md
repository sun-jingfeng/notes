# MySQL 锁

## 一、按模式

| 锁 | 说明 |
| --- | --- |
| **共享锁（S）** | 读锁；多事务可同时持有；`SELECT ... LOCK IN SHARE MODE` |
| **排他锁（X）** | 写锁；与 S、X 均互斥；INSERT/UPDATE/DELETE 默认加 X；`SELECT ... FOR UPDATE` |

***

## 二、按粒度（InnoDB）

| 锁 | 说明 |
| --- | --- |
| **行锁** | 锁单行（通过索引）；无索引可能退化为表锁 |
| **间隙锁（Gap Lock）** | 锁索引区间间隙，防幻读 |
| **临键锁（Next-Key Lock）** | 行锁+间隙锁，InnoDB 默认范围锁 |
| **表锁** | 锁整表；无索引更新等可能退化为表锁 |

***

## 三、面试答题要点

- **S/X**：共享锁可多读；排他锁与其它互斥。
- **行锁/间隙锁/临键锁**：InnoDB 行级锁与防幻读；无索引可能升级为表锁。
