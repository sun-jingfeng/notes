# Redis 缓存三大问题

## 一、缓存穿透

**现象**：查**不存在的数据**，缓存没有、数据库也没有，请求每次都打到库，高并发时压垮 DB。

**应对**：
- **缓存空值**：对“查无结果”也缓存（设较短 TTL），避免重复查库。
- **布隆过滤器**：在缓存前加一层，判断 key 是否“一定不存在”，不存在则直接返回，减少对库的无效查询。

***

## 二、缓存击穿

**现象**：**热点 key** 过期瞬间，大量请求同时发现缓存失效，一起打到数据库。

**应对**：
- **互斥锁**：只有一个线程去加载并回填缓存，其它线程等待或重试读缓存。
- **逻辑过期**：缓存里存的数据带“逻辑过期时间”，不删 key；后台异步更新，读时若逻辑过期则触发更新并可能用旧数据兜底。

***

## 三、缓存雪崩

**现象**：**大量 key 同时过期**或 **Redis 宕机**，请求全部打到数据库或服务不可用。

**应对**：
- **过期时间打散**：在基础 TTL 上加随机值，避免同时失效。
- **多级缓存**：本地缓存 + Redis，Redis 挂时本地可兜底部分流量。
- **高可用**：主从、集群、哨兵，减少单点故障。

***

## 四、面试答题要点

- **穿透**：查不存在的数据；空值缓存 + 布隆过滤器。
- **击穿**：热点 key 过期瞬间；互斥锁、逻辑过期。
- **雪崩**：大量 key 同时过期或 Redis 宕机；随机过期、多级缓存、高可用。
