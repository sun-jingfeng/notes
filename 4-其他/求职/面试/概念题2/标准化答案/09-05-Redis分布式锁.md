# Redis 分布式锁

## 一、基本要求

- **互斥**：同一时刻只有一个客户端能持有锁。
- **防误删**：只能释放自己加的锁（value 存唯一标识，如 UUID）。
- **原子性**：加锁、设置过期时间需原子（如 SET key value NX EX seconds）；释放时“判断再删”也需原子（Lua 脚本或 Redisson 等）。

***

## 二、实现要点

| 要点 | 说明 |
| --- | --- |
| **加锁** | `SET key 唯一值 NX EX 过期秒数`，保证存在则不加、并带过期 |
| **value** | 存 UUID 或线程标识，释放时校验再删，避免删别人锁 |
| **释放** | Lua 脚本里“比较 value 再 DEL”，保证原子 |
| **续期** | 业务执行时间可能超过过期时间时，用 Redisson 的 WatchDog 或自己定时续期 |
| **可重入** | Redisson 用 Hash 存 clientId 与重入次数，同一客户端可多次加锁 |

***

## 三、面试答题要点

- **加锁**：SET NX EX 原子；value 唯一标识。
- **释放**：Lua 判断 value 再 DEL，防误删。
- **续期**：WatchDog 或定时续期；可重入用 Hash 记次数。
