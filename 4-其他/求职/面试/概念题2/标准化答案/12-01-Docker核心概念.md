# Docker 核心概念

## 一、核心概念

| 概念 | 说明 |
| --- | --- |
| **镜像（Image）** | 只读模板，分层存储（UnionFS）；由 Dockerfile 或仓库构建 |
| **容器（Container）** | 镜像的运行实例，带可写层；可创建、启动、停止、删除 |
| **Dockerfile** | 构建镜像的脚本，包含 FROM、RUN、COPY、CMD 等指令 |
| **Docker Compose** | 用 YAML 定义多容器应用，一条命令启停一组服务 |
| **Volume** | 数据卷，持久化数据，容器删除后数据仍可保留 |

***

## 二、镜像与容器

- 镜像是静态的、分层的；容器是镜像的运行态，有生命周期。
- 同一镜像可启多个容器，彼此隔离；删除容器不影响镜像。

***

## 三、面试答题要点

- **镜像**：只读模板、分层；**容器**：镜像实例、可写层。
- **Dockerfile**：构建镜像；**Compose**：多容器编排；**Volume**：持久化。
