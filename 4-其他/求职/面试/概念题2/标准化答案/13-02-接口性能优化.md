# 接口性能优化思路

## 一、分层优化手段

| 层面 | 手段 |
| --- | --- |
| **SQL** | 建索引、避免全表扫描、分页用游标或延迟关联、批量操作、只查需要的列 |
| **缓存** | Redis 缓存热点数据、本地缓存（Caffeine/Guava）减少重复计算或查库 |
| **异步** | 非核心逻辑用 MQ 或 @Async 异步处理，缩短接口响应时间 |
| **并行** | 多路独立依赖用 CompletableFuture 等并行调用，取最慢一路作为耗时 |
| **数据库** | 读写分离、分库分表（数据量大时）；连接池参数合理 |
| **连接与 IO** | 数据库/HTTP 连接池大小、超时、是否复用 |

***

## 二、分析顺序

先**定位瓶颈**（慢 SQL、高 QPS 接口、锁竞争、GC 等），再用 EXPLAIN、链路追踪、压测等验证，再针对性地加缓存、改 SQL、异步化、并行化等。

***

## 三、面试答题要点

- **SQL**：索引、分页、批量、避免 *。
- **缓存**：Redis + 本地缓存。
- **异步/并行**：MQ、@Async、CompletableFuture。
- **库**：读写分离、分库分表、连接池。
