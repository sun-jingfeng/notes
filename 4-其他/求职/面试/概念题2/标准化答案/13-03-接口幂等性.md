# 如何保证接口幂等性

## 一、什么是幂等

**幂等**：同一请求执行多次与执行一次效果相同，不会因重复提交产生重复扣款、重复下单等。

***

## 二、常见方案

| 方案 | 适用场景 | 说明 |
| --- | --- | --- |
| **唯一索引** | 插入类 | 数据库唯一约束防重复插入；重复请求报错或忽略 |
| **Token 机制** | 表单/提交类 | 前端先拿一次性 token，提交时带 token，后端校验后删除；重复提交 token 无效 |
| **乐观锁** | 更新类 | 用 version 或条件更新：`UPDATE ... SET version=version+1 WHERE id=? AND version=?`，影响行数为 0 则说明已处理或条件不满足 |
| **状态机** | 有状态流转的业务 | 仅允许从某状态到下一状态（如待支付→已支付），重复请求状态不对则拒绝 |
| **去重表** | 通用 | 用请求唯一 ID 或业务唯一键建表/存 Redis，先查再处理，防重复 |

***

## 三、面试答题要点

- **唯一索引**：防重复插入。
- **Token**：一次性 token，防重复提交。
- **乐观锁**：version 或条件更新。
- **状态机**：按状态流转拒绝非法重复。
- **去重表/请求 ID**：记录已处理请求，重复则忽略或返回原结果。
