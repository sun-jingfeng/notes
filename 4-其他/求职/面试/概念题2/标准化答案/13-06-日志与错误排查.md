# 日志和错误排查

## 一、前端

| 手段 | 说明 |
| --- | --- |
| **Sentry 等** | 上报 JS 异常、Promise 未捕获、资源错误，配合 Source Map 定位源码 |
| **Console + Source Map** | 本地或测试环境保留 source map，便于控制台定位到具体文件与行 |
| **Performance / Network** | 分析加载、长任务、请求耗时，定位性能与接口问题 |

***

## 二、后端

| 手段 | 说明 |
| --- | --- |
| **Logback/SLF4J** | 分级（ERROR/WARN/INFO/DEBUG）、按包配置、落文件或滚动文件 |
| **ELK / Loki 等** | 日志采集、检索、聚合，便于全链路排查 |
| **链路追踪** | Sleuth + Zipkin、SkyWalking 等，用 traceId 串联一次请求经过的所有服务，看耗时与异常所在服务 |

***

## 三、通用

- **结构化日志**：统一包含 traceId、userId、接口等，便于检索与串联。
- **告警**：对 ERROR、超时、关键业务失败等配置告警，及时发现问题。

***

## 四、面试答题要点

- **前端**：Sentry、Source Map、Performance/Network。
- **后端**：分级日志、ELK 等集中检索、链路追踪（traceId）。
- **通用**：结构化日志、告警。
