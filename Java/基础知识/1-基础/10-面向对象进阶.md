## 一、继承（Inheritance）

### 1. 什么是继承

继承是面向对象三大特征之一，让一个类（子类）获取另一个类（父类）的属性和方法。

**核心思想**：共性抽取，代码复用

### 2. 继承的语法

```java
public class 子类 extends 父类 {
    // 子类特有内容
}
```

### 3. 继承示例

```java
// ========== 父类 ==========
public class Animal {
    String name;
    int age;
    
    public void eat() {
        System.out.println("动物吃东西");
    }
    
    public void sleep() {
        System.out.println("动物睡觉");
    }
}

// ========== 子类 ==========
public class Dog extends Animal {
    // 继承了 name、age、eat()、sleep()
    
    // 子类特有的方法
    public void bark() {
        System.out.println("狗在叫");
    }
}

public class Cat extends Animal {
    public void catchMouse() {
        System.out.println("猫抓老鼠");
    }
}
```

### 4. 继承的特点

| 特点       | 说明                       |
| -------- | ------------------------ |
| **单继承**  | Java 只支持单继承，一个类只能有一个直接父类 |
| **多层继承** | 可以 A → B → C 形成继承链       |
| **共同父类** | 所有类都直接或间接继承自 Object 类    |

```java
class A { }
class B extends A { }
class C extends B { }  // C → B → A → Object
```

### 5. 子类能继承什么？

| 成员类型           | 能否继承                   |
| -------------- | ---------------------- |
| 父类构造方法         | ❌ 不能继承（但可以通过 super 调用） |
| 父类 private 成员  | ❌ 不能直接访问（但存在于子类对象中）    |
| 父类非 private 成员 | ✅ 可以继承和使用              |

### 6. 方法重写（Override）

子类可以重写父类的方法：

```java
public class Dog extends Animal {
    @Override  // 重写注解，建议加上
    public void eat() {
        System.out.println("狗吃骨头");  // 覆盖父类的 eat 方法
    }
}
```

**重写规则：**

| 规则        | 说明                            |
| --------- | ----------------------------- |
| 方法名       | 必须相同                          |
| 参数列表      | 必须相同                          |
| 返回值       | 相同或是父类返回值的子类（协变返回类型）          |
| 访问权限      | 不能比父类更严格（可以更宽松）               |
| 异常        | 不能抛出比父类更多/更大的编译时异常            |
| @Override | 建议加上，编译器会检查                   |
| 不能重写      | `private`、`final`、`static` 方法 |

```java
// 协变返回类型示例
public class Animal {
    public Animal create() { return new Animal(); }
}

public class Dog extends Animal {
    @Override
    public Dog create() {  // ✅ 返回值可以是子类类型
        return new Dog(); 
    }
}
```

### 7. 继承中的构造方法

```java
public class Animal {
    String name;
    
    public Animal() {
        System.out.println("Animal 无参构造");
    }
    
    public Animal(String name) {
        this.name = name;
    }
}

public class Dog extends Animal {
    int age;
    
    public Dog() {
        super();  // 默认调用父类无参构造（可省略）
        System.out.println("Dog 无参构造");
    }
    
    public Dog(String name, int age) {
        super(name);  // 调用父类有参构造
        this.age = age;
    }
}
```

**规则**：

*   子类构造方法**第一行**默认是 `super()`
*   如果父类没有无参构造，子类必须显式调用 `super(参数)`
*   `super()` 和 `this()` **不能同时出现**在构造方法中（都必须在第一行）

### 8. this 和 super 对比

| 关键字       | 访问成员变量        | 访问成员方法          | 访问构造方法        |
| --------- | ------------- | --------------- | ------------- |
| **this**  | this.变量名（本类）  | this.方法名()（本类）  | this()（本类构造）  |
| **super** | super.变量名（父类） | super.方法名()（父类） | super()（父类构造） |

```java
public class Dog extends Animal {
    String name = "小狗";
    
    public void show() {
        String name = "局部";
        System.out.println(name);        // 局部
        System.out.println(this.name);   // 小狗（本类成员）
        System.out.println(super.name);  // 父类的 name
    }
}
```

## 二、Object 类

### 1. Object 类的地位

`Object` 是 Java 中**所有类的根类**（祖宗类），所有类都直接或间接继承自 Object。

```java
class Person { }  
// 等价于
class Person extends Object { }
```

### 2. Object 类的常用方法

| 方法                    | 说明            |
| --------------------- | ------------- |
| `toString()`          | 返回对象的字符串表示    |
| `equals(Object obj)`  | 比较两个对象是否相等    |
| `hashCode()`          | 返回对象的哈希码值     |
| `getClass()`          | 返回对象的运行时类     |
| `clone()`             | 创建并返回对象的副本    |
| `wait()` / `notify()` | 线程等待/唤醒（多线程用） |
| `finalize()`          | 垃圾回收前调用（已废弃）  |

### 3. toString() 方法

```java
public class Student {
    private String name;
    private int age;
    
    // 不重写 toString，打印对象会输出：类名@地址值
    // 如：Student@1b6d3586
    
    // 重写 toString
    @Override
    public String toString() {
        return "Student{name='" + name + "', age=" + age + "}";
    }
}

// 使用
Student s = new Student("张三", 18);
System.out.println(s);  // Student{name='张三', age=18}
```

### 4. equals() 方法

```java
public class Student {
    private String name;
    private int age;
    
    // 不重写 equals，比较的是地址值（== 的效果）
    
    // 重写 equals，按内容比较
    @Override
    public boolean equals(Object o) {
        // 1. 如果是同一个对象，返回 true
        if (this == o) return true;
        
        // 2. 如果为 null 或类型不同，返回 false
        if (o == null || getClass() != o.getClass()) return false;
        
        // 3. 转换类型，比较属性
        Student student = (Student) o;
        return age == student.age && 
               Objects.equals(name, student.name);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }
}
```

**注意**：重写 `equals()` 时，通常也要重写 `hashCode()`（HashMap、HashSet 依赖此约定）

### 5. clone() 方法

```java
// 1. 实现 Cloneable 接口（标记接口）
public class Person implements Cloneable {
    String name;
    int[] scores;
    
    // 2. 重写 clone 方法
    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();  // 浅拷贝
    }
}
```

**浅拷贝 vs 深拷贝：**

| 类型      | 说明              | 引用类型成员       |
| ------- | --------------- | ------------ |
| **浅拷贝** | `super.clone()` | 拷贝引用（指向同一对象） |
| **深拷贝** | 手动拷贝引用类型成员      | 创建新对象        |

```java
// 深拷贝实现
@Override
protected Object clone() throws CloneNotSupportedException {
    Person copy = (Person) super.clone();
    // 手动拷贝引用类型成员
    copy.scores = this.scores.clone();
    return copy;
}
```

### 6. getClass() 方法

```java
Student s = new Student();
Class<?> clazz = s.getClass();

System.out.println(clazz.getName());        // com.example.Student
System.out.println(clazz.getSimpleName());  // Student
```

### 7. IDEA 快捷生成

`Alt + Insert`（Win）/ `Cmd + N`（Mac）→ 选择 `toString()` 或 `equals() and hashCode()`

## 三、final 关键字

### 1. final 的作用

`final` 表示**最终的、不可改变的**，可以修饰类、方法、变量、参数。

### 2. final 修饰类

```java
// final 修饰的类不能被继承
public final class Math {
    // ...
}

// 错误！不能继承 final 类
// public class MyMath extends Math { }  ❌
```

**Java 中的 final 类**：String、Math、System、包装类等

### 3. final 修饰方法

```java
public class Animal {
    // final 修饰的方法不能被重写
    public final void sleep() {
        System.out.println("动物睡觉");
    }
}

public class Dog extends Animal {
    // 错误！不能重写 final 方法
    // @Override
    // public void sleep() { }  ❌
}
```

### 4. final 修饰变量

#### 4.1 final 修饰成员变量

**规则：final 成员变量必须初始化，且只能赋值一次。初始化有三种方式：**

**方式 1：声明时初始化**

```java
public class Demo {
    // ✅ 声明时直接初始化
    private final int MAX_VALUE = 100;
}
```

**方式 2：构造函数中初始化（推荐）⭐**

```java
public class UserController {
    // ✅ 声明时未初始化
    private final UserService userService;

    // ✅ 在构造函数中初始化（完全合法！）
    public UserController(UserService userService) {
        this.userService = userService;  // 第一次赋值（初始化）
    }

    public void someMethod() {
        // ❌ 错误：不能再次赋值
        // this.userService = new UserService();  // 编译错误！
    }
}
```

**方式 3：初始化块中初始化**

```java
public class Demo {
    private final int value;

    {
        // 初始化块（在构造函数之前执行）
        this.value = 100;
    }
}
```

#### 4.2 final 修饰局部变量

```java
public void test() {
    // final 修饰局部变量
    final int num = 10;
    // num = 20;  ❌ 不能修改

    // final 局部变量可以先声明后赋值
    final int count;
    count = 5;  // ✅ 第一次赋值
    // count = 10;  ❌ 不能再次赋值
}
```

#### 4.3 final 修饰引用类型变量

```java
public void test() {
    // final 修饰引用类型
    final int[] arr = {1, 2, 3};
    // arr = new int[5];  ❌ 不能修改引用（地址）
    arr[0] = 100;  // ✅ 可以修改内容

    // final 修饰对象引用
    final User user = new User("张三");
    // user = new User("李四");  ❌ 不能修改引用
    user.setName("李四");  // ✅ 可以修改对象属性
}
```

**关键理解**：`final` 修饰引用类型时，**引用（地址）不能改变**，但**对象内容可以改变**

#### 4.4 final 修饰方法参数

```java
public void process(final String name, final int count) {
    // name = "新值";  ❌ 不能修改 final 参数
    // count = 100;    ❌ 不能修改 final 参数
    
    System.out.println(name + ": " + count);
}
```

**使用场景**：防止在方法内部意外修改参数值，增强代码安全性

### 5. 常量的命名规范

```java
// 常量名全大写，多个单词用下划线分隔
public static final int MAX_VALUE = 100;
public static final String DEFAULT_NAME = "未知";
```

**`static final` vs `final`：**

| 修饰符            | 特点                 |
| -------------- | ------------------ |
| `final`        | 实例常量，每个对象一份        |
| `static final` | 类常量，所有对象共享，类加载时初始化 |

### 6. 实际开发应用：Spring 依赖注入（推荐）⭐

```java
@RestController
public class UserController {
    // ✅ final 字段 + 构造函数注入（Spring 推荐方式）
    private final UserService userService;
    private final OrderService orderService;

    // @Autowired 可以省略（Spring 4.3+，只有一个构造函数时）
    public UserController(UserService userService, OrderService orderService) {
        this.userService = userService;
        this.orderService = orderService;
    }
}
```

**为什么推荐？**

*   ✅ 不可变（final）
*   ✅ 线程安全
*   ✅ 强制依赖（必须提供）
*   ✅ 测试友好

### 7. final 总结

| 修饰对象       | 效果              | 说明                                                   |
| ---------- | --------------- | ---------------------------------------------------- |
| **类**      | 不能被继承           | `public final class Math { }`                        |
| **方法**     | 不能被重写           | `public final void sleep() { }`                      |
| **基本类型变量** | 值不能改变           | `final int num = 10;`                                |
| **引用类型变量** | 地址不能改变（内容可以改）   | `final int[] arr = {1,2,3}; arr[0] = 100; ✅`         |
| **成员变量**   | 必须在声明时或构造函数中初始化 | `private final UserService userService;` 在构造函数中初始化 ✅ |
| **方法参数**   | 参数值不能被修改        | `void process(final String name) { }`                |

## 四、抽象类（Abstract Class）

### 1. 什么是抽象类

当父类的某些方法**只知道名字，不知道具体实现**时，可以定义为抽象方法，包含抽象方法的类必须是抽象类。

### 2. 抽象类的定义

```java
// 抽象类
public abstract class Animal {
    private String name;
    
    // 普通成员变量、构造方法、普通方法都可以有
    public Animal(String name) {
        this.name = name;
    }
    
    public String getName() {
        return name;
    }
    
    // 抽象方法：只有方法声明，没有方法体
    public abstract void eat();
    
    public abstract void sleep();
}
```

### 3. 抽象类的使用

```java
// 子类必须重写所有抽象方法（除非子类也是抽象类）
public class Dog extends Animal {
    
    public Dog(String name) {
        super(name);
    }
    
    @Override
    public void eat() {
        System.out.println(getName() + "吃骨头");
    }
    
    @Override
    public void sleep() {
        System.out.println(getName() + "趴着睡");
    }
}
```

### 4. 抽象类的特点

| 特点       | 说明                   |
| -------- | -------------------- |
| 不能实例化    | `new Animal()` ❌ 不允许 |
| 可以有构造方法  | 供子类调用                |
| 可以有普通方法  | 不强制都是抽象的             |
| 可以没有抽象方法 | 抽象类可以不包含抽象方法         |
| 抽象方法     | 必须被子类重写              |
| 子类要求     | 要么重写所有抽象方法，要么自己也是抽象类 |

### 5. 模板方法设计模式 ⭐

抽象类的典型应用：定义算法骨架，将某些步骤延迟到子类实现。

```java
// 抽象类：定义模板
public abstract class AbstractTemplate {
    
    // 模板方法（final 防止子类修改算法骨架）
    public final void templateMethod() {
        step1();        // 固定步骤
        step2();        // 抽象步骤（子类实现）
        hook();         // 钩子方法（子类可选重写）
        step3();        // 固定步骤
    }
    
    // 固定实现
    private void step1() {
        System.out.println("步骤1：准备工作");
    }
    
    // 抽象方法（子类必须实现）
    protected abstract void step2();
    
    // 钩子方法（子类可选重写）
    protected void hook() {
        // 默认空实现
    }
    
    // 固定实现
    private void step3() {
        System.out.println("步骤3：收尾工作");
    }
}

// 具体实现类
public class ConcreteA extends AbstractTemplate {
    @Override
    protected void step2() {
        System.out.println("步骤2：A 的具体实现");
    }
}

public class ConcreteB extends AbstractTemplate {
    @Override
    protected void step2() {
        System.out.println("步骤2：B 的具体实现");
    }
    
    @Override
    protected void hook() {
        System.out.println("B 重写了钩子方法");
    }
}
```

### 6. 抽象类的意义

*   **强制规范**：定义统一的方法名，子类必须实现
*   **代码复用**：普通方法和成员变量可以被继承
*   **多态基础**：可以用抽象类类型接收子类对象
*   **模板设计**：定义算法骨架，延迟具体实现

## 五、接口（Interface）

### 1. 什么是接口

接口是一种**规范**，定义了一组抽象方法，实现类必须实现这些方法。

### 2. 接口的定义

```java
public interface Swimmable {
    // JDK 7 及之前：只能有常量和抽象方法
    
    // 常量（默认 public static final）
    int MAX_SPEED = 100;
    
    // 抽象方法（默认 public abstract）
    void swim();
}

public interface Flyable {
    void fly();
}
```

### 3. 接口的实现

```java
// 使用 implements 实现接口，可以实现多个接口
public class Duck implements Swimmable, Flyable {
    
    @Override
    public void swim() {
        System.out.println("鸭子在游泳");
    }
    
    @Override
    public void fly() {
        System.out.println("鸭子在飞");
    }
}
```

### 4. JDK 8+ 接口新特性

```java
public interface MyInterface {
    // 1. 抽象方法（必须实现）
    void abstractMethod();
    
    // 2. 默认方法（可以不重写，有默认实现）
    default void defaultMethod() {
        privateMethod();  // 可以调用私有方法
        System.out.println("默认方法");
    }
    
    // 3. 静态方法（通过接口名调用）
    static void staticMethod() {
        System.out.println("静态方法");
    }
    
    // 4. 私有方法（JDK 9+，接口内部使用）
    private void privateMethod() {
        System.out.println("私有方法");
    }
    
    // 5. 私有静态方法（JDK 9+）
    private static void privateStaticMethod() {
        System.out.println("私有静态方法");
    }
}

// 调用
MyInterface.staticMethod();  // 静态方法通过接口名调用
```

### 5. 默认方法冲突解决

当类实现多个接口，且接口有同名默认方法时：

```java
interface A {
    default void show() { System.out.println("A"); }
}

interface B {
    default void show() { System.out.println("B"); }
}

// 实现类必须重写冲突的默认方法
class C implements A, B {
    @Override
    public void show() {
        A.super.show();  // 调用 A 的默认方法
        // B.super.show();  // 或调用 B 的默认方法
        System.out.println("C");
    }
}
```

### 6. 标记接口

**标记接口**：没有任何方法的接口，用于标记某种能力。

```java
// JDK 中的标记接口
public interface Serializable { }    // 可序列化
public interface Cloneable { }       // 可克隆
public interface RandomAccess { }    // 支持随机访问

// 使用标记接口
public class Person implements Serializable, Cloneable {
    // ...
}

// 判断是否具有某种能力
if (obj instanceof Serializable) {
    // 可以序列化
}
```

### 7. 接口和类的关系

| 关系      | 说明            | 数量  |
| ------- | ------------- | --- |
| 类 - 类   | extends 继承    | 单继承 |
| 类 - 接口  | implements 实现 | 多实现 |
| 接口 - 接口 | extends 继承    | 多继承 |

```java
// 接口多继承
public interface C extends A, B {
    // 继承 A 和 B 的所有抽象方法
}

// 类可以继承一个类，同时实现多个接口
public class Dog extends Animal implements Swimmable, Runnable {
    // ...
}
```

### 8. 抽象类 vs 接口

| 对比项   | 抽象类            | 接口                         |
| ----- | -------------- | -------------------------- |
| 关键字   | abstract class | interface                  |
| 成员变量  | 可以有普通变量        | 只能有常量（public static final） |
| 构造方法  | 有              | 没有                         |
| 普通方法  | 可以有            | JDK 8 前没有，JDK 8+ 有默认方法     |
| 继承/实现 | 单继承            | 多实现                        |
| 设计理念  | is-a 关系（是一种）   | has-a 能力（有这个能力）            |
| 使用场景  | 有共同的属性和行为      | 定义规范和能力                    |

### 9. 使用建议

*   **抽象类**：描述**是什么**（Dog is an Animal）
*   **接口**：描述**能做什么**（Duck can Swim, can Fly）

## 六、多态（Polymorphism）

### 1. 什么是多态

多态是面向对象三大特征之一，指**同一个行为在不同对象上有不同的表现形式**。

### 2. 多态的前提

1.  要有**继承/实现**关系
2.  要有**方法重写**
3.  要有**父类引用指向子类对象**

### 3. 多态的语法

```java
// 父类类型 变量名 = new 子类对象();
Animal a = new Dog();
a.eat();  // 调用的是 Dog 重写后的 eat 方法
```

### 4. 多态示例

```java
public class Animal {
    public void eat() {
        System.out.println("动物吃东西");
    }
}

public class Dog extends Animal {
    @Override
    public void eat() {
        System.out.println("狗吃骨头");
    }
    
    public void bark() {
        System.out.println("狗在叫");
    }
}

public class Cat extends Animal {
    @Override
    public void eat() {
        System.out.println("猫吃鱼");
    }
}
```

```java
public class Test {
    public static void main(String[] args) {
        // 多态：父类引用指向子类对象
        Animal a1 = new Dog();
        Animal a2 = new Cat();
        
        a1.eat();  // 狗吃骨头
        a2.eat();  // 猫吃鱼
        
        // a1.bark();  ❌ 编译错误！父类引用无法调用子类特有方法
    }
}
```

### 5. 多态的成员访问特点

| 成员   | 编译看 | 运行看          |
| ---- | --- | ------------ |
| 成员变量 | 父类  | 父类           |
| 成员方法 | 父类  | **子类**（重写后的） |

**口诀**：编译看左边，运行看右边（针对方法）

```java
public class Animal {
    String name = "动物";
}

public class Dog extends Animal {
    String name = "狗";
}

Animal a = new Dog();
System.out.println(a.name);  // 动物（成员变量看父类）
a.eat();      // 编译时看 Animal 有没有 eat，运行时执行 Dog 的 eat
// a.bark();  // 编译失败，Animal 没有 bark 方法
```

### 6. 多态的好处和弊端

| 好处                      | 弊端             |
| ----------------------- | -------------- |
| 提高代码扩展性                 | 不能使用子类特有功能     |
| 定义方法时用父类类型作参数，可接收所有子类对象 | 需要向下转型才能调用子类方法 |

### 7. 类型转换

```java
Animal a = new Dog();  // 向上转型（自动）

// 向下转型（强制）
if (a instanceof Dog) {  // 先判断类型
    Dog d = (Dog) a;
    d.bark();  // 可以调用子类特有方法
}

// JDK 14+ 模式匹配（预览），JDK 16+ 正式
if (a instanceof Dog d) {
    d.bark();  // 直接使用，无需手动转型
}

// JDK 21+ switch 模式匹配
String result = switch (a) {
    case Dog d -> "狗：" + d.bark();
    case Cat c -> "猫：" + c.catchMouse();
    default -> "未知动物";
};
```

### 8. 多态的应用场景

```java
// 场景：方法参数使用父类类型
public class AnimalOperator {
    // 可以接收所有 Animal 的子类
    public void feed(Animal animal) {
        animal.eat();
    }
}

// 使用
AnimalOperator op = new AnimalOperator();
op.feed(new Dog());  // 狗吃骨头
op.feed(new Cat());  // 猫吃鱼
```

## 七、代码块

### 1. 代码块分类

| 类型        | 定义位置           | 执行时机             |
| --------- | -------------- | ---------------- |
| **局部代码块** | 方法内            | 方法调用时执行          |
| **构造代码块** | 类中方法外          | 每次创建对象时，在构造方法前执行 |
| **静态代码块** | 类中方法外，加 static | 类加载时执行，只执行一次     |

### 2. 代码示例

```java
public class Demo {
    // 静态代码块：类加载时执行，只执行一次
    static {
        System.out.println("1. 静态代码块");
    }
    
    // 构造代码块：每次创建对象时执行
    {
        System.out.println("2. 构造代码块");
    }
    
    // 构造方法
    public Demo() {
        System.out.println("3. 构造方法");
    }
    
    public void test() {
        // 局部代码块
        {
            int x = 10;
            System.out.println("4. 局部代码块：" + x);
        }
        // System.out.println(x);  ❌ x 在这里不可见
    }
}
```

### 3. 执行顺序测试

```java
public class Test {
    public static void main(String[] args) {
        System.out.println("--- 第一次创建对象 ---");
        Demo d1 = new Demo();
        
        System.out.println("--- 第二次创建对象 ---");
        Demo d2 = new Demo();
    }
}
```

**输出：**

    1. 静态代码块
    --- 第一次创建对象 ---
    2. 构造代码块
    3. 构造方法
    --- 第二次创建对象 ---
    2. 构造代码块
    3. 构造方法

### 4. 继承中的执行顺序

```java
class Parent {
    static { System.out.println("1. 父类静态代码块"); }
    { System.out.println("3. 父类构造代码块"); }
    Parent() { System.out.println("4. 父类构造方法"); }
}

class Child extends Parent {
    static { System.out.println("2. 子类静态代码块"); }
    { System.out.println("5. 子类构造代码块"); }
    Child() { System.out.println("6. 子类构造方法"); }
}

// 执行 new Child() 输出顺序：
// 1. 父类静态代码块
// 2. 子类静态代码块
// 3. 父类构造代码块
// 4. 父类构造方法
// 5. 子类构造代码块
// 6. 子类构造方法
```

### 5. 使用场景

| 代码块类型 | 使用场景                     |
| ----- | ------------------------ |
| 静态代码块 | 加载驱动、初始化静态资源（只需要执行一次的操作） |
| 构造代码块 | 多个构造方法的公共代码提取            |
| 局部代码块 | 限制变量作用域（较少使用）            |

```java
public class JDBCUtils {
    static {
        // 加载数据库驱动（只需执行一次）
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}
```

## 八、内部类

### 1. 什么是内部类

定义在一个类**内部**的类，叫做内部类。

### 2. 内部类的分类

| 类型        | 定义位置           | 特点          |
| --------- | -------------- | ----------- |
| 成员内部类     | 类中方法外          | 可以访问外部类所有成员 |
| 静态内部类     | 类中方法外，加 static | 只能访问外部类静态成员 |
| 局部内部类     | 方法内            | 作用域仅在方法内    |
| **匿名内部类** | 方法内            | 没有名字，用于简化代码 |

### 3. 成员内部类

```java
public class Outer {
    private int num = 10;
    private static int staticNum = 20;
    
    // 成员内部类
    public class Inner {
        private int num = 30;  // 内部类自己的成员
        
        public void show() {
            int num = 40;  // 局部变量
            
            System.out.println(num);            // 40（局部变量）
            System.out.println(this.num);       // 30（内部类成员）
            System.out.println(Outer.this.num); // 10（外部类成员）⭐
        }
    }
}

// 使用
Outer.Inner inner = new Outer().new Inner();
inner.show();
```

### 4. 静态内部类

```java
public class Outer {
    private static int staticNum = 20;
    private int num = 10;
    
    // 静态内部类
    public static class Inner {
        public void show() {
            System.out.println("staticNum = " + staticNum);
            // System.out.println(num);  ❌ 不能访问非静态成员
            
            // 如果需要访问非静态成员，必须创建外部类对象
            Outer outer = new Outer();
            System.out.println("num = " + outer.num);
        }
    }
}

// 使用（不需要外部类对象）
Outer.Inner inner = new Outer.Inner();
inner.show();
```

### 5. 局部内部类

```java
public class Outer {
    public void method() {
        final int localVar = 100;  // 必须是 effectively final
        
        // 局部内部类
        class LocalInner {
            public void show() {
                System.out.println("localVar = " + localVar);
                // localVar = 200;  ❌ 不能修改
            }
        }
        
        LocalInner inner = new LocalInner();
        inner.show();
    }
}
```

**注意**：局部内部类访问局部变量时，该变量必须是 `final` 或 **effectively final**（事实上的 final，即值不会改变）

### 6. 匿名内部类（重点）

**没有名字的内部类**，本质是一个继承了类或实现了接口的**子类匿名对象**。

```java
// 接口
public interface Swimmable {
    void swim();
}

// 使用匿名内部类
public class Test {
    public static void main(String[] args) {
        // 传统方式：需要定义实现类
        // class Duck implements Swimmable { ... }
        // Swimmable s = new Duck();
        
        // 匿名内部类：直接创建实现接口的对象
        Swimmable s = new Swimmable() {
            @Override
            public void swim() {
                System.out.println("游泳中...");
            }
        };
        
        s.swim();  // 游泳中...
    }
}
```

### 7. 匿名内部类的格式

```java
new 类名或接口名() {
    // 重写方法
};
```

### 8. 匿名内部类的使用场景

**作为方法参数传递**：

```java
public class Test {
    public static void main(String[] args) {
        // 场景1：作为参数直接传递
        swim(new Swimmable() {
            @Override
            public void swim() {
                System.out.println("鸭子游泳");
            }
        });
        
        // 场景2：简化按钮点击事件（Swing/Android）
        button.addActionListener(new ActionListener() {
            @Override
            public void actionPerformed(ActionEvent e) {
                System.out.println("按钮被点击");
            }
        });
    }
    
    public static void swim(Swimmable s) {
        s.swim();
    }
}
```

### 9. 匿名内部类访问外部变量

```java
public void method() {
    int count = 10;  // effectively final
    
    Runnable r = new Runnable() {
        @Override
        public void run() {
            System.out.println(count);  // ✅ 可以访问
            // count = 20;  ❌ 不能修改（必须是 effectively final）
        }
    };
    
    // count = 30;  ❌ 如果这里修改，上面的访问也会报错
}
```

## 九、函数式接口（Functional Interface）

### 1. 什么是函数式接口

**函数式接口**是**只包含一个抽象方法**的接口，是 Lambda 表达式的前提条件。

```java
// 函数式接口：只有一个抽象方法
@FunctionalInterface
public interface Swimmable {
    void swim();  // 唯一的抽象方法
}
```

### 2. @FunctionalInterface 注解

| 作用       | 说明               |
| -------- | ---------------- |
| **编译检查** | 检查接口是否只有一个抽象方法   |
| **文档说明** | 明确告诉其他开发者这是函数式接口 |
| **可选的**  | 不加注解也可以是函数式接口    |

```java
// ✅ 正确：只有一个抽象方法
@FunctionalInterface
public interface Calculator {
    int calculate(int a, int b);
}

// ❌ 编译错误：有两个抽象方法
@FunctionalInterface
public interface NotFunctional {
    void method1();
    void method2();  // 编译报错
}
```

### 3. 函数式接口可以包含的内容

```java
@FunctionalInterface
public interface MyInterface {
    // ✅ 唯一的抽象方法（必须）
    void abstractMethod();
    
    // ✅ 可以有默认方法（不影响）
    default void defaultMethod() {
        System.out.println("默认方法");
    }
    
    // ✅ 可以有静态方法（不影响）
    static void staticMethod() {
        System.out.println("静态方法");
    }
    
    // ✅ 可以有私有方法（JDK 9+，不影响）
    private void privateMethod() {
        System.out.println("私有方法");
    }
    
    // ✅ 可以重写 Object 的方法（不影响）
    @Override
    String toString();
    
    @Override
    boolean equals(Object obj);
}
```

### 4. JDK 内置函数式接口

Java 在 `java.util.function` 包中提供了常用的函数式接口：

**四大核心接口：**

| 接口                  | 参数 | 返回值     | 方法          | 用途   |
| ------------------- | -- | ------- | ----------- | ---- |
| **Consumer\<T>**    | T  | void    | `accept(T)` | 消费数据 |
| **Supplier\<T>**    | 无  | T       | `get()`     | 提供数据 |
| **Function\<T, R>** | T  | R       | `apply(T)`  | 转换数据 |
| **Predicate\<T>**   | T  | boolean | `test(T)`   | 判断条件 |

```java
import java.util.function.*;

public class FunctionalDemo {
    public static void main(String[] args) {
        // Consumer：消费数据（有参数，无返回值）
        Consumer<String> consumer = s -> System.out.println(s);
        consumer.accept("Hello");  // 输出：Hello
        
        // Supplier：提供数据（无参数，有返回值）
        Supplier<Double> supplier = () -> Math.random();
        System.out.println(supplier.get());  // 输出随机数
        
        // Function：转换数据（有参数，有返回值）
        Function<String, Integer> function = s -> s.length();
        System.out.println(function.apply("Hello"));  // 5
        
        // Predicate：判断条件（有参数，返回 boolean）
        Predicate<Integer> predicate = n -> n > 10;
        System.out.println(predicate.test(15));  // true
    }
}
```

**双参数版本（Bi 系列）：**

| 接口                       | 参数   | 返回值     | 方法             |
| ------------------------ | ---- | ------- | -------------- |
| **BiConsumer\<T, U>**    | T, U | void    | `accept(T, U)` |
| **BiFunction\<T, U, R>** | T, U | R       | `apply(T, U)`  |
| **BiPredicate\<T, U>**   | T, U | boolean | `test(T, U)`   |

```java
// BiFunction：两个参数，一个返回值
BiFunction<String, String, Integer> biFunc = (a, b) -> a.length() + b.length();
System.out.println(biFunc.apply("Hello", "World"));  // 10

// BiConsumer：两个参数，无返回值
BiConsumer<String, Integer> biConsumer = (name, age) -> 
    System.out.println(name + ": " + age);
biConsumer.accept("张三", 18);  // 张三: 18

// BiPredicate：两个参数，返回 boolean
BiPredicate<String, Integer> biPredicate = (s, n) -> s.length() > n;
System.out.println(biPredicate.test("Hello", 3));  // true
```

**Operator 系列（特殊的 Function）：**

| 接口                     | 说明               | 用途     |
| ---------------------- | ---------------- | ------ |
| **UnaryOperator\<T>**  | T → T            | 一元运算   |
| **BinaryOperator\<T>** | (T, T) → T       | 二元运算   |
| **IntUnaryOperator**   | int → int        | int 一元 |
| **IntBinaryOperator**  | (int, int) → int | int 二元 |

```java
// UnaryOperator：一元运算（输入输出类型相同）
UnaryOperator<String> upper = s -> s.toUpperCase();
System.out.println(upper.apply("hello"));  // HELLO

// BinaryOperator：二元运算（两个输入和输出类型相同）
BinaryOperator<Integer> add = (a, b) -> a + b;
System.out.println(add.apply(10, 20));  // 30

// 常用于 reduce 操作
List<Integer> nums = Arrays.asList(1, 2, 3, 4, 5);
int sum = nums.stream().reduce(0, (a, b) -> a + b);
```

**基本类型特化版本（避免装箱拆箱）：**

| 接口                | 说明            |
| ----------------- | ------------- |
| IntConsumer       | int → void    |
| IntSupplier       | () → int      |
| IntFunction\<R>   | int → R       |
| IntPredicate      | int → boolean |
| ToIntFunction\<T> | T → int       |

### 5. 自定义函数式接口

### 7. 函数式接口的组合

```java
// Predicate 组合
Predicate<Integer> p1 = n -> n > 10;
Predicate<Integer> p2 = n -> n < 20;

Predicate<Integer> and = p1.and(p2);      // n > 10 && n < 20
Predicate<Integer> or = p1.or(p2);        // n > 10 || n < 20
Predicate<Integer> negate = p1.negate();  // !(n > 10)

// Function 组合
Function<String, String> f1 = s -> s.toUpperCase();
Function<String, Integer> f2 = s -> s.length();

Function<String, Integer> composed = f1.andThen(f2);  // 先转大写，再取长度
System.out.println(composed.apply("hello"));  // 5

// Consumer 组合
Consumer<String> c1 = s -> System.out.print("1: " + s);
Consumer<String> c2 = s -> System.out.println(" 2: " + s);
Consumer<String> combined = c1.andThen(c2);
combined.accept("test");  // 1: test 2: test
```

| 对比项        | 函数式接口                    | 普通接口  |
| ---------- | ------------------------ | ----- |
| 抽象方法数量     | 只能有 1 个                  | 可以有多个 |
| 能否用 Lambda | ✅ 可以                     | ❌ 不可以 |
| 注解         | @FunctionalInterface（可选） | 无     |
| 默认/静态方法    | 可以有                      | 可以有   |

## 十、Lambda 表达式

### 1. 什么是 Lambda

Lambda 表达式是 JDK 8 引入的新特性，用于**简化匿名内部类**的写法。

### 2. Lambda 的前提

只能用于**函数式接口**（接口中只有一个抽象方法）

```java
@FunctionalInterface  // 函数式接口注解
public interface Swimmable {
    void swim();  // 只有一个抽象方法
}
```

### 3. Lambda 的语法

```java
(参数列表) -> {
    方法体
}
```

### 4. 从匿名内部类到 Lambda

```java
// 接口
interface Calculator {
    int calc(int a, int b);
}

public class Test {
    public static void main(String[] args) {
        // 1. 匿名内部类写法
        Calculator c1 = new Calculator() {
            @Override
            public int calc(int a, int b) {
                return a + b;
            }
        };
        
        // 2. Lambda 标准写法
        Calculator c2 = (int a, int b) -> {
            return a + b;
        };
        
        // 3. Lambda 简化写法
        Calculator c3 = (a, b) -> a + b;
        
        System.out.println(c3.calc(10, 20));  // 30
    }
}
```

### 5. Lambda 的省略规则

| 可省略项        | 条件            |
| ----------- | ------------- |
| 参数类型        | 所有参数类型都可省略    |
| 小括号         | 只有一个参数时可省略    |
| 大括号和 return | 方法体只有一条语句时可省略 |

```java
// 无参无返回值
Runnable r = () -> System.out.println("运行");

// 一个参数（可省略括号）
Consumer<String> c = s -> System.out.println(s);
// 等价于
Consumer<String> c = (String s) -> { System.out.println(s); };

// 多个参数有返回值
Comparator<Integer> comp = (a, b) -> a - b;
// 等价于
Comparator<Integer> comp = (Integer a, Integer b) -> { return a - b; };
```

### 6. Lambda 变量捕获

Lambda 表达式可以访问外部变量，但变量必须是 **effectively final**：

```java
public void method() {
    int count = 10;  // effectively final（值不变）
    String name = "张三";
    
    Runnable r = () -> {
        System.out.println(count);  // ✅ 可以访问
        System.out.println(name);   // ✅ 可以访问
        // count = 20;  ❌ 不能修改（必须是 effectively final）
    };
    
    // count = 30;  ❌ 如果这里修改，Lambda 中的访问也会报错
}
```

### 7. 方法引用（Method Reference）⭐

方法引用是 Lambda 的进一步简化，当 Lambda 只是调用一个已存在的方法时使用。

**四种形式：**

| 类型           | 语法          | Lambda 等价写法                     |
| ------------ | ----------- | ------------------------------- |
| **静态方法引用**   | `类名::静态方法名` | `(args) -> 类名.静态方法(args)`       |
| **实例方法引用**   | `对象::实例方法名` | `(args) -> 对象.实例方法(args)`       |
| **类的实例方法引用** | `类名::实例方法名` | `(obj, args) -> obj.实例方法(args)` |
| **构造方法引用**   | `类名::new`   | `(args) -> new 类名(args)`        |

```java
// ========== 1. 静态方法引用 ==========
// Lambda
Function<String, Integer> f1 = s -> Integer.parseInt(s);
// 方法引用
Function<String, Integer> f2 = Integer::parseInt;

// ========== 2. 实例方法引用（特定对象的方法）==========
String str = "Hello";
// Lambda
Supplier<Integer> s1 = () -> str.length();
// 方法引用
Supplier<Integer> s2 = str::length;

// ========== 3. 类的实例方法引用（第一个参数作为调用者）==========
// Lambda
BiFunction<String, String, Boolean> bf1 = (a, b) -> a.equals(b);
// 方法引用
BiFunction<String, String, Boolean> bf2 = String::equals;

// Comparator 示例
Comparator<String> comp1 = (a, b) -> a.compareTo(b);
Comparator<String> comp2 = String::compareTo;

// ========== 4. 构造方法引用 ==========
// Lambda
Supplier<ArrayList<String>> sup1 = () -> new ArrayList<>();
// 方法引用
Supplier<ArrayList<String>> sup2 = ArrayList::new;

// 带参数的构造方法
Function<String, StringBuilder> func1 = s -> new StringBuilder(s);
Function<String, StringBuilder> func2 = StringBuilder::new;
```

### 8. 方法引用常用场景

```java
import java.util.*;

public class MethodReferenceDemo {
    public static void main(String[] args) {
        List<String> list = Arrays.asList("banana", "apple", "cherry");
        
        // 场景1：遍历打印
        list.forEach(s -> System.out.println(s));  // Lambda
        list.forEach(System.out::println);          // 方法引用 ✅
        
        // 场景2：排序
        list.sort((a, b) -> a.compareTo(b));  // Lambda
        list.sort(String::compareTo);          // 方法引用 ✅
        
        // 场景3：类型转换
        List<Integer> nums = list.stream()
            .map(s -> s.length())          // Lambda
            .map(String::length)           // 方法引用 ✅
            .toList();
        
        // 场景4：构造对象
        List<StringBuilder> builders = list.stream()
            .map(s -> new StringBuilder(s))  // Lambda
            .map(StringBuilder::new)          // 方法引用 ✅
            .toList();
    }
}
```

### 9. Lambda vs 匿名内部类

| 对比项     | 匿名内部类           | Lambda                |
| ------- | --------------- | --------------------- |
| 适用范围    | 类、抽象类、接口        | 只能用于函数式接口             |
| 语法      | 需要完整的类定义        | 简洁的箭头函数               |
| 编译结果    | 生成独立的 .class 文件 | 不生成额外 .class 文件       |
| this 指向 | 指向匿名内部类对象       | 指向外部类对象               |
| 访问外部变量  | 必须是 final       | 必须是 effectively final |

```java
public class ThisDemo {
    private String name = "外部类";
    
    public void test() {
        // 匿名内部类中的 this
        Runnable r1 = new Runnable() {
            private String name = "匿名内部类";
            @Override
            public void run() {
                System.out.println(this.name);  // 匿名内部类
            }
        };
        
        // Lambda 中的 this
        Runnable r2 = () -> {
            System.out.println(this.name);  // 外部类
        };
    }
}
```
