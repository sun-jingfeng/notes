## 一、Java 背景介绍

### 1. Java 是什么

Java 是一种**面向对象**的**编程语言**，由 Sun Microsystems（后被 Oracle 收购）于 1995 年发布。

### 2. Java 之父

**James Gosling**（詹姆斯·高斯林）

### 3. Java 的发展历程

| 时间     | 事件                                    |
| ------ | ------------------------------------- |
| 1995 年 | Java 1.0 发布                           |
| 2004 年 | Java 5.0 发布（重大更新：泛型、注解、枚举等）           |
| 2009 年 | Oracle 收购 Sun Microsystems            |
| 2014 年 | Java 8 发布（重大更新：Lambda 表达式、Stream API） |
| 2017 年 | Java 9 发布，开始每 6 个月发布新版本               |
| 2018 年 | Java 11（LTS 长期支持版本）                   |
| 2021 年 | Java 17（LTS 长期支持版本）                   |
| 2023 年 | Java 21（LTS 长期支持版本）                   |
| 2024 年 | Java 22/23 发布                         |
| 2025 年 | Java 24/25（预计 25 为下一个 LTS）            |

**版本发布规律**：

*   每 6 个月发布一个新版本（3 月和 9 月）
*   每 2 年发布一个 LTS 版本（长期支持 8 年）

### 4. Java 的特点

| 特点       | 说明                                    |
| -------- | ------------------------------------- |
| **跨平台**  | "一次编写，到处运行"（Write Once, Run Anywhere） |
| **面向对象** | 封装、继承、多态                              |
| **简单易学** | 语法清晰，去掉了 C++ 中复杂的特性（如指针）              |
| **安全性高** | 强类型检查、内存管理、安全机制                       |
| **健壮性**  | 异常处理、垃圾回收                             |
| **多线程**  | 内置多线程支持                               |
| **高性能**  | JIT 即时编译优化                            |

### 5. Java 的三大版本

| 版本          | 全称                 | 用途                              |
| ----------- | ------------------ | ------------------------------- |
| **Java SE** | Standard Edition   | 标准版，桌面应用、基础开发                   |
| **Java EE** | Enterprise Edition | 企业版，Web 开发、企业级应用（现称 Jakarta EE） |
| **Java ME** | Micro Edition      | 微型版，嵌入式、移动设备（现已较少使用）            |

### 6. Java 的应用领域

    Java 应用领域
    ├─ 企业级应用（银行、电商、政府系统）
    ├─ Web 应用（Spring、Spring Boot）
    ├─ Android 开发（Kotlin 逐渐主流，但仍可用 Java）
    ├─ 大数据（Hadoop、Spark、Flink）
    ├─ 分布式系统（Dubbo、Spring Cloud）
    ├─ 云原生（Kubernetes Java 客户端）
    └─ 游戏开发（Minecraft）

## 二、JDK 介绍与选择

### 1. JDK、JRE、JVM 的关系

| 名称      | 全称                       | 说明                  |
| ------- | ------------------------ | ------------------- |
| **JDK** | Java Development Kit     | Java 开发工具包（开发 + 运行） |
| **JRE** | Java Runtime Environment | Java 运行时环境（只能运行）    |
| **JVM** | Java Virtual Machine     | Java 虚拟机（执行字节码）     |

**包含关系**：

    JDK（Java 开发工具包）
    ├─ JRE（Java 运行环境）
    │   ├─ JVM（Java 虚拟机）
    │   └─ 核心类库（rt.jar 等）
    ├─ javac（编译器）
    ├─ java（运行工具）
    ├─ javadoc（文档工具）
    ├─ jar（打包工具）
    ├─ jshell（交互式工具，JDK 9+）
    └─ 其他开发工具

> **注意**：JDK 11 开始，不再单独提供 JRE，JDK 包含了运行所需的一切。

### 2. Oracle JDK vs OpenJDK

| 对比项      | Oracle JDK        | OpenJDK      |
| -------- | ----------------- | ------------ |
| **费用**   | 商用需付费（JDK 17+ 免费） | 完全免费         |
| **开源**   | 部分开源              | 完全开源         |
| **更新周期** | LTS 版本支持更久        | 每个版本仅支持 6 个月 |
| **性能**   | 基本一致              | 基本一致         |
| **推荐场景** | 企业生产环境            | 学习、开发、开源项目   |

### 3. 常见 JDK 发行版

| 发行版                    | 提供商       | 特点             | 推荐场景        |
| ---------------------- | --------- | -------------- | ----------- |
| **Oracle JDK**         | Oracle    | 官方版本，商用需授权     | 企业生产        |
| **Eclipse Temurin**    | Eclipse   | 原 AdoptOpenJDK | 通用推荐        |
| **Amazon Corretto**    | Amazon    | AWS 优化，长期支持    | AWS 云环境     |
| **Azul Zulu**          | Azul      | 多平台支持，有商业版     | 嵌入式、企业      |
| **Microsoft Build**    | Microsoft | Azure 优化       | Azure 云环境   |
| **GraalVM**            | Oracle    | 高性能，支持多语言      | 云原生、微服务     |
| **Alibaba Dragonwell** | 阿里巴巴      | 针对大规模场景优化      | 大规模 Java 应用 |

**初学者推荐**：Eclipse Temurin 或 Oracle JDK

### 4. 版本选择建议

| 场景        | 推荐版本                       |
| --------- | -------------------------- |
| **初学者**   | JDK 17 或 JDK 21（LTS）       |
| **企业开发**  | JDK 8、JDK 11、JDK 17、JDK 21 |
| **新项目**   | JDK 21（最新 LTS）             |
| **学习新特性** | 最新版本（如 JDK 23）             |

### 2. Windows 安装

**方式一：安装包安装**

    1. 下载 JDK 安装包（.exe 或 .msi）
    2. 双击运行安装程序
    3. 选择安装路径（建议不要有中文和空格）
       推荐路径：D:\Java\jdk-17
    4. 点击"下一步"完成安装
    5. 配置环境变量（见后文）

**方式二：Scoop 安装（推荐）**

```powershell
# 安装 Scoop（如果没有）
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
irm get.scoop.sh | iex

# 添加 Java bucket
scoop bucket add java

# 安装 JDK（自动配置环境变量）
scoop install temurin17-jdk

# 切换 JDK 版本
scoop reset temurin17-jdk
scoop reset temurin21-jdk
```

### 3. macOS 安装

**方式一：官方安装包**

```bash
# 下载 .dmg 文件，双击安装
# 安装位置：/Library/Java/JavaVirtualMachines/jdk-17.jdk
```

**方式二：Homebrew 安装**

```bash
# 安装 Homebrew（如果没有）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 JDK 17
brew install openjdk@17

# 创建符号链接
sudo ln -sfn /opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-17.jdk
```

**方式三：SDKMAN 安装（推荐，支持多版本管理）**

```bash
# 安装 SDKMAN
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"

# 查看可用 JDK 版本
sdk list java

# 安装 JDK
sdk install java 17.0.9-tem    # Eclipse Temurin 17
sdk install java 21.0.1-tem    # Eclipse Temurin 21

# 切换默认版本
sdk default java 17.0.9-tem

# 临时切换版本（仅当前终端）
sdk use java 21.0.1-tem
```

## 四、环境变量配置

### 1. 为什么要配置环境变量

    问题：在命令行输入 javac，系统找不到命令

    原因：系统不知道 javac 在哪里

    解决：告诉系统 JDK 的安装位置（配置环境变量）

### 2. Windows 配置环境变量

    1. 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"

    2. 新建系统变量 JAVA_HOME
       变量名：JAVA_HOME
       变量值：D:\Java\jdk-17（JDK 安装路径）

    3. 编辑系统变量 Path
       新建：%JAVA_HOME%\bin

    4. 点击"确定"保存

    5. 打开新的命令行窗口，验证
       java -version
       javac -version

**PowerShell 临时设置**：

```powershell
$env:JAVA_HOME = "D:\Java\jdk-17"
$env:Path = "$env:JAVA_HOME\bin;$env:Path"
```

### 3. macOS 配置环境变量

**编辑配置文件**：

```bash
# Zsh（默认 shell）
nano ~/.zshrc

# 或 Bash
nano ~/.bash_profile
```

**添加以下内容**：

```bash
# Java 环境变量配置
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
# 或者指定具体路径
# export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home

export PATH=$JAVA_HOME/bin:$PATH
```

**使配置生效**：

```bash
source ~/.zshrc
```

**验证**：

```bash
echo $JAVA_HOME
java -version
```

### 4. Linux 配置环境变量

**编辑配置文件**：

```bash
# 用户级别（推荐）
nano ~/.bashrc

# 或系统级别
sudo nano /etc/profile
```

**添加以下内容**：

```bash
# Java 环境变量配置
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
export PATH=$JAVA_HOME/bin:$PATH
```

**使配置生效**：

```bash
source ~/.bashrc
```

### 5. 环境变量说明

| 变量名            | 说明                                   |
| -------------- | ------------------------------------ |
| **JAVA\_HOME** | JDK 安装目录，很多工具（Maven、Gradle、IDEA）需要用到 |
| **PATH**       | 系统命令搜索路径，添加 JDK 的 bin 目录             |

### 6. 多版本切换

**Windows（手动切换）**：

```powershell
# 修改 JAVA_HOME 指向不同版本
# 或使用脚本切换
```

**macOS/Linux（使用 SDKMAN）**：

```bash
# 切换默认版本
sdk default java 17.0.9-tem

# 临时切换（仅当前终端）
sdk use java 21.0.1-tem

# 查看当前版本
sdk current java
```

**macOS（使用 java\_home）**：

```bash
# 查看已安装的 JDK
/usr/libexec/java_home -V

# 临时切换
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
export JAVA_HOME=$(/usr/libexec/java_home -v 21)
```

## 五、第一个 Java 程序 HelloWorld

### 1. 编写代码

创建文件 `HelloWorld.java`：

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

### 2. 代码解析

```java
public class HelloWorld {           // 定义一个公共类，类名必须与文件名一致
    public static void main(String[] args) {  // 主方法，程序入口
        System.out.println("Hello, World!");  // 输出语句
    }
}
```

| 代码部分                                     | 说明           |
| ---------------------------------------- | ------------ |
| `public`                                 | 访问修饰符，公共的    |
| `class`                                  | 关键字，定义类      |
| `HelloWorld`                             | 类名，必须与文件名一致  |
| `public static void main(String[] args)` | 主方法，程序入口     |
| `System.out.println()`                   | 输出语句，打印内容并换行 |

### 3. 编译和运行

**传统方式（先编译后运行）**：

```bash
# 1. 编译：将 .java 文件编译成 .class 文件
javac HelloWorld.java

# 2. 运行：执行 .class 文件（不需要写 .class 后缀）
java HelloWorld

# 输出
Hello, World!
```

**单文件执行（JDK 11+，推荐初学者）**：

```bash
# 直接运行 .java 文件，无需先编译
java HelloWorld.java

# 输出
Hello, World!
```

> **注意**：单文件执行仅适用于单个源文件，多文件项目仍需编译。

### 4. JDK 21+ 简化写法（预览特性）

```java
// 传统写法
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

// JDK 21+ 简化写法（预览特性）
// 无需 public、static、String[] args
void main() {
    System.out.println("Hello, World!");
}

// 运行时需启用预览特性
// java --enable-preview --source 21 HelloWorld.java
```

### 5. jshell 交互式编程（JDK 9+）

```bash
# 启动 jshell
jshell

# 直接输入代码，立即执行
jshell> System.out.println("Hello, World!")
Hello, World!

jshell> int a = 10
a ==> 10

jshell> int b = 20
b ==> 20

jshell> a + b
$3 ==> 30

jshell> String name = "Java"
name ==> "Java"

jshell> name.toUpperCase()
$5 ==> "JAVA"

# 查看变量
jshell> /vars

# 查看方法
jshell> /methods

# 查看历史
jshell> /list

# 退出
jshell> /exit
```

**jshell 用途**：

*   快速测试代码片段
*   学习和实验 API
*   无需创建完整类

### 6. 注意事项

| 注意点 | 说明                    |
| --- | --------------------- |
| 文件名 | 必须与公共类名一致，后缀为 `.java` |
| 类名  | 大驼峰命名（HelloWorld）     |
| 主方法 | 固定写法，一个字都不能错          |
| 分号  | 每条语句以分号 `;` 结尾        |
| 大小写 | Java 区分大小写            |
| 编码  | 建议使用 UTF-8 编码         |

### 7. 常见错误

| 错误类型                | 原因         | 解决方法                       |
| ------------------- | ---------- | -------------------------- |
| `'javac' 不是内部或外部命令` | 环境变量未配置    | 配置 JAVA\_HOME 和 PATH       |
| `找不到文件`             | 路径不对或文件名错误 | 检查当前目录和文件名                 |
| `类 HelloWorld 是公共的` | 文件名与类名不一致  | 保持一致                       |
| `错误: 需要';'`         | 缺少分号       | 添加分号                       |
| `乱码`                | 编码问题       | 使用 `javac -encoding UTF-8` |
| `无法加载主类`            | 包名问题或类路径错误 | 检查包声明和运行目录                 |

## 六、Java 程序执行原理

### 1. 编译执行过程

    源代码（.java）
         ↓ javac 编译
    字节码（.class）
         ↓ java 运行
    JVM 加载并执行
         ↓
    操作系统
         ↓
    硬件（CPU）

### 2. 详细流程

    ┌─────────────────────────────────────────────────────────────┐
    │ 1. 编写源代码                                                │
    │    HelloWorld.java                                          │
    │    public class HelloWorld {                                │
    │        public static void main(String[] args) {             │
    │            System.out.println("Hello, World!");             │
    │        }                                                    │
    │    }                                                        │
    └─────────────────────────────────────────────────────────────┘
                              ↓ javac HelloWorld.java
    ┌─────────────────────────────────────────────────────────────┐
    │ 2. 编译成字节码                                              │
    │    HelloWorld.class                                         │
    │    （二进制文件，JVM 可以执行）                               │
    │    ├─ 类信息、方法信息                                       │
    │    ├─ 常量池                                                │
    │    └─ 字节码指令                                             │
    └─────────────────────────────────────────────────────────────┘
                              ↓ java HelloWorld
    ┌─────────────────────────────────────────────────────────────┐
    │ 3. JVM 加载并执行                                            │
    │    ├─ 类加载器加载 .class 文件                               │
    │    │   ├─ 加载（Loading）                                   │
    │    │   ├─ 链接（Linking）：验证、准备、解析                   │
    │    │   └─ 初始化（Initialization）                          │
    │    ├─ 解释执行 / JIT 编译                                    │
    │    └─ 输出：Hello, World!                                   │
    └─────────────────────────────────────────────────────────────┘

### 3. 为什么 Java 能跨平台？

    源代码 → 字节码（与平台无关）→ 不同平台的 JVM 执行

    Windows JVM ─┐
                 ├─ 执行相同的字节码
    macOS JVM ───┤
                 │
    Linux JVM ───┘

    "一次编写，到处运行"

**关键**：

*   字节码是与平台无关的中间代码
*   不同平台有不同的 JVM 实现
*   JVM 负责将字节码转换为机器码

### 4. JVM 执行方式

| 执行方式       | 说明                  | 特点         |
| ---------- | ------------------- | ---------- |
| **解释执行**   | 逐行解释字节码为机器码         | 启动快，执行慢    |
| **JIT 编译** | 将热点代码编译为机器码并缓存      | 启动慢，执行快    |
| **混合模式**   | 先解释执行，热点代码 JIT 编译   | 平衡启动和执行速度  |
| **AOT 编译** | 提前编译为机器码（GraalVM 等） | 启动极快，无预热时间 |

**JIT（Just-In-Time）编译**：

    ┌─────────────────────────────────────────────────────────────┐
    │                      JIT 编译过程                            │
    ├─────────────────────────────────────────────────────────────┤
    │ 1. 程序启动，解释执行字节码                                   │
    │ 2. JVM 统计方法调用次数（热点探测）                          │
    │ 3. 当方法调用超过阈值（如 10000 次）                         │
    │ 4. JIT 编译器将该方法编译为本地机器码                        │
    │ 5. 后续调用直接执行机器码（无需再解释）                      │
    └─────────────────────────────────────────────────────────────┘

### 5. JVM 的作用

| 功能        | 说明                 |
| --------- | ------------------ |
| **加载字节码** | 将 .class 文件加载到内存   |
| **验证字节码** | 确保代码安全、合法          |
| **执行代码**  | 解释执行或 JIT 编译执行     |
| **内存管理**  | 自动分配和回收内存（垃圾回收 GC） |
| **跨平台**   | 屏蔽不同操作系统的差异        |
| **安全管理**  | 类加载检查、安全沙箱         |

### 6. 编译型 vs 解释型

| 类型         | 说明                   | 语言示例                   |
| ---------- | -------------------- | ---------------------- |
| **编译型**    | 源代码一次性编译成机器码         | C、C++、Go、Rust          |
| **解释型**    | 源代码逐行解释执行            | Python、JavaScript、Ruby |
| **半编译半解释** | 先编译成字节码，再由虚拟机解释/编译执行 | **Java**、C#、Kotlin     |

**Java 的特点**：

*   编译成字节码（javac）
*   JVM 解释执行 + JIT 即时编译
*   兼顾跨平台和性能

## 七、JDK 目录结构

### 1. JDK 安装目录

    jdk-17/
    ├─ bin/              # 可执行文件（javac、java、jshell 等）
    ├─ conf/             # 配置文件
    ├─ include/          # C 语言头文件（JNI 开发用）
    ├─ jmods/            # 模块文件（JDK 9+）
    ├─ legal/            # 法律文件
    ├─ lib/              # 类库文件
    │   ├─ modules       # 模块化的类库（JDK 9+ 取代 rt.jar）
    │   ├─ src.zip       # 源代码
    │   └─ ...
    └─ release           # 版本信息

### 2. bin 目录常用工具

### 3. JDK 9+ 模块化

    JDK 9 之前：
    ├─ rt.jar（包含所有核心类库，约 60MB）
    └─ 应用程序无法按需裁剪

    JDK 9 之后：
    ├─ jmods/（模块化的类库）
    │   ├─ java.base.jmod（核心模块）
    │   ├─ java.sql.jmod（数据库模块）
    │   ├─ java.xml.jmod（XML 模块）
    │   └─ ...（约 70 个模块）
    └─ 可使用 jlink 创建自定义运行时

**创建自定义运行时**：

```bash
# 只包含 java.base 模块，体积更小
jlink --module-path $JAVA_HOME/jmods \
      --add-modules java.base \
      --output custom-jre

# 查看大小（约 30MB vs 完整 JDK 300MB+）
du -sh custom-jre
```

## 九、常见问题

### Q1：学习 Java 应该用哪个版本？

**推荐**：JDK 17 或 JDK 21（LTS 长期支持版本）

### Q2：JDK 和 JRE 有什么区别？

| JDK         | JRE       |
| ----------- | --------- |
| 开发 + 运行     | 只能运行      |
| 包含编译器 javac | 不包含编译器    |
| 开发者需要       | 用户只需要 JRE |

> **注意**：JDK 11 开始不再单独提供 JRE

### Q3：为什么配置 JAVA\_HOME？

*   很多工具（Maven、Gradle、Tomcat、IDEA）需要用到
*   方便切换 JDK 版本
*   规范化管理

### Q4：编译和运行有什么区别？

| 编译             | 运行        |
| -------------- | --------- |
| javac          | java      |
| .java → .class | 执行 .class |
| 检查语法错误         | 执行程序逻辑    |
| 生成字节码          | JVM 加载执行  |

### Q5：Oracle JDK 和 OpenJDK 选哪个？

*   **学习开发**：两者皆可，推荐 Eclipse Temurin（OpenJDK 发行版）
*   **企业生产**：根据公司要求，考虑授权和支持
*   **功能差异**：基本一致，Oracle JDK 有少量商业特性

### Q6：如何管理多个 JDK 版本？

*   **Linux/macOS**：使用 SDKMAN
*   **Windows**：使用 Scoop 或手动切换 JAVA\_HOME
*   **IDEA**：项目级别设置不同 SDK
