## 一、什么是异常

### 1. 异常的概念

异常是程序在**运行过程中**出现的非正常情况，如果不处理，程序会终止运行。

```java
public class Demo {
    public static void main(String[] args) {
        int[] arr = {1, 2, 3};
        System.out.println(arr[10]);  // ❌ 数组越界异常
        System.out.println("程序继续执行");  // 不会执行
    }
}
```

**输出：**

    Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: Index 10 out of bounds for length 3

### 2. 异常 vs 错误

| 对比项    | 异常（Exception）  | 错误（Error）   |
| ------ | -------------- | ----------- |
| **定义** | 程序可以处理的问题      | 程序无法处理的严重问题 |
| **处理** | 可以捕获并处理        | 无法处理，只能避免   |
| **示例** | 空指针、数组越界、文件不存在 | 内存溢出、栈溢出    |
| **建议** | 应该处理           | 不应该捕获       |

## 二、异常体系结构

### 1. 继承关系图

    java.lang.Throwable（可抛出的）
        │
        ├── Error（错误）—— 严重问题，不处理
        │       ├── OutOfMemoryError（内存溢出）
        │       ├── StackOverflowError（栈溢出）
        │       ├── NoClassDefFoundError（类定义未找到）
        │       └── ...
        │
        └── Exception（异常）—— 可以处理
                │
                ├── RuntimeException（运行时异常）—— 可以不处理
                │       ├── NullPointerException（空指针）
                │       ├── ArrayIndexOutOfBoundsException（数组越界）
                │       ├── ClassCastException（类型转换）
                │       ├── NumberFormatException（数字格式）
                │       ├── ArithmeticException（算术异常）
                │       ├── IllegalArgumentException（非法参数）
                │       ├── IllegalStateException（非法状态）
                │       ├── UnsupportedOperationException（不支持的操作）
                │       ├── ConcurrentModificationException（并发修改）
                │       └── ...
                │
                └── 其他 Exception（编译时异常）—— 必须处理
                        ├── IOException（IO异常）
                        ├── FileNotFoundException（文件不存在）
                        ├── SQLException（SQL异常）
                        ├── ParseException（解析异常）
                        ├── ClassNotFoundException（类未找到）
                        ├── InterruptedException（线程中断）
                        └── ...

### 2. 两大分类

| 分类                             | 说明            | 是否必须处理   | 示例         |
| ------------------------------ | ------------- | -------- | ---------- |
| **运行时异常**<br>RuntimeException  | 编译时不检查，运行时才发现 | 可以不处理    | 空指针、数组越界   |
| **编译时异常**<br>Checked Exception | 编译时就检查，必须处理   | **必须处理** | IO异常、SQL异常 |

```java
// 运行时异常：编译通过，运行报错
public void test1() {
    int[] arr = null;
    System.out.println(arr.length);  // 运行时才报 NullPointerException
}

// 编译时异常：编译就报错，必须处理
public void test2() {
    FileInputStream fis = new FileInputStream("a.txt");  // ❌ 编译错误
    // 必须处理 FileNotFoundException
}
```

## 三、异常处理方式

### 方式一：try-catch-finally

#### 1. 基本语法

```java
try {
    // 可能出现异常的代码
} catch (异常类型1 变量名) {
    // 处理异常1
} catch (异常类型2 变量名) {
    // 处理异常2
} finally {
    // 无论是否异常，都会执行（通常用于释放资源）
}
```

#### 2. 执行流程

    try 中的代码
        │
        ├── 无异常 ──→ 跳过 catch ──→ 执行 finally ──→ 继续执行后面代码
        │
        └── 有异常 ──→ 匹配 catch ──→ 执行 catch ──→ 执行 finally ──→ 继续执行后面代码

#### 3. 基本示例

```java
public class TryCatchDemo {
    public static void main(String[] args) {
        try {
            int[] arr = {1, 2, 3};
            System.out.println(arr[10]);  // 这里会出异常
            System.out.println("try中异常后面的代码");  // 不会执行
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("捕获到数组越界异常");
            e.printStackTrace();  // 打印异常信息
        } finally {
            System.out.println("finally 始终执行");
        }
        System.out.println("程序继续执行");
    }
}
```

**输出：**

    捕获到数组越界异常
    java.lang.ArrayIndexOutOfBoundsException: Index 10 out of bounds for length 3
        at TryCatchDemo.main(TryCatchDemo.java:5)
    finally 始终执行
    程序继续执行

#### 4. 多个 catch

```java
public static void main(String[] args) {
    try {
        int[] arr = {1, 2, 3};
        System.out.println(arr[10]);      // 数组越界
        System.out.println(10 / 0);        // 算术异常
    } catch (ArrayIndexOutOfBoundsException e) {
        System.out.println("数组越界异常");
    } catch (ArithmeticException e) {
        System.out.println("算术异常");
    } catch (Exception e) {
        // 父类异常放最后，兜底处理
        System.out.println("其他异常：" + e.getMessage());
    }
}
```

**注意：** 多个 catch 时，**子类异常在前，父类异常在后**

#### 5. JDK 7+ 多异常合并捕获

```java
try {
    // 可能出现异常的代码
} catch (ArrayIndexOutOfBoundsException | ArithmeticException e) {
    // 同时捕获多种异常
    System.out.println("发生异常：" + e.getMessage());
}
```

#### 6. finally 的作用

**finally 中的代码一定会执行**，通常用于释放资源。

```java
FileInputStream fis = null;
try {
    fis = new FileInputStream("a.txt");
    // 读取文件...
} catch (IOException e) {
    e.printStackTrace();
} finally {
    // 释放资源（无论是否异常都要关闭流）
    if (fis != null) {
        try {
            fis.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

#### 7. finally 和 return

```java
public static int test() {
    try {
        return 10;
    } finally {
        System.out.println("finally执行");  // 仍然会执行
    }
}
// 输出：finally执行
// 返回：10
```

**特殊情况：finally 中有 return**

```java
public static int test() {
    try {
        return 10;
    } finally {
        return 20;  // ⚠️ 会覆盖 try 中的 return
    }
}
// 返回：20（不推荐这样写！）
```

#### 8. finally 不执行的情况

```java
// 情况1：System.exit() 终止 JVM
try {
    System.out.println("try");
    System.exit(0);  // JVM 退出
} finally {
    System.out.println("finally");  // 不会执行
}

// 情况2：守护线程被终止
// 当所有非守护线程结束时，守护线程中的 finally 可能不执行

// 情况3：线程被强制终止
// 如 Thread.stop()（已废弃）
```

### 方式二：throws 声明抛出

#### 1. 基本语法

```java
修饰符 返回值类型 方法名(参数列表) throws 异常类型1, 异常类型2 {
    // 方法体
}
```

#### 2. 作用

*   **不处理异常**，将异常抛给调用者处理
*   调用者可以继续 throws，也可以 try-catch 处理

#### 3. 示例

```java
public class ThrowsDemo {
    public static void main(String[] args) {
        try {
            readFile();  // 调用者处理异常
        } catch (IOException e) {
            System.out.println("文件读取失败：" + e.getMessage());
        }
    }
    
    // 方法声明抛出异常，自己不处理
    public static void readFile() throws IOException {
        FileInputStream fis = new FileInputStream("a.txt");
        fis.close();
    }
}
```

#### 4. throws 链

```java
// 一层层往上抛
public static void main(String[] args) throws IOException {
    method1();
}

public static void method1() throws IOException {
    method2();
}

public static void method2() throws IOException {
    FileInputStream fis = new FileInputStream("a.txt");
}

// 最终抛给 JVM，JVM 打印异常信息并终止程序
```

### 方式三：try-with-resources（JDK 7+）

#### 1. 自动关闭资源

```java
// 传统方式：需要在 finally 中手动关闭
FileInputStream fis = null;
try {
    fis = new FileInputStream("a.txt");
    // 使用资源...
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fis != null) {
        try {
            fis.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

// ✅ try-with-resources：自动关闭
try (FileInputStream fis = new FileInputStream("a.txt")) {
    // 使用资源...
} catch (IOException e) {
    e.printStackTrace();
}
// fis 会自动关闭，无需 finally
```

#### 2. 多个资源

```java
try (
    FileInputStream fis = new FileInputStream("a.txt");
    FileOutputStream fos = new FileOutputStream("b.txt")
) {
    // 使用资源...
} catch (IOException e) {
    e.printStackTrace();
}
// fis 和 fos 都会自动关闭（按声明的逆序关闭）
```

#### 3. JDK 9+ 增强

```java
// JDK 9 之前：资源必须在 try() 中声明
FileInputStream fis = new FileInputStream("a.txt");
try (FileInputStream fis2 = fis) {  // 需要额外变量
    // 使用资源...
}

// JDK 9+：可以使用外部声明的 final 或 effectively final 变量
FileInputStream fis = new FileInputStream("a.txt");
BufferedReader br = new BufferedReader(new InputStreamReader(fis));
try (fis; br) {  // ✅ 直接使用外部变量
    // 使用资源...
}
```

#### 4. 要求

资源类必须实现 `AutoCloseable` 或 `Closeable` 接口

```java
// JDK 中的 IO 流都实现了 Closeable 接口
public abstract class InputStream implements Closeable {
    public void close() throws IOException { }
}

// 自定义可关闭资源
public class MyResource implements AutoCloseable {
    @Override
    public void close() throws Exception {
        System.out.println("资源关闭");
    }
}
```

## 四、throw 关键字

### 1. throw vs throws

| 对比项     | throw      | throws      |
| ------- | ---------- | ----------- |
| **位置**  | 方法体内       | 方法声明处       |
| **作用**  | 手动抛出一个异常对象 | 声明方法可能抛出的异常 |
| **数量**  | 一次只能抛一个    | 可以声明多个      |
| **后面跟** | 异常对象       | 异常类型        |

### 2. throw 语法

```java
throw new 异常类型("异常信息");
```

### 3. 使用示例

```java
public class ThrowDemo {
    public static void main(String[] args) {
        try {
            checkAge(-5);
        } catch (IllegalArgumentException e) {
            System.out.println("参数错误：" + e.getMessage());
        }
    }
    
    public static void checkAge(int age) {
        if (age < 0) {
            // 手动抛出异常
            throw new IllegalArgumentException("年龄不能为负数：" + age);
        }
        if (age > 150) {
            throw new IllegalArgumentException("年龄不能超过150：" + age);
        }
        System.out.println("年龄合法：" + age);
    }
}
```

### 4. throw 运行时异常 vs 编译时异常

```java
// 运行时异常：可以不处理
public static void method1(int num) {
    if (num < 0) {
        throw new IllegalArgumentException("参数不能为负");  // ✅ 可以不声明
    }
}

// 编译时异常：必须 throws 声明或 try-catch 处理
public static void method2() throws IOException {
    throw new IOException("IO异常");  // 必须声明或处理
}
```

## 五、异常链

### 1. 什么是异常链

异常链是指一个异常引起另一个异常，通过 `cause` 记录原始异常。

```java
try {
    // 底层操作
} catch (SQLException e) {
    // 包装为业务异常，保留原始异常
    throw new ServiceException("数据库操作失败", e);
}
```

### 2. 构造异常链

```java
// 方式一：通过构造方法
public ServiceException(String message, Throwable cause) {
    super(message, cause);
}

// 使用
throw new ServiceException("操作失败", originalException);

// 方式二：通过 initCause() 方法
ServiceException se = new ServiceException("操作失败");
se.initCause(originalException);
throw se;
```

### 3. 获取原始异常

```java
try {
    service.doSomething();
} catch (ServiceException e) {
    System.out.println("业务异常：" + e.getMessage());
    
    // 获取原始异常
    Throwable cause = e.getCause();
    if (cause != null) {
        System.out.println("原始异常：" + cause.getMessage());
        cause.printStackTrace();
    }
}
```

### 4. 完整示例

```java
public class ExceptionChainDemo {
    public static void main(String[] args) {
        try {
            queryUser(1);
        } catch (ServiceException e) {
            System.out.println("服务异常：" + e.getMessage());
            System.out.println("根本原因：" + e.getCause());
        }
    }
    
    public static void queryUser(int id) {
        try {
            // 模拟数据库查询
            throw new SQLException("数据库连接失败");
        } catch (SQLException e) {
            // 转换为业务异常，保留原始异常
            throw new ServiceException("查询用户失败", e);
        }
    }
}

class ServiceException extends RuntimeException {
    public ServiceException(String message, Throwable cause) {
        super(message, cause);
    }
}
```

## 六、自定义异常

### 1. 为什么需要自定义异常

*   JDK 提供的异常不够用，需要表达特定的业务异常
*   让异常信息更明确，更有意义

### 2. 自定义异常步骤

1.  定义类，继承 `Exception`（编译时异常）或 `RuntimeException`（运行时异常）
2.  提供多个构造方法
3.  添加 serialVersionUID（可选但推荐）

### 3. 标准的自定义异常模板

```java
/**
 * 自定义业务异常
 */
public class BusinessException extends RuntimeException {
    
    // 序列化版本号（推荐添加）
    private static final long serialVersionUID = 1L;
    
    // 错误码（可选）
    private String errorCode;
    
    // 无参构造
    public BusinessException() {
        super();
    }
    
    // 带消息构造
    public BusinessException(String message) {
        super(message);
    }
    
    // 带消息和原因构造（用于异常链）
    public BusinessException(String message, Throwable cause) {
        super(message, cause);
    }
    
    // 带原因构造
    public BusinessException(Throwable cause) {
        super(cause);
    }
    
    // 带错误码和消息构造
    public BusinessException(String errorCode, String message) {
        super(message);
        this.errorCode = errorCode;
    }
    
    public String getErrorCode() {
        return errorCode;
    }
}
```

### 4. 自定义运行时异常

```java
// 自定义异常类
public class AgeException extends RuntimeException {
    
    private static final long serialVersionUID = 1L;
    
    public AgeException() {
    }
    
    public AgeException(String message) {
        super(message);
    }
    
    public AgeException(String message, Throwable cause) {
        super(message, cause);
    }
}

// 使用自定义异常
public class Student {
    private int age;
    
    public void setAge(int age) {
        if (age < 0 || age > 150) {
            throw new AgeException("年龄不合法：" + age);
        }
        this.age = age;
    }
}

// 测试
public static void main(String[] args) {
    Student s = new Student();
    try {
        s.setAge(-5);
    } catch (AgeException e) {
        System.out.println(e.getMessage());  // 年龄不合法：-5
    }
}
```

### 5. 自定义编译时异常

```java
// 自定义编译时异常
public class LoginException extends Exception {
    
    private static final long serialVersionUID = 1L;
    
    public LoginException() {
    }
    
    public LoginException(String message) {
        super(message);
    }
    
    public LoginException(String message, Throwable cause) {
        super(message, cause);
    }
}

// 使用
public class UserService {
    
    public void login(String username, String password) throws LoginException {
        if (!"admin".equals(username)) {
            throw new LoginException("用户名不存在");
        }
        if (!"123456".equals(password)) {
            throw new LoginException("密码错误");
        }
        System.out.println("登录成功");
    }
}

// 调用者必须处理
public static void main(String[] args) {
    UserService service = new UserService();
    try {
        service.login("admin", "wrong");
    } catch (LoginException e) {
        System.out.println("登录失败：" + e.getMessage());
    }
}
```

### 6. 选择继承哪个

| 继承类                | 类型    | 特点    | 使用场景        |
| ------------------ | ----- | ----- | ----------- |
| `RuntimeException` | 运行时异常 | 可以不处理 | 参数校验、业务逻辑异常 |
| `Exception`        | 编译时异常 | 必须处理  | 强制调用者处理的异常  |

**推荐：** 一般自定义异常继承 `RuntimeException`，更灵活

## 七、异常常用方法

### Throwable 类的方法

```java
try {
    int[] arr = {1, 2, 3};
    System.out.println(arr[10]);
} catch (Exception e) {
    
    // 1. getMessage() - 获取异常信息
    String msg = e.getMessage();
    System.out.println(msg);  // Index 10 out of bounds for length 3
    
    // 2. toString() - 获取异常类型和信息
    String str = e.toString();
    System.out.println(str);  // java.lang.ArrayIndexOutOfBoundsException: Index 10...
    
    // 3. printStackTrace() - 打印完整的异常堆栈信息（最常用）
    e.printStackTrace();
    /*
    java.lang.ArrayIndexOutOfBoundsException: Index 10 out of bounds for length 3
        at Demo.main(Demo.java:5)
    */
    
    // 4. getStackTrace() - 获取堆栈跟踪元素数组
    StackTraceElement[] stackTrace = e.getStackTrace();
    for (StackTraceElement element : stackTrace) {
        System.out.println("类名：" + element.getClassName());
        System.out.println("方法名：" + element.getMethodName());
        System.out.println("文件名：" + element.getFileName());
        System.out.println("行号：" + element.getLineNumber());
    }
}
```

### 异常链相关方法

```java
try {
    throw new RuntimeException("外层异常", new IOException("内层异常"));
} catch (RuntimeException e) {
    
    // 5. getCause() - 获取原始异常
    Throwable cause = e.getCause();
    System.out.println(cause);  // java.io.IOException: 内层异常
    
    // 6. initCause() - 设置原始异常（一般在构造时就设置）
    RuntimeException re = new RuntimeException("新异常");
    re.initCause(new IOException("原因"));
}
```

### 被抑制的异常（JDK 7+）

```java
// try-with-resources 中，close() 抛出的异常会被抑制
try (MyResource res = new MyResource()) {
    throw new RuntimeException("主异常");
} catch (Exception e) {
    // 7. getSuppressed() - 获取被抑制的异常
    Throwable[] suppressed = e.getSuppressed();
    for (Throwable t : suppressed) {
        System.out.println("被抑制的异常：" + t.getMessage());
    }
    
    // 8. addSuppressed() - 添加被抑制的异常
    e.addSuppressed(new IOException("额外异常"));
}
```

### 方法速查表

| 方法                         | 返回值                  | 说明          |
| -------------------------- | -------------------- | ----------- |
| `getMessage()`             | String               | 获取异常信息      |
| `toString()`               | String               | 获取异常类型和信息   |
| `printStackTrace()`        | void                 | 打印完整堆栈信息    |
| `getStackTrace()`          | StackTraceElement\[] | 获取堆栈元素数组    |
| `getCause()`               | Throwable            | 获取原始异常（异常链） |
| `initCause(Throwable)`     | Throwable            | 设置原始异常      |
| `getSuppressed()`          | Throwable\[]         | 获取被抑制的异常    |
| `addSuppressed(Throwable)` | void                 | 添加被抑制的异常    |

## 八、常见异常类型

### 1. 运行时异常（RuntimeException）

| 异常类型                              | 说明       | 常见场景           |
| --------------------------------- | -------- | -------------- |
| `NullPointerException`            | 空指针异常    | 对 null 调用方法或属性 |
| `ArrayIndexOutOfBoundsException`  | 数组越界     | 访问不存在的索引       |
| `StringIndexOutOfBoundsException` | 字符串越界    | 字符串索引超出范围      |
| `IndexOutOfBoundsException`       | 索引越界（父类） | 各种索引越界         |
| `ClassCastException`              | 类型转换异常   | 强制类型转换失败       |
| `NumberFormatException`           | 数字格式异常   | 字符串转数字失败       |
| `ArithmeticException`             | 算术异常     | 除以零            |
| `IllegalArgumentException`        | 非法参数异常   | 方法参数不合法        |
| `IllegalStateException`           | 非法状态异常   | 对象状态不正确        |
| `UnsupportedOperationException`   | 不支持的操作   | 调用不支持的方法       |
| `ConcurrentModificationException` | 并发修改异常   | 遍历时修改集合        |

```java
// NullPointerException
String s = null;
s.length();  // ❌ 空指针

// ArrayIndexOutOfBoundsException
int[] arr = {1, 2, 3};
arr[10] = 100;  // ❌ 数组越界

// ClassCastException
Object obj = "Hello";
Integer num = (Integer) obj;  // ❌ 类型转换失败

// NumberFormatException
int n = Integer.parseInt("abc");  // ❌ 不是数字

// ArithmeticException
int result = 10 / 0;  // ❌ 除以零

// UnsupportedOperationException
List<String> list = Arrays.asList("a", "b");
list.add("c");  // ❌ 固定大小列表不支持添加

// ConcurrentModificationException
List<String> list = new ArrayList<>(Arrays.asList("a", "b", "c"));
for (String s : list) {
    list.remove(s);  // ❌ 遍历时修改
}
```

## 九、异常处理原则

### 1. 基本原则

```java
// ❌ 不要捕获异常后什么都不做
try {
    // 可能出异常的代码
} catch (Exception e) {
    // 空的！不推荐
}

// ❌ 不要只打印堆栈而不处理
try {
    // 可能出异常的代码
} catch (Exception e) {
    e.printStackTrace();  // 仅打印，没有真正处理
}

// ✅ 正确处理：记录日志 + 业务处理
try {
    // 可能出异常的代码
} catch (Exception e) {
    logger.error("操作失败", e);  // 记录日志
    // 进行业务处理，如回滚、重试、返回错误信息等
}
```

### 2. 不要捕获过于宽泛的异常

```java
// ❌ 不推荐：捕获所有异常
try {
    // ...
} catch (Exception e) {
    // ...
}

// ❌ 更不推荐：捕获 Throwable
try {
    // ...
} catch (Throwable t) {
    // 连 Error 都捕获了
}

// ✅ 推荐：捕获具体的异常
try {
    // ...
} catch (FileNotFoundException e) {
    // 文件不存在的处理
} catch (IOException e) {
    // 其他 IO 异常的处理
}
```

### 3. 异常转换

```java
// 将底层异常转换为业务异常
public User findUser(int id) {
    try {
        return userDao.findById(id);
    } catch (SQLException e) {
        // 转换为业务异常，隐藏底层实现
        throw new ServiceException("查询用户失败", e);
    }
}
```

### 4. 使用 try-with-resources

```java
// ❌ 传统方式：繁琐且容易出错
FileInputStream fis = null;
try {
    fis = new FileInputStream("a.txt");
    // ...
} catch (IOException e) {
    // ...
} finally {
    if (fis != null) {
        try {
            fis.close();
        } catch (IOException e) {
            // ...
        }
    }
}

// ✅ 推荐：try-with-resources
try (FileInputStream fis = new FileInputStream("a.txt")) {
    // ...
} catch (IOException e) {
    // ...
}
```

### 5. 方法重写与异常

```java
class Parent {
    public void method() throws IOException {
        // ...
    }
}

class Child extends Parent {
    // ✅ 可以不抛异常
    @Override
    public void method() {
        // ...
    }
    
    // ✅ 可以抛相同异常
    @Override
    public void method() throws IOException {
        // ...
    }
    
    // ✅ 可以抛子类异常
    @Override
    public void method() throws FileNotFoundException {
        // ...
    }
    
    // ❌ 不能抛更大的异常
    @Override
    public void method() throws Exception {  // 编译错误
        // ...
    }
}
```

**规则：** 子类重写方法时，抛出的异常**不能比父类更大**

### 6. 提前校验避免异常

```java
// ❌ 依赖异常处理
try {
    return list.get(index);
} catch (IndexOutOfBoundsException e) {
    return null;
}

// ✅ 提前校验
if (index >= 0 && index < list.size()) {
    return list.get(index);
}
return null;

// ❌ 依赖异常判断 null
try {
    obj.doSomething();
} catch (NullPointerException e) {
    // 处理 null
}

// ✅ 提前判断 null
if (obj != null) {
    obj.doSomething();
}
```

### 7. 异常信息要有意义

```java
// ❌ 信息不明确
throw new IllegalArgumentException("参数错误");

// ✅ 信息明确
throw new IllegalArgumentException("用户ID不能为负数：" + userId);

// ✅ 更好：包含上下文
throw new IllegalArgumentException(
    String.format("用户ID不能为负数，当前值：%d，方法：%s", userId, "findUser")
);
```

## 十、综合案例

### 案例：用户注册

```java
// 1. 自定义异常
public class RegisterException extends RuntimeException {
    
    private static final long serialVersionUID = 1L;
    private String errorCode;
    
    public RegisterException(String message) {
        super(message);
    }
    
    public RegisterException(String errorCode, String message) {
        super(message);
        this.errorCode = errorCode;
    }
    
    public RegisterException(String message, Throwable cause) {
        super(message, cause);
    }
    
    public String getErrorCode() {
        return errorCode;
    }
}

// 2. 用户服务类
public class UserService {
    // 模拟数据库中已存在的用户名
    private static final String[] EXISTING_USERS = {"admin", "zhangsan", "lisi"};
    
    /**
     * 用户注册
     * @param username 用户名
     * @param password 密码
     */
    public void register(String username, String password) {
        // 1. 校验用户名
        if (username == null || username.trim().isEmpty()) {
            throw new RegisterException("E001", "用户名不能为空");
        }
        if (username.length() < 3 || username.length() > 20) {
            throw new RegisterException("E002", "用户名长度必须在3-20之间");
        }
        
        // 2. 校验密码
        if (password == null || password.length() < 6) {
            throw new RegisterException("E003", "密码长度不能少于6位");
        }
        
        // 3. 检查用户名是否已存在
        for (String user : EXISTING_USERS) {
            if (user.equals(username)) {
                throw new RegisterException("E004", "用户名已存在：" + username);
            }
        }
        
        // 4. 模拟保存到数据库
        try {
            saveToDatabase(username, password);
        } catch (Exception e) {
            throw new RegisterException("保存用户失败", e);
        }
        
        System.out.println("注册成功：" + username);
    }
    
    private void saveToDatabase(String username, String password) throws Exception {
        // 模拟数据库操作
        System.out.println("保存用户到数据库...");
    }
}

// 3. 测试类
public class RegisterTest {
    public static void main(String[] args) {
        UserService service = new UserService();
        
        // 测试各种情况
        testRegister(service, null, "123456");       // 用户名为空
        testRegister(service, "ab", "123456");       // 用户名太短
        testRegister(service, "admin", "123456");    // 用户名已存在
        testRegister(service, "wangwu", "123");      // 密码太短
        testRegister(service, "wangwu", "123456");   // 注册成功
    }
    
    private static void testRegister(UserService service, String username, String password) {
        try {
            service.register(username, password);
        } catch (RegisterException e) {
            String errorCode = e.getErrorCode();
            String msg = errorCode != null 
                ? String.format("[%s] %s", errorCode, e.getMessage())
                : e.getMessage();
            System.out.println("注册失败：" + msg);
            
            // 如果有原始异常，打印
            if (e.getCause() != null) {
                System.out.println("  原因：" + e.getCause().getMessage());
            }
        }
    }
}
```

    注册失败：[E002] 用户名长度必须在3-20之间
    注册失败：[E004] 用户名已存在：admin
    注册失败：[E003] 密码长度不能少于6位
    保存用户到数据库...
    注册成功：wangwu

## 十一、快速参考表

### 关键字

| 关键字       | 位置           | 作用            |
| --------- | ------------ | ------------- |
| `try`     | 包裹可能出异常的代码   | 尝试执行          |
| `catch`   | try 后面       | 捕获并处理异常       |
| `finally` | try-catch 后面 | 无论如何都执行（释放资源） |
| `throw`   | 方法体内         | 手动抛出异常对象      |
| `throws`  | 方法声明处        | 声明方法可能抛出的异常   |

### 异常分类

| 异常类型             | 继承关系      | 是否必须处理   |
| ---------------- | --------- | -------- |
| Error            | Throwable | 不处理      |
| RuntimeException | Exception | 可以不处理    |
| 其他 Exception     | Exception | **必须处理** |

### 常用方法

| 方法                  | 作用        |
| ------------------- | --------- |
| `getMessage()`      | 获取异常信息    |
| `toString()`        | 获取异常类型和信息 |
| `printStackTrace()` | 打印完整堆栈信息  |
| `getCause()`        | 获取原始异常    |
| `getStackTrace()`   | 获取堆栈元素数组  |
| `getSuppressed()`   | 获取被抑制的异常  |

### 异常处理选择

| 情况      | 建议                 |
| ------- | ------------------ |
| 能处理的异常  | try-catch 处理       |
| 无法处理的异常 | throws 向上抛         |
| 需要关闭资源  | try-with-resources |
| 参数校验失败  | throw 运行时异常        |
| 业务逻辑异常  | 自定义运行时异常           |
| 强制调用者处理 | 自定义编译时异常           |
