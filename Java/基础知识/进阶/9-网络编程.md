## 一、网络编程基本概念

### 1. 什么是网络编程

网络编程是指通过网络让**不同计算机上的程序**进行数据交互。

    ┌─────────────┐                          ┌─────────────┐
    │   计算机A    │  ←───── 网络 ─────→      │   计算机B    │
    │  （客户端）   │        数据传输          │  （服务器）   │
    └─────────────┘                          └─────────────┘

### 2. 软件架构

| 架构      | 说明                      | 优点       | 缺点      | 示例       |
| ------- | ----------------------- | -------- | ------- | -------- |
| **C/S** | Client/Server（客户端/服务器）  | 体验好、安全   | 需要安装客户端 | QQ、微信、游戏 |
| **B/S** | Browser/Server（浏览器/服务器） | 无需安装、跨平台 | 体验相对较差  | 网页应用     |

### 3. 网络模型

#### OSI 七层模型 vs TCP/IP 四层模型

    OSI 七层模型              TCP/IP 四层模型           对应协议
    ┌─────────────┐          ┌─────────────┐
    │   应用层    │          │             │
    ├─────────────┤          │   应用层    │         HTTP、FTP、DNS、SMTP
    │   表示层    │          │             │
    ├─────────────┤          └─────────────┘
    │   会话层    │          
    ├─────────────┤          ┌─────────────┐
    │   传输层    │          │   传输层    │         TCP、UDP
    ├─────────────┤          └─────────────┘
    │   网络层    │          ┌─────────────┐
    ├─────────────┤          │   网络层    │         IP、ICMP
    │  数据链路层  │          └─────────────┘
    ├─────────────┤          ┌─────────────┐
    │   物理层    │          │  网络接口层  │         以太网、Wi-Fi
    └─────────────┘          └─────────────┘

## 二、网络编程三要素

### 1. IP 地址

**作用**：标识网络中的设备（计算机）

| 分类       | 说明                  | 示例                               |
| -------- | ------------------- | -------------------------------- |
| **IPv4** | 4 个字节，32 位，点分十进制    | `192.168.1.100`                  |
| **IPv6** | 16 个字节，128 位，冒分十六进制 | `2001:0db8:85a3::8a2e:0370:7334` |

#### 特殊 IP 地址

| IP 地址                     | 说明                |
| ------------------------- | ----------------- |
| `127.0.0.1`               | 本机回环地址（localhost） |
| `192.168.x.x`             | 局域网地址（C 类私有地址）    |
| `10.x.x.x`                | 局域网地址（A 类私有地址）    |
| `172.16.x.x ~ 172.31.x.x` | 局域网地址（B 类私有地址）    |
| `255.255.255.255`         | 广播地址              |
| `0.0.0.0`                 | 表示任意地址（服务器绑定时使用）  |

#### Java 中的 InetAddress 类

```java
import java.net.InetAddress;

public class Demo {
    public static void main(String[] args) throws Exception {
        // 获取本机 IP
        InetAddress localHost = InetAddress.getLocalHost();
        System.out.println("主机名：" + localHost.getHostName());
        System.out.println("IP地址：" + localHost.getHostAddress());
        
        // 获取回环地址（127.0.0.1）
        InetAddress loopback = InetAddress.getLoopbackAddress();
        System.out.println("回环地址：" + loopback.getHostAddress());
        
        // 通过域名获取 IP（返回第一个）
        InetAddress baidu = InetAddress.getByName("www.baidu.com");
        System.out.println("百度IP：" + baidu.getHostAddress());
        
        // 获取域名的所有 IP（负载均衡场景）
        InetAddress[] allIPs = InetAddress.getAllByName("www.baidu.com");
        for (InetAddress ip : allIPs) {
            System.out.println("IP：" + ip.getHostAddress());
        }
        
        // 判断是否可达（ping），超时时间 3000ms
        System.out.println("是否可达：" + baidu.isReachable(3000));
    }
}
```

#### InetAddress 常用方法

| 方法                          | 说明                     |
| --------------------------- | ---------------------- |
| `getLocalHost()`            | 获取本机 IP 地址             |
| `getLoopbackAddress()`      | 获取回环地址（127.0.0.1）      |
| `getByName(String host)`    | 根据主机名/IP获取 InetAddress |
| `getAllByName(String host)` | 获取域名的所有 IP（用于负载均衡）     |
| `getHostName()`             | 获取主机名                  |
| `getHostAddress()`          | 获取 IP 地址字符串            |
| `isReachable(int timeout)`  | 测试是否可达                 |

### 2. 端口号

**作用**：标识设备上的应用程序

| 说明         | 范围             |
| ---------- | -------------- |
| 端口号是一个整数   | 0 \~ 65535     |
| 系统端口（知名端口） | 0 \~ 1023      |
| 注册端口       | 1024 \~ 49151  |
| 动态/私有端口    | 49152 \~ 65535 |

#### 常见端口

| 端口   | 应用         |
| ---- | ---------- |
| 21   | FTP        |
| 22   | SSH        |
| 23   | Telnet     |
| 25   | SMTP（邮件发送） |
| 80   | HTTP       |
| 443  | HTTPS      |
| 3306 | MySQL      |
| 6379 | Redis      |
| 8080 | Tomcat     |

**注意**：同一设备上，端口号不能重复

***

### 3. 协议

**作用**：规定数据传输的规则

| 协议      | 特点            | 类比            |
| ------- | ------------- | ------------- |
| **UDP** | 无连接、不可靠、速度快   | 发短信（不管对方收没收到） |
| **TCP** | 面向连接、可靠、速度相对慢 | 打电话（先建立连接）    |

***

### 三要素总结

    网络通信 = IP地址（找到哪台电脑） + 端口号（找到哪个程序） + 协议（怎么传数据）

    类比：
    - IP地址 = 小区地址
    - 端口号 = 门牌号
    - 协议   = 快递公司（顺丰/普通快递）

### 2. 命令详解

#### ipconfig - 查看 IP 配置

```bash
# 查看简要 IP 配置
ipconfig

# 查看详细 IP 配置（包括 MAC 地址、DNS 等）
ipconfig /all

# 释放 IP 地址（DHCP）
ipconfig /release

# 重新获取 IP 地址（DHCP）
ipconfig /renew

# 刷新 DNS 缓存
ipconfig /flushdns
```

**输出示例**：

    以太网适配器 本地连接:
       IPv4 地址 . . . . . . . . . . : 192.168.1.100

       默认网关. . . . . . . . . . . : 192.168.1.1

***

#### ping - 测试网络连通性

```bash
# 测试能否连通百度
ping www.baidu.com

# 指定 ping 次数（4 次）
ping -n 4 www.baidu.com

# 持续 ping（Ctrl+C 停止）
ping -t www.baidu.com

# 指定数据包大小（字节）
ping -l 1000 www.baidu.com

# ping 本机（测试网卡是否正常）
ping 127.0.0.1
```

**输出示例**：

    正在 Ping www.baidu.com [110.242.68.66] 具有 32 字节的数据:
    来自 110.242.68.66 的回复: 字节=32 时间=15ms TTL=52
    来自 110.242.68.66 的回复: 字节=32 时间=14ms TTL=52

**常见问题诊断**：

| 现象      | 可能原因          |
| ------- | ------------- |
| 请求超时    | 目标主机不可达或防火墙阻止 |
| TTL 过期  | 路由跳数过多        |
| 目标主机不可达 | 路由问题或目标不存在    |

## 四、UDP 协议收发数据

### 1. UDP 特点

| 特点       | 说明           |
| -------- | ------------ |
| **无连接**  | 发送前不需要建立连接   |
| **不可靠**  | 不保证数据一定送达    |
| **速度快**  | 没有连接建立的开销    |
| **大小限制** | 每个数据包最大 64KB |

**适用场景**：视频通话、直播、游戏（允许丢包）

***

### 2. UDP 相关类

| 类                | 说明               |
| ---------------- | ---------------- |
| `DatagramSocket` | 数据报套接字，用于发送和接收数据 |
| `DatagramPacket` | 数据报包，封装要发送/接收的数据 |

***

### 3. UDP 发送数据

```java
import java.net.DatagramPacket;
import java.net.DatagramSocket;
import java.net.InetAddress;

public class UDPSender {
    public static void main(String[] args) {
        // 使用 try-with-resources 自动关闭资源
        try (DatagramSocket socket = new DatagramSocket()) {
            // 1. 准备数据
            String message = "你好，UDP！";
            byte[] data = message.getBytes();
            
            // 2. 创建 DatagramPacket（数据包）
            //    参数：数据、长度、目标IP、目标端口
            DatagramPacket packet = new DatagramPacket(
                data,
                data.length,
                InetAddress.getByName("127.0.0.1"),
                10086
            );
            
            // 3. 发送数据
            socket.send(packet);
            System.out.println("数据已发送");
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

***

### 4. UDP 接收数据

```java
import java.net.DatagramPacket;
import java.net.DatagramSocket;

public class UDPReceiver {
    public static void main(String[] args) {
        // 使用 try-with-resources 自动关闭资源
        try (DatagramSocket socket = new DatagramSocket(10086)) {
            System.out.println("接收端启动，等待数据...");
            
            // 1. 创建数据包（用于接收数据）
            byte[] buffer = new byte[1024];
            DatagramPacket packet = new DatagramPacket(buffer, buffer.length);
            
            // 2. 接收数据（阻塞方法，等待数据到来）
            socket.receive(packet);
            
            // 3. 解析数据
            String message = new String(
                packet.getData(),      // 数据
                0,                     // 起始位置
                packet.getLength()     // 实际长度
            );
            String senderIP = packet.getAddress().getHostAddress();
            int senderPort = packet.getPort();
            
            System.out.println("收到来自 " + senderIP + ":" + senderPort + " 的消息：" + message);
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

***

### 5. UDP 通信流程图

    发送端                                     接收端
    ┌─────────────────┐                      ┌─────────────────┐
    │ DatagramSocket  │                      │ DatagramSocket  │
    │   （任意端口）   │                      │  （绑定端口）    │
    └────────┬────────┘                      └────────┬────────┘
             │                                        │
             │  1. 创建数据包                          │  1. 创建空数据包
             ↓                                        ↓
    ┌─────────────────┐                      ┌─────────────────┐
    │ DatagramPacket  │                      │ DatagramPacket  │
    │ (数据+目标地址)  │───── UDP传输 ────────→│   (接收数据)     │
    └─────────────────┘                      └─────────────────┘
             │                                        │
             │  2. send(packet)                       │  2. receive(packet)
             ↓                                        ↓
          发送完成                                  解析数据

***

### 6. UDP 广播与组播

#### 广播（Broadcast）

向同一网段的所有设备发送数据。

```java
// 发送广播
try (DatagramSocket socket = new DatagramSocket()) {
    socket.setBroadcast(true);  // 允许广播
    
    String message = "这是广播消息";
    byte[] data = message.getBytes();
    
    // 广播地址：255.255.255.255 或 网段广播地址如 192.168.1.255
    DatagramPacket packet = new DatagramPacket(
        data, data.length,
        InetAddress.getByName("255.255.255.255"),
        10086
    );
    
    socket.send(packet);
}
```

#### 组播（Multicast）

向特定组内的设备发送数据（组播地址：224.0.0.0 \~ 239.255.255.255）。

```java
import java.net.InetAddress;
import java.net.MulticastSocket;

// 组播发送
try (MulticastSocket socket = new MulticastSocket()) {
    String message = "这是组播消息";
    byte[] data = message.getBytes();
    
    DatagramPacket packet = new DatagramPacket(
        data, data.length,
        InetAddress.getByName("224.0.0.1"),  // 组播地址
        10086
    );
    
    socket.send(packet);
}

// 组播接收
try (MulticastSocket socket = new MulticastSocket(10086)) {
    // 加入组播组
    InetAddress group = InetAddress.getByName("224.0.0.1");
    socket.joinGroup(group);
    
    byte[] buffer = new byte[1024];
    DatagramPacket packet = new DatagramPacket(buffer, buffer.length);
    socket.receive(packet);
    
    // 离开组播组
    socket.leaveGroup(group);
}
```

| 类型 | 说明      | 地址范围                         |
| -- | ------- | ---------------------------- |
| 单播 | 一对一通信   | 普通 IP                        |
| 广播 | 一对所有    | 255.255.255.255              |
| 组播 | 一对多（组内） | 224.0.0.0 \~ 239.255.255.255 |

## 五、TCP 协议收发数据

### 1. TCP 特点

| 特点       | 说明               |
| -------- | ---------------- |
| **面向连接** | 先建立连接，再传输数据      |
| **可靠传输** | 保证数据不丢失、不重复、按序到达 |
| **三次握手** | 建立连接需要三次握手       |
| **四次挥手** | 断开连接需要四次挥手       |

**适用场景**：文件传输、网页浏览、邮件（不允许丢包）

***

### 2. 三次握手

    客户端                                    服务器
       │                                        │
       │ ─────── 1. SYN（请求连接） ────────→   │
       │                                        │
       │ ←────── 2. SYN+ACK（同意连接） ─────   │
       │                                        │
       │ ─────── 3. ACK（确认） ────────────→   │
       │                                        │
       │           连接建立成功                   │

**口诀**：

1.  客户端：我要连接你（SYN）
2.  服务器：好的，我同意（SYN+ACK）
3.  客户端：那我们开始吧（ACK）

***

### 3. 四次挥手

    客户端                                    服务器
       │                                        │
       │ ─────── 1. FIN（我要断开） ─────────→  │
       │                                        │
       │ ←────── 2. ACK（收到） ──────────────  │
       │                                        │
       │ ←────── 3. FIN（我也要断开） ────────  │
       │                                        │
       │ ─────── 4. ACK（收到） ─────────────→  │
       │                                        │
       │           连接断开                       │

**口诀**：

1.  客户端：我要断开了
2.  服务器：好的，收到
3.  服务器：我也准备好断开了
4.  客户端：好的，再见

> 💡 **为什么是四次挥手而不是三次？**
> 因为 TCP 是全双工通信，双方都可以发送数据。客户端说"我不发了"，服务器可能还有数据要发，所以需要分别关闭两个方向的连接。

***

### 4. TCP 相关类

| 类              | 说明                    |
| -------------- | --------------------- |
| `Socket`       | 客户端套接字，用于连接服务器、收发数据   |
| `ServerSocket` | 服务器套接字，用于监听端口、接受客户端连接 |

***

### 5. TCP 客户端

```java
import java.io.*;
import java.net.Socket;

public class TCPClient {
    public static void main(String[] args) {
        // 使用 try-with-resources 自动关闭资源
        try (Socket socket = new Socket("127.0.0.1", 10086)) {
            System.out.println("连接服务器成功！");
            
            // 获取输出流，发送数据
            OutputStream os = socket.getOutputStream();
            os.write("你好，服务器！".getBytes());
            
            // 获取输入流，接收服务器响应
            InputStream is = socket.getInputStream();
            byte[] buffer = new byte[1024];
            int len = is.read(buffer);
            String response = new String(buffer, 0, len);
            System.out.println("收到服务器响应：" + response);
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

***

### 6. TCP 服务器

```java
import java.io.*;
import java.net.ServerSocket;
import java.net.Socket;

public class TCPServer {
    public static void main(String[] args) {
        try (ServerSocket serverSocket = new ServerSocket(10086)) {
            System.out.println("服务器启动，等待连接...");
            
            // 监听客户端连接（阻塞方法）
            try (Socket socket = serverSocket.accept()) {
                String clientIP = socket.getInetAddress().getHostAddress();
                System.out.println("客户端 " + clientIP + " 已连接");
                
                // 获取输入流，接收客户端数据
                InputStream is = socket.getInputStream();
                byte[] buffer = new byte[1024];
                int len = is.read(buffer);
                String message = new String(buffer, 0, len);
                System.out.println("收到客户端消息：" + message);
                
                // 获取输出流，响应客户端
                OutputStream os = socket.getOutputStream();
                os.write("消息已收到！".getBytes());
            }
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### 9. Socket 常用配置

| 方法                          | 说明                                       |
| --------------------------- | ---------------------------------------- |
| `setSoTimeout(int ms)`      | 设置读取超时时间（毫秒），超时抛出 SocketTimeoutException |
| `setKeepAlive(boolean)`     | 开启心跳检测，长时间无数据时检测连接是否存活                   |
| `setTcpNoDelay(boolean)`    | 禁用 Nagle 算法，减少延迟（适合小数据实时传输）              |
| `setSendBufferSize(int)`    | 设置发送缓冲区大小                                |
| `setReceiveBufferSize(int)` | 设置接收缓冲区大小                                |
| `shutdownOutput()`          | 半关闭：关闭输出流，仍可接收数据                         |
| `shutdownInput()`           | 半关闭：关闭输入流，仍可发送数据                         |

```java
Socket socket = new Socket("127.0.0.1", 10086);

// 设置读取超时（5秒无数据则超时）
socket.setSoTimeout(5000);

// 开启心跳检测
socket.setKeepAlive(true);

// 禁用 Nagle 算法（适合实时通信）
socket.setTcpNoDelay(true);
```

### 10. 半关闭（Half-Close）

当客户端发送完数据后，需要通知服务器"我发完了"，但仍需要接收服务器响应。

```java
// 客户端
try (Socket socket = new Socket("127.0.0.1", 10086)) {
    OutputStream os = socket.getOutputStream();
    InputStream is = socket.getInputStream();
    
    // 发送数据
    os.write("大量数据...".getBytes());
    
    // 半关闭：告诉服务器"我发完了"
    socket.shutdownOutput();
    
    // 仍然可以接收服务器响应
    byte[] buffer = new byte[1024];
    int len = is.read(buffer);
    System.out.println("响应：" + new String(buffer, 0, len));
}
```

```java
// 服务器
try (Socket socket = serverSocket.accept()) {
    InputStream is = socket.getInputStream();
    OutputStream os = socket.getOutputStream();
    
    // 读取直到客户端关闭输出流
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    byte[] buffer = new byte[1024];
    int len;
    while ((len = is.read(buffer)) != -1) {
        baos.write(buffer, 0, len);
    }
    System.out.println("收到：" + baos.toString());
    
    // 发送响应
    os.write("处理完成！".getBytes());
}
```

### 11. TCP 服务器处理多个客户端

```java
import java.io.*;
import java.net.ServerSocket;
import java.net.Socket;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class TCPServerMulti {
    public static void main(String[] args) {
        // 使用线程池处理客户端连接
        ExecutorService threadPool = Executors.newFixedThreadPool(10);
        
        try (ServerSocket serverSocket = new ServerSocket(10086)) {
            System.out.println("服务器启动，等待连接...");
            
            // 循环接受多个客户端
            while (true) {
                Socket socket = serverSocket.accept();
                System.out.println("客户端 " + socket.getInetAddress().getHostAddress() + " 已连接");
                
                // 提交到线程池处理
                threadPool.submit(() -> handleClient(socket));
            }
            
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            threadPool.shutdown();
        }
    }
    
    private static void handleClient(Socket socket) {
        try (socket;
             BufferedReader reader = new BufferedReader(
                 new InputStreamReader(socket.getInputStream()));
             PrintWriter writer = new PrintWriter(
                 socket.getOutputStream(), true)) {
            
            String message;
            while ((message = reader.readLine()) != null) {
                System.out.println("收到消息：" + message);
                writer.println("服务器已收到：" + message);
            }
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 六、UDP vs TCP 对比

| 对比项        | UDP            | TCP                 |
| ---------- | -------------- | ------------------- |
| **连接**     | 无连接            | 面向连接（三次握手）          |
| **可靠性**    | 不可靠            | 可靠                  |
| **速度**     | 快              | 相对慢                 |
| **数据大小**   | 有限制（64KB）      | 无限制                 |
| **传输方式**   | 数据报            | 字节流                 |
| **资源消耗**   | 少              | 多                   |
| **应用场景**   | 直播、视频、游戏       | 文件传输、网页、邮件          |
| **Java 类** | DatagramSocket | Socket、ServerSocket |

        ├── CMD 命令
        │     ├── ipconfig：查看 IP 配置
        │     ├── ping：测试连通性
        │     ├── netstat -ano：查看端口占用
